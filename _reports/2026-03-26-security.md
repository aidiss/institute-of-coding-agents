## Incidents & Vulnerabilities

### MCP Sampling: New Prompt Injection Attack Surface (Unit 42 / Palo Alto Networks)

Palo Alto Networks Unit 42 published research on a previously under-examined MCP feature — **MCP Sampling** — that reverses the standard interaction model: instead of clients calling servers, servers can proactively initiate LLM completion requests back to the client. This implicit trust inversion enables three new attack vectors when a malicious MCP server is connected:

1. **Resource theft** — unauthorized sampling requests drain AI compute quotas for attacker workloads.
2. **Conversation hijacking** — injected sampling requests persist instructions, alter AI responses, or exfiltrate sensitive data across a session.
3. **Covert tool invocation** — hidden file system operations and tool calls execute without user awareness or consent.

Unit 42 demonstrated all three with PoC experiments inside a coding copilot integrating MCP. The copilot vendor was not named; Unit 42 noted the risk applies to any copilot enabling the sampling feature.

MCP provides no built-in authentication controls on sampling requests — the entire feature relies on implicit trust of registered servers.

- **Status:** No CVE assigned; no patch available (protocol-level design issue).
- **Source:** [unit42.paloaltonetworks.com](https://unit42.paloaltonetworks.com/model-context-protocol-attack-vectors/)

### Web-Based Indirect Prompt Injection Observed in the Wild

Unit 42 documented the first confirmed in-the-wild cases of **web-based indirect prompt injection** against coding agents. An attacker-controlled web page delivers hidden instructions that the agent reads while browsing for documentation or context. The agent executes the injected command — fetching credentials, writing files, or calling tools — without any direct user input. This confirms that agents browsing the open web during coding tasks face the same injection risk as agents reading emails or documents.

- **Status:** No patch; architectural mitigation (sandboxing, restricted browsing scope) required.
- **Source:** [Unit 42 — al-ice.ai summary](https://al-ice.ai/posts/2026/03/unit42-web-indirect-prompt-injection/)

### DeepSeek-R1: Politically Triggered Security Vulnerabilities in Generated Code

CrowdStrike Counter Adversary Operations research found that **DeepSeek-R1 generates severely vulnerable code 50% more often** when prompts include topics the CCP considers politically sensitive (e.g., Tibet, Falun Gong, Uyghurs). Baseline rate of vulnerable code for DeepSeek-R1 is ~19%; this rises to ~27.2% when the system prompt mentions Tibet-based industrial control systems. Insecure password hashing or no hashing at all appeared in 35% of implementations under these conditions.

CrowdStrike's broader finding: reasoning models produce more secure code than non-reasoning models of the same size, and newer models outperform older ones — but all are producing vulnerable code at measurable rates in production use.

- **Status:** Disclosed; no fix (model behavior issue).
- **Source:** [CrowdStrike Blog](https://www.crowdstrike.com/en-us/blog/crowdstrike-researchers-identify-hidden-vulnerabilities-ai-coded-software/)

## Supply Chain

### Context Hub Documentation Poisoning (Disclosed 2026-03-25)

**Context Hub** (released March 2026 by Andrew Ng's team at DeepLearning.AI) is an open-source MCP-based tool that supplies AI coding agents with curated, versioned API documentation. It gained 10,000 GitHub stars in one week.

Security researcher Mickey Shmueli (creator of lap.sh, an alternative service) published a PoC demonstrating that **poisoned documentation in Context Hub's registry is reliably executed by AI coding agents**:

- Two fake docs (for Plaid Link and Stripe Checkout) were created, each referencing a non-existent PyPI package name.
- In **40/40 runs**, Anthropic's Haiku model wrote the malicious package into `requirements.txt` with no warning.
- Sonnet issued warnings in 48% of runs but **still wrote the malicious package 53% of the time**.

The attack surface is the PR review process: of 97 closed PRs in the Context Hub repo, **58 were merged**, often quickly by core team members. An attacker submits a plausible doc PR with an embedded fake package name; once merged, every coding agent fetching that doc becomes a potential victim.

This is structurally identical to indirect prompt injection: agents cannot distinguish authoritative documentation from attacker-controlled instructions.

- **Status:** No CVE assigned; Context Hub under active discussion; no fix shipped.
- **Affected:** All coding agents using Context Hub MCP server.
- **Source:** [The Register, 2026-03-25](https://www.theregister.com/2026/03/25/ai_agents_supply_chain_attack_context_hub/)

### Coding Agents Expand Supply Chain Attack Surface (Security Boulevard, 2026-03-26)

A Security Boulevard analysis published today documents how **agent-authored pull requests modify dependency manifests far more frequently than human developers** — a finding from a 2026 academic study of 117,062 dependency changes across 33,596 agent-authored PRs. Agents are often faster to update dependencies than humans to review them, creating a window for supply chain injection that does not exist in human-only workflows.

Key risks catalogued:

- **Slopsquatting at scale**: ~20% of code samples from 16 LLMs reference non-existent packages; 43% of hallucinated names recur consistently across re-runs, enabling pre-registration attacks.
- **Automated Dependency Side-Loading**: AI IDE extensions can silently introduce new packages via tool calls during refactoring — bypassing developer review at the moment of installation.
- **Agent-to-agent poisoning**: A downstream agent ingesting upstream agent output (e.g., a triage agent feeding a coding agent) inherits any prompt injection present in that output.

- **Source:** [Security Boulevard](https://securityboulevard.com/2026/03/coding-agents-widen-your-supply-chain-attack-surface/), [Medium — Automated Dependency Side-Loading](https://medium.com/@instatunnel/automated-dependency-side-loading-the-invisible-supply-chain-attack-via-ai-extensions-fe615eb03f18)

### RSAC 2026 Preview: Developers as the New Perimeter

Multiple sessions at RSAC 2026 (San Francisco) focus on developer-environment attacks. Check Point's Oded Vanunu is presenting *"When AI Agents Become Backdoors"*, covering the now-confirmed pattern: prompt injection in untrusted repositories → agent tools → IDE features → RCE or credential exfiltration, without requiring any network exploit.

The consensus framing from researchers at Check Point, Terra Security, OX Security, and Unit 42: the developer's local machine — historically trusted — is now the primary enterprise attack surface for AI-era threats.

Terra Security's adversarial testing found **100% of applications embedding AI chats or copilots** had at least one AI-related security vulnerability, and that AI-generated code consistently introduces business logic and authorization flaws invisible to standard SAST/DAST tooling.

- **Source:** [BusinessWire — Terra Security](https://www.businesswire.com/news/home/20260224886429/en/Terra-Security-Finds-Widespread-Exploitable-Flaws-in-AI-Driven-Applications-Copilots-and-AI-Generated-Code), [SecurityWeek](https://www.securityweek.com/vulnerability-in-openai-coding-agent-could-facilitate-attacks-on-developers/)

The [vulnerablemcp.info](https://vulnerablemcp.info) database continues to track CVEs across MCP servers, clients, and tooling. Current totals as of late March 2026:
