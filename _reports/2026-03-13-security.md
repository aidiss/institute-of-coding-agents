## Incidents & Vulnerabilities

### IDEsaster: 30+ Flaws, 24 CVEs Across Major AI Coding Tools

A six-month research effort disclosed **over 30 vulnerabilities** across 10+ products including GitHub Copilot, Cursor, Windsurf, Kiro.dev, Zed.dev, Roo Code, JetBrains Junie, Cline, Gemini CLI, and Claude Code, resulting in **24 assigned CVEs** and an AWS advisory (AWS-2025-019). Every tested AI IDE was vulnerable.

The attack chain follows three stages: **Prompt Injection → Tools → Base IDE Features**. Attackers hijack context via malicious rule files, MCP servers, deeplinks, or file names, then leverage the AI agent to modify IDE configuration files (`.vscode/settings.json`, `.idea/workspace.xml`, `.cursor/mcp.json`) to achieve RCE.

Sources: [The Hacker News](https://thehackernews.com/2025/12/researchers-uncover-30-flaws-in-ai.html) · [GBHackers](https://gbhackers.com/ai-developer-tools/) · [CyberSecurityNews](https://cybersecuritynews.com/critical-vulnerabilities-in-github-copilot-gemini-cli-claude/)

## Supply Chain

### Slopsquatting — Active Exploitation (March 2026)

"Slopsquatting" (term coined by Python PSF's Seth Larson) is a supply chain attack exploiting LLM hallucinations: attackers register package names that AI coding tools repeatedly invent, then serve malicious payloads when developers install the AI-recommended package.

**Scale (academic study, 576,000–756,000 code samples, 16 LLMs):**
- ~20% of AI-recommended packages do not exist in any registry
- Commercial models hallucinate at ~5.2%; open-source models at ~21.7%
- 200,000+ unique hallucinated package names documented
- 43% of hallucinated names are repeatable across prompts — making them predictable attack targets

**Active in the wild (late Feb / early March 2026):** Malicious Rust crates impersonating `timeapi.io` discovered POSTing `.env` file contents (credentials, secrets) to an attacker-controlled lookalike domain.

**Earlier precedent:** "huggingface-cli" PyPI package registered as a proof-of-concept after AI models repeatedly hallucinated it; adopted by thousands of developers including Alibaba teams within days.

Sources: [BleepingComputer](https://www.bleepingcomputer.com/news/security/ai-hallucinated-code-dependencies-become-new-supply-chain-risk/) · [Aikido Security](https://www.aikido.dev/blog/slopsquatting-ai-package-hallucination-attacks) · [Socket.dev](https://socket.dev/blog/slopsquatting-how-ai-hallucinations-are-fueling-a-new-class-of-supply-chain-attacks) · [Trend Micro](https://www.trendmicro.com/vinfo/us/security/news/cybercrime-and-digital-threats/slopsquatting-when-ai-agents-hallucinate-malicious-packages)

### Rules File Backdoor (GitHub Copilot & Cursor)

Disclosed by Pillar Security. Attackers embed hidden Unicode characters in Cursor `.cursorrules` / Copilot instruction files to silently steer AI output toward inserting malicious code. The injected code bypasses typical code review because the malicious instruction is invisible to human eyes but parsed by the LLM. GitHub has since added a warning for files containing hidden Unicode on github.com.

### Cline / OpenClaw npm Incident (February 2026)

Prompt injection in a Claude-powered GitHub Actions issue-triage workflow led to a compromised npm package that silently installed a persistent daemon on approximately 4,000 developer machines, exfiltrating credentials and SSH keys.

### PackageGate — npm/pnpm/vlt/Bun Zero-Days (January 2026)

Security researchers at Koi disclosed **six zero-day vulnerabilities** across npm, pnpm, vlt, and Bun — collectively dubbed "PackageGate" — that undermine the previously recommended defenses of disabling lifecycle scripts and relying on lockfiles. Software supply chain attacks doubled in 2025, with estimated global losses of $60 billion.

**Attacker weaponization of MCP:** Researchers found infrastructure for AI-assisted intrusion workflows linking compromised systems to LLMs via MCP, enabling automated reconnaissance, vulnerability assessment, and attack planning.

### Defensive Recommendations

| Layer | Recommendation |
|-------|---------------|
| Agent config | Review `.claude/settings.json`, `.cursor/mcp.json`, `.vscode/settings.json` for unexpected hooks or MCP entries |
| Dependencies | Verify all AI-recommended packages exist before install; generate and sign SBOMs |
| MCP servers | Only connect to authenticated, encrypted servers; treat tool definitions as potentially mutable |
| IDE settings | Enable human-in-the-loop approval for file writes; do not auto-approve tool calls |
| CI/CD | Run automated dep-scan / Safety CLI; isolate AI-generated installs in ephemeral containers |
| Model config | Lower temperature for dependency-resolution prompts to reduce hallucination rate |
| Architecture | Sandbox agent operations; apply least-privilege to all LLM tool calls; assume prompt injection is always possible |
