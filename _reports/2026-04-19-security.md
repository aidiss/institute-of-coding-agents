# Security Brief — 2026-04-19

## Incidents & Vulnerabilities

### Comment and Control: Prompt Injection → Credential Theft (Claude Code, GitHub Copilot, Gemini CLI)

Researchers disclosed a class of prompt injection attacks dubbed "Comment and Control" targeting three major AI coding agents. Attackers embed malicious instructions in GitHub PR titles, issue bodies, or hidden HTML comments; agents process these without sanitization and exfiltrate `ANTHROPIC_API_KEY`, `GITHUB_TOKEN`, and other secrets.

- **Affected:** Claude Code (Anthropic), GitHub Copilot (Microsoft), Gemini CLI (Google)
- **Status:** Disclosed. No CVE assigned. No vendor patch or advisory published as of 2026-04-19.
- **Sources:** [SecurityWeek](https://www.securityweek.com/claude-code-gemini-cli-github-copilot-agents-vulnerable-to-prompt-injection-via-comments/); [oddguan.com](https://oddguan.com/blog/comment-and-control-prompt-injection-credential-theft-claude-code-gemini-cli-github-copilot/)

### CVE-2025-53773 — GitHub Copilot RCE via Prompt Injection (CVSS 7.8)

Attackers embed malicious prompts in source files, web pages, or GitHub issues using invisible Unicode characters. The payload modifies `.vscode/settings.json` to enable `chat.tools.autoApprove: true` ("YOLO mode"), granting unrestricted shell command execution on Windows, macOS, and Linux.

- **Affected:** GitHub Copilot in Visual Studio 2022 (before version 17.14.12)
- **Disclosed:** 2025-06-29 (responsible disclosure); 2025-08-12 (public)
- **Status:** Patched — Visual Studio 2022 v17.14.12 (August 2025 Patch Tuesday)
- **Sources:** [Wiz](https://www.wiz.io/vulnerability-database/cve/cve-2025-53773); [embracethered.com](https://embracethered.com/blog/posts/2025/github-copilot-remote-code-execution-via-prompt-injection/)

### CVE-2026-25253 — OpenClaw Agent Framework RCE via URL Parameter Bypass

The OpenClaw open-source agent framework (135,000+ GitHub stars) exposed a one-click RCE in its Control UI through unvalidated URL parameters. Cross-site WebSocket hijacking allowed full instance takeover. Additionally, 1,184 confirmed malicious skills were found in the ClawHub marketplace.

- **Affected:** OpenClaw prior to v2026.1.29
- **Status:** Patched in v2026.1.29
- **Sources:** [OWASP GenAI Q1 2026](https://genai.owasp.org/2026/04/14/owasp-genai-exploit-round-up-report-q1-2026/); [reco.ai](https://www.reco.ai/blog/openclaw-the-ai-agent-security-crisis-unfolding-right-now)

### AI IDE Survey: 100% Vulnerable to Prompt Injection RCE

A comprehensive survey of AI-enhanced IDEs (Cursor, Windsurf, GitHub Copilot, Zed, Roo Code, Junie) assigned 24 CVEs and found all tested tools vulnerable to prompt injection enabling RCE and data exfiltration. Additional findings:

- 87% of AI-generated code introduced vulnerabilities in reviewed pull requests
- 45% of AI-generated code introduces known security flaws
- 41% of AI-generated backend code includes overly broad permission settings
- Broken access control appeared in 100% of tested applications

- **Status:** Ongoing research; individual CVEs assigned per tool
- **Source:** [cloudsecuritynewsletter.com — IDEsaster](https://www.cloudsecuritynewsletter.com/p/idesaster-ai-ide-vulnerabilities-attack-surface)

## Supply Chain

### Hallucinated Dependencies ("Slopsquatting") — Active Exploitation on npm & PyPI

LLMs hallucinate non-existent package names in approximately 20% of code generation samples; 58% of hallucinated names are repeatable across runs, making them predictable targets. Attackers pre-register these names and serve payloads that steal API keys, environment variables, and SSH credentials. Recent campaigns:

- **"claw" naming pattern campaign (Feb–Apr 2026):** 1,000+ malicious packages deployed; targeted Fortune 500 cloud environments.
- **SANDWORM_MODE (Feb 2026):** 19 typosquatting npm packages with credential theft, project infection, and self-propagation logic.
- **LiteLLM / Mercor incident (Apr 2026):** TeamPCP hacking group inserted credential-harvesting code into LiteLLM (millions of daily downloads); Mercor ($10B AI startup) was compromised as a downstream consumer.

- **Status:** PyPI deployed auto-detection for typosquatting; npm protections remain incomplete. Active exploitation ongoing.
- **Sources:** [BleepingComputer](https://www.bleepingcomputer.com/news/security/ai-hallucinated-code-dependencies-become-new-supply-chain-risk/); [Tencent xLab](https://xlab.tencent.com/en/2026/02/28/ghost-dependency-agentic-coding-supply-chain-threat/); [Fortune](https://fortune.com/2026/04/02/mercor-ai-startup-security-incident-10-billion/); [HelpNetSecurity](https://www.helpnetsecurity.com/2026/02/24/npm-worm-sandworm-mode-supply-cain-attack/)

## Research & Advisories

### MCP STDIO Design Flaw — ~200,000 Servers Exposed

A systemic flaw in the Model Context Protocol's STDIO execution model allows arbitrary command execution regardless of process start success. The architectural issue affects official Anthropic MCP SDKs (Python, TypeScript, Java, Rust). An April 2026 paper documents exploitation paths including malicious code execution, remote access control, and credential theft.

Trend Micro identified 492 MCP servers exposed to the internet with zero authentication. On 2026-04-03, Microsoft's `@azure-devops/mcp` npm package was found to lack an authentication layer, exposing Azure DevOps work items, repositories, pipelines, and API keys.

- **Affected:** ~200,000 MCP servers; all applications using official Anthropic MCP SDKs
- **Anthropic's position:** Behavior is "by design"; developer responsibility to sanitize inputs
- **Status:** Disclosed, unfixed at protocol level. No CVE assigned.
- **Research:** [arxiv.org/abs/2504.03767 — MCP Safety Audit](https://arxiv.org/abs/2504.03767)
- **Sources:** [The Register](https://www.theregister.com/2026/04/16/anthropic_mcp_design_flaw/); [Infosecurity Magazine](https://www.infosecurity-magazine.com/news/systemic-flaw-mcp-expose-150/)
