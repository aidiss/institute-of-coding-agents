# Security Brief — AI Coding Agents
**2026-04-05**

Track vulnerabilities, supply chain attacks, and safety incidents affecting AI coding agents.

---

## Incidents & Vulnerabilities

### MCPJam Inspector — Unauthenticated RCE
**CVE-2026-23744** | CVSS 9.8 (Critical) | **Status:** Patched (versions > 1.4.2)

MCPJam Inspector (≤ v1.4.2) bound its server endpoint to `0.0.0.0` with no authentication. A crafted HTTP request was sufficient to install an arbitrary MCP server and execute code on the host with full user privileges. The vulnerability follows the now-established MCP exploitation pattern: unauthenticated local/network access → implicit tool trust → arbitrary execution. Researchers demonstrated escalation to cloud credential theft (AWS, GCP, Azure tokens). Patch: upgrade to v1.4.3 or later.

---

### Azure MCP Server — Cloud Credential Compromise via RCE
**No CVE filed at time of writing** | **Status:** Research to be presented at RSAC 2026

Researchers identified an RCE vulnerability in Microsoft's Azure MCP server capable of compromising entire cloud environments. Exploitation chains prompt injection through AI context into tool invocation, ultimately reaching Azure credentials. The attack leverages the MCP protocol's implicit tool trust model — agents cannot distinguish legitimate server instructions from attacker-supplied ones. Expected public technical disclosure at RSAC 2026.

---

### MCP TypeScript SDK — Cross-Client Data Leak
**CVE-2026-25536** | CVSS 7.1 (High) | **Status:** Patched; affected versions 1.10.0–1.25.3

When a single `McpServer` instance is reused across multiple client connections — a common deployment pattern — responses can leak across client boundaries, exposing context from one user's session to another. This affects multi-tenant MCP deployments (shared inference infrastructure, team MCP proxies). Fix: upgrade to SDK version 1.25.4+; audit server instantiation patterns.

---

### CrewAI — Four Chained CVEs (Prompt Injection → RCE / SSRF / File Read)
**Multiple CVEs** | **Status:** Fix availability unclear at time of writing

Four CVEs in the CrewAI agentic framework allow an attacker to chain prompt injection into remote code execution, SSRF, and arbitrary file read. The Code Interpreter tool and default agent configurations are the primary attack surface. The chain is exploitable with a single malicious prompt reaching an agent with code execution capabilities. Severity is elevated because CrewAI is widely used in production coding agent pipelines.

### GitHub Copilot — Additional CVEs (Late 2025, Ongoing Impact)
**Status:** Patched; downstream exposure depends on version pinning

| CVE | Description | CVSS |
|---|---|---|
| CVE-2025-62449 | Path traversal in Copilot file operations | 6.8 |
| CVE-2025-62453 | Improper validation of AI-generated output before use | 5.0 |
| CVE-2025-59286 | Spoofing of Copilot identity in multi-agent workflows | Medium |
| CamoLeak (unfiled) | Silent code exfiltration via invisible Unicode in GitHub Issues | 9.6 |

**CamoLeak** is the most severe: invisible Unicode characters (zero-width joiners, bidirectional markers) embedded in GitHub Issues instruct Copilot to silently exfiltrate code to attacker-controlled infrastructure. No visible diff, no user alert. Overlaps with the Rules File Backdoor attack class documented in earlier reports. GitHub's current mitigation — a banner warning on github.com for files containing hidden Unicode — does not cover Issues.

### Axios npm Package — Maintainer Account Compromise
**Disclosed:** 2026-03-31 | **Status:** Malicious versions pulled; clean release available

On March 31, 2026, attackers compromised an Axios maintainer's npm account and injected malicious code into official Axios releases. The compromise propagated rapidly through automated `npm install` pipelines before detection. Axios is one of the most-downloaded JavaScript packages; exposure was broad. AI coding agents that autonomously run `npm install` and accept AI-suggested dependencies are particularly affected — no human review step exists to catch malicious updates mid-pipeline.

## Supply Chain

### LiteLLM PyPI Supply Chain Attack — Active Exploit by TeamPCP
**Incident date:** 2026-03-24 | **Packages:** litellm==1.82.7, litellm==1.82.8 | **Status:** Packages quarantined; clean versions available

A supply chain attack against LiteLLM — a popular AI infrastructure gateway with ~3.4 million daily PyPI downloads — was uncovered on March 26, 2026. Two malicious versions were live for approximately 40 minutes (10:39 UTC, March 24) before PyPI quarantined them.

**Payload mechanics:**
- `v1.82.7`: malicious code injected into `proxy_server.py`
- `v1.82.8`: malicious code in `proxy_server.py` **plus** a `litellm_init.pth` file that auto-executes on **every Python process startup** when the package is installed — including in CI/CD environments

**Objectives:** Harvest high-value secrets: AWS, GCP, and Azure access tokens; SSH keys; Kubernetes credentials. Designed to enable lateral movement and long-term persistence across compromised CI/CD systems and production infrastructure.

**Attribution:** Threat group **TeamPCP**, also linked to attacks on Aqua Security's Trivy scanner, KICS, and Telnyx. Compromised PyPI credentials are believed to have originated from the Trivy breach. The LiteLLM attack was part of a coordinated March 2026 supply chain campaign.

**Remediation:** Audit Python environments for `litellm==1.82.7` or `1.82.8`. Check for `litellm_init.pth` in site-packages. Rotate all secrets accessible from affected environments.

### Slopsquatting — Continued Threat; Attack Volume Increasing
**Ongoing** | **Coined by:** Seth Larson (Python PSF)

Slopsquatting — registering package names that AI models consistently hallucinate — remains an active and escalating threat as AI coding agent adoption grows. Key data points:

- ~20% of AI-generated Python/JavaScript code (576,000 samples studied, March 2025) referenced non-existent packages.
- 43% of hallucinated package names reappear on every re-run of the same prompt; 58% reappear more than once — providing reliable, predictable attack targets.
- Commercial models (GPT-4-class): ~5% hallucination rate. Open-source models (CodeLlama, DeepSeek, Mistral): significantly higher.
- The `huggingface-cli` incident: a researcher registered the consistently-hallucinated PyPI name; over 30,000 downloads followed within three months — including from Alibaba engineering teams.

Coding agents (Cursor, Claude Code, GitHub Copilot Workspace) amplify this risk because they can autonomously execute `pip install` and `npm install` without human review of each dependency selection.

### Coding Agents as Supply Chain Multipliers
**Analysis:** Security Boulevard, 2026-03

AI coding agents participate directly in the software supply chain — generating dependency selections, executing package installs, writing CI/CD configuration, and pushing commits — at a speed that outpaces human review capacity. Key risks identified:

- Agents accept AI-suggested package versions where 44–49% of imported dependency versions carry known CVEs (Endor Labs data).
- Agents can be manipulated via prompt injection to modify `package.json`, `requirements.txt`, or lockfiles to pin malicious versions.
- Automated pipelines with agent-generated code have no natural checkpoint to catch supply chain tampering before deployment.

## Research & Advisories

### Prompt Injection Surge — 340% Increase in 2026
**Source:** OWASP LLM Security Project, March 2026

OWASP classified prompt injection as the single highest-severity vulnerability category for deployed LLMs — above data poisoning, model theft, and insecure output handling. Key findings:
- Prompt injection attacks increased **340%** in 2026.
- **Indirect injection** (attacks embedded in documents, emails, web pages, database content) now accounts for **>80%** of documented enterprise attacks; direct user-typed attacks represent <20%.
- Attack success rates against state-of-the-art defenses **exceed 85%** when adaptive strategies are used (arXiv:2601.17548, meta-analysis of 78 studies).

The most effective available defense remains sandboxing: isolating agent operations from the host system.

---

### Palo Alto Networks Unit 42 — MCP Sampling Attack Vectors
**Published:** 2026 Q1 | **Source:** Unit 42 / Palo Alto Networks

Unit 42 research documents how the MCP `sampling` feature — which allows servers to request that a client LLM generate text — is exploitable by malicious MCP servers to conduct prompt injection, context poisoning, and lateral tool abuse. Notable findings:
- **Context poisoning:** Malicious data injected into agent context propagates through multi-agent workflows; downstream agents make decisions on corrupted inputs without indication of tampering.
- The MCP protocol specification mandates session identifiers in URLs, exposing them in logs and enabling session hijacking — a design-level flaw.
- Malicious MCP servers can exploit the sampling handshake to extract system prompts, tool definitions, and conversation history from client agents.

---

### The Register — AI Code Not More Secure Than Human Code
**Published:** 2026-03-26

Analysis of production codebases found no statistically significant reduction in vulnerability density in AI-assisted versus manually written code — and in several categories (CSRF, SSRF, injection), AI-generated code performed worse. The report contextualizes the March 2026 CVE surge: 35 AI-attributed CVEs in a single month is not an artifact of researcher attention but reflects a genuine quality signal.

---

### Adversa AI — April 2026 Agentic Security Resource Roundup
**Published:** 2026-04

Summary of notable agentic AI security research themes entering April 2026:
- **Memory poisoning** as a persistent threat: agent memory stores are not sanitized between sessions, enabling long-term behavioral manipulation.
- **Multi-agent trust chains:** Agent-to-agent delegation without re-validation of identity or scope is now the dominant unexplored attack surface.
- **OWASP Top 10 for Agentic Applications** (released December 2025, 100+ researchers) continues to be the primary reference framework; ASI01 (Goal Hijack) and ASI02 (Tool Misuse) are most relevant to coding agent deployments.
