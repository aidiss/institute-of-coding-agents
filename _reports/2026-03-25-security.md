## Incidents & Vulnerabilities

### IDEsaster: 30+ Vulnerabilities Across AI Coding IDEs (December 2025)

Researcher Ari Marzouk (MaccariTA) disclosed **IDEsaster** on December 6, 2025 — a novel vulnerability class affecting 10+ AI coding tools including Cursor, Windsurf, GitHub Copilot, Zed, Roo Code, JetBrains Junie, Cline, and Claude Code. **24 CVEs were assigned**; AWS issued advisory AWS-2025-019. 100% of tested AI IDEs were found vulnerable.

The core attack chain is: **Prompt Injection → Agent Tools → Base IDE Features**. Attackers inject into agent context (malicious rule files, MCP servers, filenames, deeplinks), then have the agent invoke legitimate IDE capabilities to achieve RCE or data exfiltration. Legacy IDE features were not designed for autonomous agent invocation.

Key CVEs from IDEsaster:

| CVE | Tool | Description | Status |
|---|---|---|---|
| CVE-2025-64660 | GitHub Copilot | Prompt injection via `*.code-workspace` → RCE | Patched |
| CVE-2025-61590 | Cursor | Workspace config injection → RCE | Patched |
| CVE-2025-58372 | Roo Code | Workspace config injection → RCE (requires auto-approve) | Patched |
| CVE-2025-49150 | Cursor | Write JSON with remote schema → data exfiltration via GET | Patched |

A separate OX Security report found Cursor and Windsurf run on outdated Chromium/Electron versions, exposing 1.8 million developers to 94+ known browser-engine CVEs. Researchers successfully weaponized **CVE-2025-7656** (a patched Chromium flaw) against the latest Cursor and Windsurf releases.

### Claude Code: RCE and API Key Exfiltration

Check Point Research (Aviv Donenfeld, Oded Vanunu) disclosed two critical vulnerabilities in Claude Code, both deliverable via a malicious repository a developer clones.

**CVE-2025-59536** (CVSS 8.7, disclosed October 2025): RCE via the Hooks feature. A `.claude/settings.json` in a repository can define hooks that execute shell commands at lifecycle events. A malicious hook runs arbitrary commands the moment a developer opens the project — before any trust dialog appears.
- Affected: Claude Code < 1.0.111; partially remediated; fully patched in 2.0.65.

**CVE-2026-21852** (CVSS 5.3, disclosed January 2026): API key exfiltration. `ANTHROPIC_BASE_URL` in a project's configuration redirects all Claude Code API calls through an attacker-controlled proxy; the Authorization header exposes the full API key in plaintext. With Claude Workspaces, a single stolen key can expose an entire team's workspace.
- Affected: Claude Code < 2.0.65. Patched by blocking API requests before trust confirmation.

**CVE-2026-25725**: Sandbox bypass enabling unauthorized filesystem access.
- Patched in Claude Code 2.1.2.

### GitHub Copilot Vulnerabilities

**CVE-2025-53773** (CVSS 9.6): RCE via prompt injection. An attacker embeds an injection in public repository code comments; when a victim opens the repo with Copilot active, the injection modifies `.vscode/settings.json` to enable auto-execute mode, after which subsequent commands run without user interaction. Trail of Bits demonstrated this technique can insert backdoors that pass human review.
- Status: Patched by Microsoft (2025).

**CVE-2025-32711 "EchoLeak"** (CVSS 9.3): Zero-click data exfiltration in Microsoft 365 Copilot. A hidden prompt injection in an email or document causes Copilot to return the victim's emails, OneDrive, SharePoint, or Teams content to the attacker — no user click required.
- Status: Patched server-side by Microsoft.

## Supply Chain

### Slopsquatting: Hallucinated Dependencies as an Attack Vector

University of Texas San Antonio, University of Oklahoma, and Virginia Tech published research (March 2025) on 576,000 Python/JavaScript code samples generated by 16 LLMs:

- ~20% of generated code references non-existent packages.
- Hallucination rates: 0.22% (best commercial models) to 46.15% (open-source models).
- 43% of hallucinated package names are **repeated consistently** across 10 re-runs of the same prompt — this repeatability is what makes them exploitable.
- 8.7% of hallucinated Python package names exist in npm, suggesting cross-ecosystem confusion.

The attack, termed **slopsquatting**, works by registering hallucinated package names on PyPI or npm with malicious code. Developers installing AI-suggested packages receive the malware without any error.

A proof-of-concept: a researcher registered the AI-hallucinated name `huggingface-cli` on PyPI as a benign test; within days, thousands of developers including Alibaba teams had pulled it into production. In January 2025, Google's AI Overview suggested the npm package `@async-mutex/mutex` — a typosquatted version of the legitimate `async-mutex`.

RAG and fine-tuning can reduce hallucination rates by up to 85% but with measurable impact on code quality.

### MCP Registry Supply Chain: Smithery Path Traversal

A `dockerBuildPath: ".."` entry in `smithery.yaml` caused the Smithery hosted MCP registry to build Docker images from the builder's home directory, leaking `~/.docker/config.json` including a Fly.io API token with control over 3,000+ hosted MCP servers.

### First Confirmed Malicious MCP Server on npm

The package `postmark-mcp` on npm was discovered silently BCC'ing all outgoing emails from AI agents to the attacker — the first confirmed malicious MCP server found in a public registry.

## Research & Advisories

### MCP Ecosystem Vulnerabilities (30+ CVEs)

MCP has accumulated 30+ documented CVEs across its ecosystem since adoption accelerated in 2025. Catalogued at [vulnerablemcp.info](https://vulnerablemcp.info).

**Anthropic mcp-server-git three-CVE chain:**
- CVE-2025-68145: Path validation bypass
- CVE-2025-68143: Unrestricted `git_init` targeting `.ssh` directory
- CVE-2025-68144: Argument injection in `git_diff`

Combined with the Filesystem MCP server, these chain to full RCE via a malicious `.git/config` file. Patched.

**CVE-2026-23744 — MCPJam Inspector** (CVSS 9.8): Zero-interaction RCE against developer tooling. Unauthenticated HTTP listener allows an attacker to run arbitrary commands on a developer's machine simply by having the victim inspect a malicious MCP server, or by driving the inspector from a browser.

**CVE-2026-25253 — OpenClaw "ClawJacked"** (CVSS 8.8): Full agent takeover via a malicious webpage. 135,000+ public instances were exposed at time of disclosure.

**gemini-mcp-tool** (CVSS 9.8, unpatched zero-day): Passes unsanitized user input directly to `execAsync` shell commands with no authentication.

**Microsoft MarkItDown MCP server**: Unpatched SSRF — fetches arbitrary URLs without validation, enabling access to AWS EC2 instance metadata and cloud credentials. Microsoft classified as low severity.

**Scale assessment**: Analysis of 67,057 MCP servers across 6 public registries found:
- ~7,000 public MCP servers
- ~50% lack authentication or encryption
- 36.7% vulnerable to SSRF
- 33% allow unrestricted outbound network access
- 22% access files outside intended boundaries

### Additional Attack Techniques

**CamoLeak**: Invisible Markdown comments deliver prompt injections; pre-generated Camo URLs bypass GitHub's Content Security Policy; secrets exfiltrated character-by-character via image request sequences from private repositories.

**GeminiJack**: Zero-click exploit against Google Gemini Enterprise — sharing a Google Doc or Calendar invite triggers data exfiltration with no further user action.

**Supabase/Cursor incident (mid-2025)**: Cursor's Supabase agent, running with privileged service-role credentials, processed support tickets containing user-supplied SQL injected into commands. Attackers exfiltrated integration tokens into a public support thread.

**AgentFlayer (Cursor)**: Demonstrated a five-stage attack including pipeline-based lateral movement — the most complex coding agent attack chain documented to date.

**Persistent memory poisoning**: Malicious instructions injected into an agent's memory store persist across sessions — sometimes days or weeks — establishing a durable foothold in the developer's workflow.

**WhatsApp history exfiltration via MCP tool poisoning**: Invariant Labs demonstrated a malicious MCP server instructing a legitimate `whatsapp-mcp` server to silently exfiltrate a user's full message history.
