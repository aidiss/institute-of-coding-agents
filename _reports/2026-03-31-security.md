# Security Brief — AI Coding Agents
**2026-03-31**

Track vulnerabilities, supply chain attacks, and safety incidents affecting AI coding agents.

---

## Incidents & Vulnerabilities

### IDEsaster — 30+ Vulnerabilities Across 10+ AI IDEs
**Disclosed:** Late 2025 | **CVEs:** 24 assigned | **Status:** Mostly patched; some extensions remain unpatched

Researcher Ari Marzouk (MaccariTA) disclosed a coordinated six-month research campaign finding over 30 vulnerabilities across Cursor, Windsurf, Kiro.dev, GitHub Copilot, Zed.dev, Roo Code, JetBrains Junie, Cline, and Claude Code. AWS issued advisory **AWS-2025-019** covering the cluster.

The core attack chain: **Prompt Injection → Tool Abuse → Base IDE Feature Exploitation**, manifesting in three patterns:

- **JSON Schema Data Exfiltration:** Prompt injection triggers an IDE request to an attacker-controlled remote JSON schema, leaking environment data. Affects Cursor (CVE-2025-49150), Roo Code (CVE-2025-53097), JetBrains Junie (CVE-2025-58335), GitHub Copilot, Kiro.dev, Claude Code (addressed with a warning).
- **IDE Settings RCE:** Prompt injection rewrites `.vscode/settings.json` or `.idea/workspace.xml` to point `php.validate.executablePath` or `PATH_TO_GIT` at a malicious binary. Affects GitHub Copilot (CVE-2025-53773), Cursor (CVE-2025-54130), Roo Code (CVE-2025-53536), Zed.dev (CVE-2025-55012).
- **Workspace Config RCE (zero-click):** Malicious workspace config achieves RCE with no user interaction because in-workspace files are auto-approved. Affects GitHub Copilot (CVE-2025-64660), Cursor (CVE-2025-61590), Roo Code (CVE-2025-58372).

### GitHub Copilot — RCE via Prompt Injection
**CVE-2025-53773** | CVSS 7.8 (HIGH) | **Patched:** August 2025 (VS Code 17.14.12) | 7 public PoCs exist

Malicious instructions hidden via Unicode zero-width characters in source files, issues, or web content direct Copilot to add `"chat.tools.autoApprove": true` to `.vscode/settings.json`, enabling unconfirmed shell execution ("YOLO mode"). Demonstrated as **wormable** — the payload can spread across repositories. Attack is model-agnostic (GPT-4.1, Claude Sonnet 4, Gemini all affected).

Additional: **CVE-2025-64660** — zero-click workspace config RCE, no user interaction required. **TRA-2025-53** (Tenable) — filename-based prompt injection in Copilot Chat v0.28.0 using Claude Sonnet 4.

### Anthropic mcp-server-git — Three Chained CVEs
**CVE-2025-68145, CVE-2025-68143, CVE-2025-68144** | **Status:** Fix status unclear at time of writing

Three chained vulnerabilities in Anthropic's own `mcp-server-git` reference implementation enable full RCE via malicious `.git/config` files:
- **CVE-2025-68145** — path validation bypass
- **CVE-2025-68143** — unrestricted `git_init` can convert `~/.ssh` into a Git repo
- **CVE-2025-68144** — argument injection in `git_diff`

Separately, **Anthropic's MCP Inspector** development tool allowed unauthenticated RCE: an attacker could execute arbitrary commands on a developer machine by having the victim inspect a malicious MCP server. The inspector bound to localhost/0.0.0.0 without authentication and ran with full user privileges.

### Amazon Q Developer — Supply Chain Compromise
**CVE-2025-8217** (Critical) | **Status:** v1.84.0 silently pulled; no public advisory

On July 23–24, 2025, Amazon Q VS Code extension version 1.84.0 was found to contain malicious code injected via a compromised GitHub repository. Root cause: an over-scoped GitHub token in AWS CodeBuild configurations allowed unauthorized commits to bypass review. Amazon pulled the version without a public statement.

Critical account takeover + RCE. Threat actors deployed the **Flodric botnet** through compromised Langflow instances. All stored API keys and access tokens were exposed, causing cascading cloud and SaaS compromises downstream.

### OpenClaw AI Agent — WebSocket Hijack
**CVE-2026-25253** (Critical) | **Status:** Patched

WebSocket Origin-header validation failure allows a malicious webpage to connect to the agent's local server and send arbitrary commands — full RCE without authentication. Notable as the first AI agent CVE to trigger a public warning from a national regulatory body (China's MIIT).

## Supply Chain

### Slopsquatting — Hallucinated Package Names as Attack Surface
**Coined by Python PSF (Seth Larson), early 2025**

Attackers register package names that AI models consistently hallucinate in code suggestions. Key data from a March 2025 study of 576,000 Python/JavaScript samples:

- ~20% of AI-generated code recommended non-existent packages.
- Commercial models (GPT-4): ~5% hallucination rate. Open-source (CodeLlama, DeepSeek, Mistral): significantly higher.
- 43% of hallucinated names appeared **every time** the same prompt was re-run; 58% appeared more than once.
- Over 200,000 unique hallucinated package names identified in a single study.
- Endor Labs: 34% of AI-suggested dependencies did not exist in public registries; 44–49% of AI-imported dependency versions had known CVEs.
- 2025 npm saw 3,180 confirmed malicious packages, with AI-assisted automated publishing campaigns driving much of the volume.

**huggingface-cli incident:** A researcher registered the consistently hallucinated package name on PyPI; thousands of developers including Alibaba teams adopted it before detection. A documented dark-web actor used ChatGPT to auto-generate typosquatted name variants at scale for botnet building.

### Rules File Backdoor — Persistent AI Config Poisoning
**Disclosed:** Pillar Security, March 2025

Attackers embed malicious prompts in AI configuration files (`.cursorrules`, `.github/copilot-instructions.md`) using hidden Unicode characters (zero-width joiners, bidirectional markers). Effect:
- Persistently affects all future code generation for every team member after cloning.
- Survives project forking — spreads downstream.
- Can instruct the AI to introduce backdoors, weaken cryptography, or insert credential-stealing code with no visible evidence in a diff.

GitHub's response (May 2025): Added a warning banner for files containing hidden Unicode on github.com. Both GitHub and Cursor framed this as developer responsibility rather than a platform-level fix.

### ZombAI — AI Agents Turned Into C2 Nodes
**Researcher:** Johann Rehberger (Embrace The Red), 2025

Demonstrated across multiple platforms via prompt injection through untrusted content (GitHub issues, web pages, Slack messages):

- **OpenHands:** Downloads and executes malware, connects to attacker C2.
- **Google Jules:** Leaks source code and environment variables; joins attacker botnet. Disclosed June 1, 2025 — Google closed ticket as "already tracking" with no fix timeline.
- **ChatGPT Codex:** Internet-access allowlist abused via prompt injection in GitHub issues for arbitrary RCE. OpenAI closed the report "Not Applicable" on submission day.
- **GitHub Copilot (CVE-2025-53773):** Same wormable ZombAI pattern — persistent attacker control via compromised project repositories.

## Research & Advisories

### OWASP Top 10 for Agentic Applications (2026 Edition)
**Released:** December 9, 2025 — developed by 100+ security researchers. Referenced by Microsoft, NVIDIA, AWS, GoDaddy.

Most relevant risks for coding agent environments:
- **ASI01 — Agent Goal Hijack:** Redirect agent objectives via prompt injection (e.g., EchoLeak: hidden prompts in Copilot turned it into an exfiltration engine).
- **ASI02 — Tool Misuse:** Agents invoke legitimate tools (shell, file write) in unintended destructive sequences; poisoned MCP tool descriptors.
- **ASI03 — Identity & Privilege Abuse:** Leaked credentials grant agents broader scope than intended.
- **ASI04 — Delegated Trust Abuse:** Downstream agent composition without re-validating trust chains.
- **ASI10 — Rogue Agents:** Agents that persist harmful behavior across sessions after a single injection.

Introduces the principle of **"Least Agency"** — autonomy should be explicitly scoped, not granted by default.

### LSPFuzz — Language Server Protocol Attack Surface
**ASE 2025 (IEEE/ACM)**

Dedicated grey-box hybrid fuzzer for LSP servers found bugs to be pervasive (clangd alone has 500+ crash issues across its ~300 tracked LSP implementations). No high-profile LSP exploits are yet publicly disclosed, but LSP is now integrated directly into agentic coding tools (Claude Code LSP integration shipped December 2025, v2.0.74), making LSP crash/corruption a growing attack vector.

### Prompt Injection Meta-Analysis
**arXiv:2601.17548 — "Prompt Injection Attacks on Agentic Coding Assistants"**

Meta-analysis of 78 studies (2021–2026). Attack success rates against state-of-the-art defenses **exceed 85%** when adaptive attack strategies are used.
