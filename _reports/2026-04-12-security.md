# Security Report — 2026-04-12

> Vulnerabilities, supply chain attacks, and safety incidents affecting AI coding agents.

---

## Incidents & Vulnerabilities

### RoguePilot — GitHub Copilot Silent Agent Takeover
**Status:** Disclosed | No CVE assigned at time of writing

Researchers at Orca Security demonstrated that attackers can craft hidden instructions inside a GitHub Issue that are automatically processed by GitHub Copilot. The technique gives attackers silent control of the in-Codespaces AI agent — including reading repository files, exfiltrating secrets, and committing code — without any visible indication to the developer. Attack requires no repository access; any party able to open an issue (i.e., anyone with a GitHub account on a public repo) can trigger the chain.

---

### CamoLeak — GitHub Copilot Chat Silent Source Code Exfiltration
**CVE-2025-59145** | CVSS: 9.6 | **Status:** Patched

A high-severity vulnerability in GitHub Copilot Chat allowed attackers to silently steal source code, API keys, and secrets from private repositories without executing a single line of malicious code. Disclosed by Legit Security, the flaw abused Copilot's data-routing behavior to redirect sensitive context to an attacker-controlled endpoint. No user interaction was required beyond opening a crafted file in the IDE.

---

### Rules File Backdoor — Cursor & GitHub Copilot Config Injection
**Researcher:** Pillar Security | **Status:** Disclosed; mitigations available

Pillar Security researchers uncovered a technique enabling attackers to silently compromise AI-generated code by injecting hidden malicious instructions into `.cursorrules`, `.github/copilot-instructions.md`, and similar configuration files used by Cursor and GitHub Copilot. Because these files control agent behavior project-wide, a single poisoned config in a shared or forked repository can affect every developer who clones it. Attack artifacts are not visible in standard code review.

---

### GitHub Issues Abused in Copilot Attack — Repository Takeover
**Researcher:** SecurityWeek / multiple | **Status:** Disclosed

A related campaign documented by SecurityWeek shows GitHub Issues being weaponized to embed invisible agent instructions that, when processed by Copilot Workspace, achieve full repository takeover — modifying code, altering CI/CD pipelines, and creating malicious pull requests autonomously. Distinct from RoguePilot in that it specifically targets the Workspace product rather than the Codespaces agent.

### AI Autonomous Compromise of Hardened OS — Research Demonstration
**Status:** Research / PoC | April 2026

Researchers demonstrated an AI agent autonomously compromising a hardened operating system configuration in under 4 hours, with no human guidance after initial task assignment. The demonstration underscores the offensive capability uplift provided by agentic systems and the speed asymmetry between automated attack chains and human defenders.

## Supply Chain

### LiteLLM PyPI Supply Chain Compromise
**Affected packages:** `litellm==1.82.7`, `litellm==1.82.8` | **Status:** Quarantined; versions yanked | **Incident date:** 2026-03-24

The most significant confirmed AI supply chain incident to date. Threat actor group **TeamPCP** published backdoored versions of the `litellm` Python package (3.4 million daily downloads) after stealing PyPI credentials via a compromised Trivy GitHub Action in LiteLLM's CI/CD pipeline. The attack was a three-stage campaign:

1. **2026-03-19:** TeamPCP compromised the Trivy open-source vulnerability scanner, injecting malware into official GitHub Actions workflows and Docker images.
2. **2026-03-24 ~10:39 UTC:** Backdoored `litellm` 1.82.7 and 1.82.8 published to PyPI. Versions were live for approximately **40 minutes** before PyPI quarantined them.
3. **Payload:** A malicious `.pth` file (`litellm_init.pth`) that executes automatically on every Python process startup, functioning as a multi-stage credential stealer targeting LLM provider API keys, cloud credentials, and environment variables.

**Downstream impact:** Mercor confirmed a security incident directly linked to the LiteLLM compromise (disclosed 2026-04-01). BerriAI engaged Mandiant for incident response. The rapid community self-coordination across GitHub, X, and security vendor channels limited real-world credential exfiltration.

**Fix:** Upgrade to `litellm>=1.82.9`. Rotate any API keys or cloud credentials present in environments where 1.82.7 or 1.82.8 were installed.

### Kaspersky: Trivy, Checkmarx, and LiteLLM Trojanization Campaign
**Actor:** TeamPCP | **Status:** Active investigation

Kaspersky published analysis confirming that the LiteLLM attack was part of a broader coordinated campaign also targeting Trivy (vulnerability scanner) and Checkmarx (SAST tool). The attack vector in each case was CI/CD pipeline compromise — injecting malicious steps into GitHub Actions workflows used by the upstream project itself. The selection of security tooling as the initial vector is deliberate: these tools run with elevated trust and broad filesystem/network access in developer environments.

### Slopsquatting — Updated Scale Metrics
**Term coined by:** Seth Larson | **Status:** Ongoing threat | No CVE

AI models hallucinate non-existent package names at scale; attackers register those names with malicious payloads. Updated figures through January 2026:

- ~20% of generated code samples reference a hallucinated package name (576,000 samples tested)
- 58% of hallucinated names recur consistently across repeated runs — indicating stable, predictable squatting targets
- 128 tracked phantom packages accumulated **121,539 downloads** (July 2025–January 2026), ~4,000/week
- `openapi-generator-cli` (npm) alone: **48,356 downloads**
- Coding agents (Cursor, Claude Code, GitHub Copilot Workspace) are particularly high-risk: they autonomously install packages without waiting for developer review

## Research & Advisories

### Prompt Injection Attack Frequency — 2026 Escalation
**Source:** OWASP LLM Security Project, multiple vendors

Prompt injection attacks against production AI deployments have increased **340%** in 2026. Key statistics:

- OWASP classifies prompt injection as the single highest-severity vulnerability category for deployed LLMs
- Indirect injection (malicious content embedded in documents, web pages, code comments) now accounts for **>80%** of documented enterprise attack attempts; direct injection is less than 20%
- **73%** of production AI deployments assessed during security audits contain at least one prompt injection vector (OWASP)
- **97%** of organizations lack proper AI access controls (Cloud Security Alliance 2026)

---

### "Lethal Trifecta" — Prompt Injection to Full Compromise
**Source:** Airia Research | Published: 2026

Security researchers formalized the "Lethal Trifecta" — the combination of three agent capabilities that transforms prompt injection into full system compromise:

1. **External content access** — agent reads attacker-controlled data (web pages, files, PR descriptions)
2. **Persistent memory or state** — injected instruction survives across sessions
3. **Action execution capability** — agent can write files, call APIs, or execute code

When all three are present, a single successful injection can achieve arbitrary code execution, credential exfiltration, or persistent backdoor installation. Most current coding agents (Claude Code, Cursor, GitHub Copilot Workspace, Devin) satisfy all three conditions by design.
