## Incidents & Vulnerabilities

### IDEsaster — 30+ CVEs Across All Major AI IDEs (Disclosed December 2025)

Researcher Ari Marzouk published **IDEsaster** after six months of testing, disclosing 30+ exploitable vulnerabilities across every major AI coding tool. The core attack chain is consistent: **Prompt Injection → AI Tool Calls → Base IDE Features → RCE or data exfiltration**. No tested tool was found clean.

**GitHub Copilot**
- **CVE-2025-53773** (CVSS 7.8) — Command injection leading to RCE. **Patched.**
- **CVE-2025-64660** — Multi-root workspace config override enabling code execution. **Patched.**
- **CVE-2025-64671** — JetBrains plugin: opening a malicious repository triggers arbitrary command execution. **Patched.**

**Cursor**
- **CVE-2025-49150** — Remote JSON schema data exfiltration. **Patched.**
- **CVE-2025-54130** — IDE settings overwrite. **Patched.**
- **CVE-2025-54135** ("CurXecute") — RCE via new `.cursor/mcp.json`, bypassing existing approval dialogs. **Patched in v1.3.9.**
- **CVE-2025-54136** ("MCPoison") — Trusted MCP server name hijack enabling silent malicious execution. **Patched in v1.3.9.**
- **CVE-2025-61590** — Multi-root workspace manipulation. **Patched.**

**Roo Code**
- **CVE-2025-53097**, **CVE-2025-53536**, **CVE-2025-58372** — Workspace configuration injection leading to code execution. **Patched.**

**OpenAI Codex CLI**
- **CVE-2025-61260** — Implicit MCP command trust at startup enables arbitrary command execution. **Patched.**

**Amazon Q / Kiro**
- **AWS-2025-019** — Prompt injection in Amazon Q Developer. **Patched.**

Sources: [The Hacker News](https://thehackernews.com/2025/12/researchers-uncover-30-flaws-in-ai.html), [CyberSecurityNews](https://cybersecuritynews.com/critical-vulnerabilities-in-github-copilot-gemini-cli-claude/), [byteiota](https://byteiota.com/idesaster-30-cves-hit-cursor-github-copilot-all-ai-ides/)

### Claude Code — Hook Injection RCE & API Key Exfiltration (Check Point Research, February 2026)

Check Point Research disclosed two critical vulnerabilities in Claude Code:

- **CVE-2025-59536** (CVSS 8.7) — Exploits the Claude Code "Hooks" feature. A malicious hook embedded in `.claude/settings.json` within a repository triggers **RCE the moment a developer opens the project**, before any trust dialog is shown. **Patched.**
- **CVE-2026-21852** — Configuration file manipulation intercepts Claude Code API communications and routes them to an attacker-controlled server, silently **harvesting the developer's Anthropic API key** with zero user interaction. **Patched.** Anthropic addressed residual risk via security documentation warnings rather than code changes alone.

Source: [Dark Reading](https://www.darkreading.com/application-security/flaws-claude-code-developer-machines-risk)

### Cursor & Windsurf — 94 Unpatched Chromium CVEs (OX Security, October 2025)

Both Cursor and Windsurf ship on outdated Electron/Chromium forks. As of their last Chromium update (v0.47.9, Chromium 132.0.6834.210), **94+ CVEs** published against the embedded Chromium/V8 have not been addressed. Researchers successfully weaponized **CVE-2025-7656** (a memory corruption bug, already patched upstream) against the latest releases of both IDEs, affecting an estimated **1.8 million developers**.

- **Windsurf** — did not respond to coordinated disclosure (October 12, 2025). **Unpatched.**
- **Cursor** — dismissed the issue as "self-DOS." **Unpatched.**

Source: [OX Security](https://www.ox.security/blog/94-vulnerabilities-in-cursor-and-windsurf-put-1-8m-developers-at-risk/)

### Microsoft Copilot — EchoLeak (CVE-2025-32711)

A prompt injection in Microsoft Copilot: specially engineered email prompts cause Copilot to **automatically exfiltrate sensitive data without user interaction**. Demonstrates cross-product risk of prompt injection in AI tools operating with broad mailbox permissions. **Patched.**

## Supply Chain

### Slopsquatting — Hallucinated Dependency Attacks

Research from UT San Antonio, Virginia Tech, and the University of Oklahoma (2025) analyzed 576,000 code samples from 16 LLMs:

- ~**20% of recommended packages did not exist** in any public registry
- **440,445 hallucinated package references**, including 205,474 unique invented names
- **43% of hallucinated packages were consistently repeated** across 10 queries, making them predictably exploitable at near-zero cost
- Commercial models (GPT-4): ~5% hallucination rate; open-source models (CodeLlama, DeepSeek, Mistral): significantly higher
- Endor Labs: **34% of AI-suggested dependencies didn't exist**, and 44–49% of suggested dependency versions had **known vulnerabilities**

Attack chain: Attacker registers hallucinated names on PyPI/npm with malicious payloads, then waits for developers (or autonomous agents) to run AI-generated install commands. No confirmed wild exploitation reported as of early 2026, but the attack surface is considered highly predictable.

Sources: [Dark Reading](https://www.darkreading.com/application-security/ai-code-tools-widely-hallucinate-packages), [Trend Micro](https://www.trendmicro.com/vinfo/us/security/news/cybercrime-and-digital-threats/slopsquatting-when-ai-agents-hallucinate-malicious-packages)

### "s1ngularity" npm Campaign (August 2025)

A sophisticated supply chain attack on a widely-used build system npm package:

- Collected cryptocurrency wallets, GitHub tokens, npm tokens, and SSH keys
- **Weaponized installed AI CLI tools** (running them with dangerous permission flags to bypass security controls)
- Specifically targeted **AI LLM client credentials** — config files and auth tokens for Claude, Gemini, and Amazon Q
- Palo Alto Unit 42 assessed with moderate confidence that **AI was used to generate the malicious scripts** based on code comment patterns

### Rules File Backdoor — Hidden Unicode (Pillar Security, 2025)

Hidden Unicode characters injected into `.cursorrules` or Copilot configuration files cause the AI assistant to **produce malicious code output invisible to human code reviewers**. GitHub implemented a warning for hidden Unicode on github.com. Cursor declined to treat this as a vulnerability.

Source: [Pillar Security](https://www.pillar.security/blog/new-vulnerability-in-github-copilot-and-cursor-how-hackers-can-weaponize-code-agents)

### Shai-Hulud 2.0 — Self-Propagating Supply Chain Worm (2026)

A self-propagating worm activating during build processes:

- Deployed a weaponized TruffleHog scanner to exfiltrate secrets
- Used stolen npm tokens to backdoor victim packages, infecting downstream users recursively
- Exfiltrated credentials sent to public GitHub repositories created with stolen victim credentials

## Research & Advisories

### MCP Infrastructure — Systemic Vulnerability (Multiple Researchers)

An analysis of **67,057 MCP servers** across 6 public registries:

- **43%** contain command injection vulnerabilities (path to RCE)
- **36.7%** vulnerable to SSRF
- **33%** allow unrestricted outbound network access
- **22%** allow file access outside intended data sources
- **5%** already seeded with active tool poisoning attacks

**Anthropic mcp-server-git — Three Chained CVEs**
- **CVE-2025-68145** — Path validation bypass
- **CVE-2025-68143** — Unrestricted `git_init` can convert `.ssh` into a git repo
- **CVE-2025-68144** — Argument injection in `git_diff`
Combined with the Filesystem MCP server, these achieve full RCE via malicious `.git/config` files. Patch status not confirmed.

**mcp-remote — CVE-2025-6514** (JFrog, Critical)
Critical OS command-injection in `mcp-remote`, a widely used OAuth proxy connecting local MCP clients to remote servers.

**Microsoft MarkItDown MCP — SSRF to Cloud Metadata**
Unpatched SSRF enabling AWS EC2 instance metadata service access to harvest cloud infrastructure credentials. Microsoft classified as low risk.

**Anthropic MCP Inspector — Unauthenticated RCE**
Anthropic's own MCP Inspector developer tool allowed unauthenticated RCE via its `inspector-proxy`. An attacker could execute arbitrary commands on a developer's machine by having the victim inspect a malicious MCP server, or by driving the inspector from a browser tab. The inspector ran with user privileges, listening on localhost/0.0.0.0 with no authentication.

**MCP Sampling Attack Vectors (Unit 42, Palo Alto Networks)**
Three attack classes exploiting MCP's sampling capability: (1) covert tool invocation — hidden file and system operations; (2) conversation hijacking — persistent instruction injection across sessions; (3) resource theft — draining AI compute quotas for unauthorized workloads.

**Smithery Platform Path Traversal**
Path traversal in Smithery's hosted MCP registry via `dockerBuildPath` — researchers set it to `".."` to initiate builds from the builder's home directory, leaking `~/.docker/config.json` including a Fly.io API token granting control over **3,000+ hosted MCP server apps**.

**WhatsApp Exfiltration via MCP Tool Poisoning (Invariant Labs)**
Demonstrated that a malicious MCP server can silently exfiltrate a user's entire WhatsApp message history by combining tool poisoning with a legitimate `whatsapp-mcp` server in the same agent context.

**Real-World Breach — Supabase Cursor Agent**
Supabase's Cursor agent, running with privileged service-role database access, processed support tickets containing user-supplied SQL. Attackers embedded SQL instructions causing the agent to exfiltrate sensitive integration tokens into a public support thread.

Sources: [AuthZed Timeline](https://authzed.com/blog/timeline-mcp-breaches), [Unit 42](https://unit42.paloaltonetworks.com/model-context-protocol-attack-vectors/), [vulnerablemcp.info](https://vulnerablemcp.info/), [eSentire](https://www.esentire.com/blog/model-context-protocol-security-critical-vulnerabilities-every-ciso-should-address-in-2025)

### Academic & Industry Research

- AI-generated code introduces bugs at **1.5–2x the rate of human-written code**, including improper password handling and insecure object references (multiple 2025 studies).
- **80% of dependencies proposed by AI coding tools fail safety standards** — Endor Labs 2025 State of Dependency Management.
- **Only 29% of organizations** deploying agentic AI report being prepared to secure those deployments — Cisco State of AI Security 2026.
- ArXiv paper "Agentic AI Security: Threats, Defenses, Evaluation, and Open Challenges" documents systematic attack taxonomies including reward hacking, power concentration, and agentic spear-phishing, with a demonstrated case using OpenAI's Operator agent for automated credential stuffing.
- Between October 2025 and January 2026, security researchers documented **over 91,000 attack sessions** specifically targeting AI infrastructure.
