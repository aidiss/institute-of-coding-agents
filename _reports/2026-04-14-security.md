## Incidents & Vulnerabilities

### CrewAI — Four Chained CVEs Enable Sandbox Escape and RCE via Prompt Injection
**CVEs:** CVE-2026-2275, CVE-2026-2287, CVE-2026-2285, CVE-2026-2286 | **Status:** Partial mitigations only — full patch pending | **Disclosed:** 2026-04-02

Four security vulnerabilities disclosed in CrewAI, an open-source Python framework for managing multi-agent AI systems, can be chained to achieve sandbox escape, remote code execution on the host, local file read, and credential theft. All four can be triggered via both direct and indirect prompt injection.

| CVE | Type | Detail |
|---|---|---|
| CVE-2026-2275 | Sandbox escape / RCE | Code Interpreter falls back to `SandboxPython` when Docker is unreachable; `SandboxPython` permits arbitrary C function calls |
| CVE-2026-2287 | Docker liveness bypass | CrewAI does not verify Docker is still running during execution, silently degrading to the insecure fallback sandbox |
| CVE-2026-2285 | Local file read | JSON loader tool lacks file-path validation, allowing access to arbitrary host files |
| CVE-2026-2286 | SSRF | RAG search tool can be abused to reach internal services and cloud metadata endpoints |

A practical attack chain: inject into an agent that uses Code Interpreter → cause Docker check to fail (or exploit CVE-2026-2287) → execute arbitrary code in the fallback sandbox → read credentials via CVE-2026-2285 or pivot internally via CVE-2026-2286.

**Mitigation:** The CrewAI team is actively developing fixes. Interim workarounds include disabling the Code Interpreter tool entirely, ensuring Docker is always reachable, and blocking outbound RAG search to untrusted endpoints. No complete patch is available at time of writing.

### IDEsaster — 30+ Flaws Across AI Coding Tools, 24 CVEs Assigned
**CVEs:** 24 assigned (full list pending coordinated disclosure) | **Status:** Disclosure in progress | **Affected:** GitHub Copilot, Cursor, Windsurf, Gemini CLI, Claude Code, Roo Code, Cline, Kiro.dev, Junie, Zed.dev

Researcher Ari Marzouk disclosed a novel vulnerability class dubbed "IDEsaster" covering over 30 flaws across market-leading AI-assisted IDEs. All tested AI IDEs were confirmed vulnerable. The root cause is that AI agents capable of reading, writing, and executing files interact with long-standing IDE mechanisms that were never designed for autonomous components.

Two primary attack primitives:

**Remote JSON Schema Exfiltration** — VS Code, JetBrains, and Zed.dev automatically trigger GET requests to resolve JSON schemas referenced in workspace configuration files. An AI agent tricked into writing a malicious `.vscode/settings.json`, `.idea/workspace.xml`, or equivalent can cause the IDE to exfiltrate data (environment variables, source trees, secrets) to an attacker-controlled domain with no further user interaction.

**IDE Settings Overwrite → RCE** — Manipulating configuration files such as `.vscode/settings.json` to define custom task runners or terminal profiles allows arbitrary command execution triggered automatically on next workspace open or build.

**Affected versions:** No single version boundary; the attack surface is inherent to the interaction model. Individual product vendors are shipping fixes incrementally as CVEs are assigned.

### CamoLeak — GitHub Copilot Silent Code Exfiltration via Unicode
**CVE:** CVE-2025-59145 | **CVSS:** 9.6 | **Status:** Patched

GitHub Copilot Chat was vulnerable to a prompt injection attack (CamoLeak) exploiting invisible Unicode characters embedded in code or documentation. An attacker could craft content that, when processed by Copilot Chat, silently exfiltrated source code and secrets to an external endpoint — with no visible indication to the developer. CVSS 9.6 reflects the combination of zero user interaction required and potential for large-scale confidential data loss.

**Fix:** Patch deployed by GitHub; no user action required for cloud-hosted Copilot. Enterprise customers using self-hosted or pinned versions should verify their deployment is current.

### Cursor — Case-Sensitivity Bypass Enables Persistent RCE Across Restarts
**CVE:** CVE-2025-59944 | **Status:** Patched | **Affected:** Cursor IDE (affected versions not yet fully enumerated)

A case-sensitivity bypass in Cursor's MCP configuration validation allowed attackers to install a persistent malicious MCP server that survived IDE restarts. Because the persistence mechanism was established through a configuration path the validator did not recognise as authoritative (due to OS-level case folding differences between the path as written and as stored), the backdoor reloaded on each IDE startup. Exploitation required an initial prompt injection to trigger the configuration write.

**Fix:** Upgrade to patched Cursor release. Inspect `~/.cursor/mcp.json` (and case variants) for unexpected server entries.

## Supply Chain

### Axios npm Supply Chain Compromise — North Korean State Actor
**Incident date:** 2026-03-31 | **Status:** Malicious versions pulled; attribution confirmed | **Affected versions:** axios@1.14.1, axios@0.30.4

On March 31, 2026, two backdoored releases of the axios npm package were published after attackers compromised the npm account of lead maintainer `jasonsaayman` via a targeted social engineering campaign. The malicious versions were live for approximately three hours before removal.

**Attack mechanism:** Both releases injected `plain-crypto-js@4.2.1`, a phantom package that had never existed on npm prior to that day. Its sole content was a `postinstall` script that connected to Sapphire Sleet C2 infrastructure and downloaded a second-stage cross-platform RAT (macOS, Windows, Linux). Because axios is commonly referenced with a range specifier (`^1.14.0`, `^0.30.0`), any project that ran `npm install` during the window received the malicious payload without any dependency-file changes visible to reviewers.

**RAT capabilities:** Arbitrary PowerShell/shell execution, filesystem enumeration, in-memory binary injection, C2-directed payload switching.

**Attribution:** Microsoft Threat Intelligence attributed the infrastructure to **Sapphire Sleet** (North Korean state actor, active since at least March 2020, historically focused on finance/crypto/VC). Google's Mandiant team separately attributed the attack to **UNC1069** — an attribution discrepancy that is unresolved at time of writing; both groups may be tracking the same cluster under different internal designations.

**Scope:** axios is one of the most widely downloaded npm packages (hundreds of millions of weekly downloads). Any CI/CD pipeline, developer machine, or deployment environment that ran `npm install` between approximately 14:00–17:00 UTC on March 31 should be treated as potentially compromised.

**Recommended actions:**
- Audit `node_modules` for `plain-crypto-js`; its presence confirms compromise
- Rotate all secrets, tokens, and credentials in affected environments
- Review network logs for outbound connections to unfamiliar hosts during the window
- Pin or lock axios to `1.14.0` / `0.30.3` or the latest clean release; verify checksums against the npm registry

### Slopsquatting — Updated Metrics and Agentic Amplification
**Status:** Ongoing threat | No CVE

Updated data on AI-hallucinated package name squatting reinforces the risk, particularly in the context of autonomous coding agents that install packages without human review:

- ~20% of generated code samples (across 756,000 tested Python and JavaScript samples) include at least one non-existent package name
- 38% of hallucinated names are derivatives of real packages; 13% are typo variants; 51% are fully fabricated
- 58% of hallucinated names are **consistent across repeated model runs** — providing stable, predictable squatting targets that can be automated by monitoring LLM outputs
- 128 tracked phantom packages accumulated over 121,000 downloads between July 2025 and January 2026

The Axios incident adds a compounding dimension: the phantom dependency technique used by Sapphire Sleet (registering a package name that had never existed, relying on postinstall execution) is structurally identical to slopsquatting exploitation. Autonomous agents running `npm install` on agent-generated code without review collapse the window between hallucination and execution to near zero.

### "Vibe Coding" CVE Surge — 35 AI-Attributed CVEs in March 2026
**Source:** Georgia Tech Vibe Security Radar / Cloud Security Alliance | **Status:** Ongoing tracking

The Cloud Security Alliance Research team recorded 35 CVEs directly attributable to AI-generated code in March 2026 — a near-sixfold increase from January's baseline of 6. Lifetime tracking now covers 74 confirmed AI-tool-attributed CVEs across approximately 50 tools. Claude Code accounts for 27 confirmed cases; researchers note this is partly because Claude Code embeds traceable signatures in generated code, making attribution easier, and that the true distribution across tools is likely more even.

Separately, Veracode data shows that teams using AI coding assistants committed code at three to four times the rate of non-AI peers, and monthly security findings in those repositories rose from ~1,000 to over 10,000 — a tenfold surge over six months.

## Research & Advisories

### Prompt Injection Surge — 340% Increase in 2026, >85% Adaptive Attack Success
**Sources:** Airia Research; arXiv:2601.17548; OWASP LLM Top 10 2026

A systematic review of 78 prompt injection studies (arXiv:2601.17548, January 2026) found every tested coding agent — Claude Code, GitHub Copilot, Cursor — vulnerable to prompt injection. Adaptive attack success rates exceed **85%** across all tested agents. Indirect injection (malicious payloads embedded in code comments, README files, PR descriptions, issue bodies, dependency changelogs) now accounts for over 80% of documented enterprise attack attempts; direct injection is below 20%.

OWASP LLM Top 10 2026 audit data shows prompt injection present in 73% of production AI deployments surveyed. Airia Research reports a 340% year-over-year increase in documented prompt injection incidents.

### Weaponizing Claude Code Lures and GitHub Release Payloads
**Source:** Trend Micro | **Status:** Active threat

Trend Micro published research on threat actors abusing the trusted reputation of Claude Code branding to deliver malware. Attack chains use convincing Claude Code update notifications or GitHub release page spoofs as lures, directing developers to execute malicious scripts under the guise of legitimate tool installation. The technique exploits the tendency of developers to run installer scripts from GitHub releases without verification.

**Countermeasure:** Verify all Claude Code installations through official Anthropic channels (`npm install -g @anthropic-ai/claude-code`) and validate package checksums. Avoid running unsigned scripts linked from social media, issue comments, or third-party blog posts.
