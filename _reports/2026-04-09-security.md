# Security Brief — AI Coding Agents
**2026-04-09**

---

## Incidents & Vulnerabilities

### PraisonAI — Critical RCE via YAML Parsing & Template Injection (CVE-2026-39890, CVE-2026-39891)

Disclosed April 8, 2026. Two high-severity vulnerabilities in the PraisonAI agent framework were published simultaneously.

**CVE-2026-39890 (CVSS 9.8):** Specially crafted YAML agent-definition files trigger arbitrary JavaScript execution on the server via insecure YAML parsing. An attacker who can upload or supply a YAML config — common in multi-tenant deployments or developer workflows that accept user-defined agents — achieves unauthenticated RCE with server privileges.

**CVE-2026-39891 (CVSS 8.8):** Unescaped user input in `agent.start()` is processed as template expressions by agent-centric tools, enabling unintended code execution through template injection. Requires authenticated access or ability to supply agent input.

**Status: Disclosed April 8, 2026. Patch status unclear. Avoid untrusted YAML agent definitions until vendor confirms a fix.**

---

### Flowise — CVSS 10.0 RCE Under Active Exploitation (CVE-2025-59528)

First disclosed September 2025; active exploitation in the wild confirmed as of the April 2026 reporting period. The `CustomMCP` node executes user-supplied JavaScript with full Node.js privileges — including `child_process` (OS command execution) and `fs` (filesystem access) — without any sandboxing or sanitization.

- **Affected versions:** All releases prior to 3.0.6
- **Fixed version:** 3.0.6+ (latest: 3.1.1)
- **Exposed instances:** 12,000–15,000 publicly accessible deployments; exploitation observed from Starlink IP ranges
- **Impact:** Full server compromise

**Status: Patched in v3.0.6. Actively exploited. Any internet-exposed Flowise instance should be treated as compromised until patched and audited for IOCs.**

---

### Langflow — Unauthenticated RCE as Root (CVE-2026-0768)

The `/validate` endpoint passes unsanitized user input directly to the Python interpreter. No authentication is required. Network-accessible Langflow instances can be fully compromised in a single unauthenticated request. A prior Langflow RCE (CVE-2025-3248) was added to the CISA Known Exploited Vulnerabilities catalog after being weaponized to deliver the Flodrix botnet; this new vulnerability follows the same pattern.

**Status: Patch availability unconfirmed as of April 9, 2026. Restrict network access to Langflow instances immediately. Do not expose to the public internet.**

### IDEsaster — 30+ Vulnerabilities Across 10 AI IDEs (December 2025 — Ongoing Remediation)

Researcher Ari Marzouk's six-month audit of AI-integrated development environments produced 30+ vulnerabilities across Cursor, Windsurf, GitHub Copilot, Roo Code, Cline, JetBrains Junie, Zed.dev, Kiro.dev, Gemini CLI, and Claude Code — with 24 CVEs assigned. Approximately 1.8 million developers were estimated at risk.

The attack chain common to 100% of tested tools: **Prompt Injection → Agent Tool Use → Base IDE Feature Weaponization**.

Three primary attack classes:

1. **Remote JSON Schema injection** (VS Code, JetBrains, Zed.dev): Prompt injection causes the agent to write a file whose JSON schema reference points to an attacker-controlled URL. When the IDE validates the schema, it makes an outbound GET request leaking file contents.
   - *CVE-2025-58335 (JetBrains Junie), CVE-2025-55012 (Zed.dev)*

2. **IDE Settings Overwrite**: Agent overwrites `.vscode/settings.json` or `.idea/workspace.xml` with attacker-controlled values (e.g., custom task runners, terminal profiles) that execute arbitrary commands on the next workspace open.
   - *CVE-2025-49150, CVE-2025-54130 (Cursor), CVE-2025-53097, CVE-2025-58372 (Roo Code)*

3. **Multi-root Workspace Hijacking**: `.code-workspace` files reference attacker-controlled folders, overriding trusted project settings.

A separate OX Security finding: Cursor and Windsurf are built on outdated Chromium versions with 94+ known unpatched CVEs, compounding the IDEsaster attack surface.

**Status: Most CVEs patched in vendor updates (Cursor 1.7, Junie update Dec 2025). Windsurf patches incomplete. Gemini CLI unresolved as of report date.**

### Claude Code — Three CVEs Covering RCE and API Key Exfiltration

Three vulnerabilities in Anthropic's Claude Code CLI were disclosed between mid-2025 and early 2026, all exploitable via malicious repository files.

| CVE | Type | Fixed Version | Disclosed |
|-----|------|--------------|-----------|
| CVE-2025-59536 | RCE via hooks in `.claude/settings.json` | 1.0.111 | Jul 2025 |
| CVE-2026-21852 | API key exfiltration via `ANTHROPIC_BASE_URL` redirect | 2.0.65 | Feb 2026 |
| CVE-2025-54795 | Argument injection in pre-approved commands | Coordinated disclosure ongoing | Oct 2025 |

The attack model is consistent across all three: a malicious repository contains configuration files that Claude Code processes automatically on launch, before any user trust prompt. Opening an untrusted repository in Claude Code is equivalent to running untrusted code.

**Status: CVE-2025-59536 and CVE-2026-21852 patched. CVE-2025-54795 in coordinated disclosure. Upgrade to latest Claude Code release.**

### GitHub Copilot — CamoLeak Zero-Click Exfiltration (CVE-2025-59145, CVSS 9.6)

Hidden instructions embedded in PR descriptions exploited GitHub's image proxy infrastructure as a CSP bypass. Copilot Chat silently exfiltrated secrets and source code from private repositories with no user clicks required. The AI itself acted as the exfiltration channel.

**Status: Patched August 14, 2025 (GitHub disabled image rendering in Copilot Chat). No further action needed for GitHub-hosted Copilot. Self-hosted or API-integrated deployments should verify the patch is applied.**

---

### GitHub Copilot + VS Code — RCE via Prompt Injection (CVE-2025-53773, CVSS 7.8)

Attackers embed prompt injection instructions in source file comments or documentation. When Copilot processes the file, the injected instructions cause it to execute attacker-controlled commands locally via shell execution flags (`find -exec`, `go test -exec`). Patched in Microsoft's August 2025 Patch Tuesday.

A related bypass (CVE-2025-64660) allowed code execution over a network via improper access control in the VS Code extension. Patched November 2025.

**Status: Both patched. Ensure VS Code and Copilot extensions are current.**

---

### Cursor — MCPoison MCP Rug-Pull (CVE-2025-54136, CVSS 7.2)

After a user approves a legitimate MCP server configuration in a shared repository, an attacker silently replaces the MCP config file contents with malicious commands. Cursor re-executes the now-malicious config on subsequent launches without re-prompting for approval.

- **Affected:** Cursor 1.2.4 and below
- **Fixed:** Cursor 1.3

A related case-sensitivity bypass (CVE-2025-59944) allowed untrusted content to overwrite protected configuration files, also leading to RCE. Fixed in Cursor 1.7.

**Status: Patched. Users should pin MCP server configurations by commit hash and treat any MCP config update as a new approval decision.**

---

### Anthropic MCP Inspector — Critical RCE via Unauthenticated Proxy (CVE-2025-49596, CVSS 9.4)

The MCP Inspector's proxy server has no authentication. Attackers exploit the "0.0.0.0 Day" browser behavior: a malicious website sends requests to the locally running Inspector at `0.0.0.0`, gains unauthenticated access, and executes arbitrary commands on the developer's machine.

**Status: Patched in version 0.14.1 (June 13, 2025).**

---

### mcp-server-git — Chained RCE (CVE-2025-68143, CVE-2025-68144, CVE-2025-68145)

Three chained vulnerabilities in Anthropic's official `mcp-server-git`:

- **CVE-2025-68145:** Path traversal bypass in `--repository` flag allows reaching repos outside the configured root.
- **CVE-2025-68143:** `git_init` accepts arbitrary filesystem paths (e.g., `~/.ssh`) without validation.
- **CVE-2025-68144:** Argument injection in `git_diff`/`git_checkout` via unsanitized GitPython arguments.

**Attack chain:** Initialize `~/.ssh` as a git repo via `git_init`, write a malicious `.git/config` with hook commands via the Filesystem MCP server, trigger execution through argument injection. Full RCE results.

Reported to Anthropic June 2025; accepted September 2025; `git_init` tool removed entirely in version 2025.12.18.

**Status: Patched December 2025. Upgrade to mcp-server-git 2025.12.18+.**

---

### mcp-remote — OS Command Injection / RCE (CVE-2025-6514, CVSS 9.6)

A malicious MCP server returns a crafted `authorization_endpoint` URL containing shell metacharacters. The `mcp-remote` client passes this unsanitized to OS command execution, granting the attacker a full shell on the client machine. This was the first demonstrated OS-level RCE via an untrusted MCP server response.

- **Affected:** versions 0.0.5–0.1.15 (437,000+ downloads at disclosure)
- **Fixed:** 0.1.16

**Status: Patched. Upgrade mcp-remote to 0.1.16+.**

## Supply Chain

### LiteLLM PyPI Compromise — TeamPCP Supply Chain Attack (CVE-2026-33634, CVSS 9.4)

Date: March 24, 2026. The most significant AI tooling supply chain incident of the period.

**Attack chain:**
1. Threat actor **TeamPCP** compromised the Trivy open-source security scanner GitHub Action used in LiteLLM's CI/CD pipeline.
2. The compromised Trivy action exfiltrated the `PYPI_PUBLISH` token from the GitHub Actions runner.
3. Attackers published `litellm` versions 1.82.7 and 1.82.8 containing malicious payloads.
4. Version 1.82.8 used Python's `.pth` file mechanism — a Python interpreter initialization hook — ensuring the payload runs on every `python` invocation on the infected system.

LiteLLM is downloaded ~3.4 million times per day. Malicious versions were quarantined within ~3 hours. TeamPCP's broader campaign simultaneously targeted GitHub Actions, Docker Hub, npm (CanisterWorm campaign), and OpenVSX.

**Status: Quarantined. Clean version: v1.83.0+. Docker users unaffected. All hosts that ran v1.82.7 or v1.82.8 should be treated as compromised and credentials rotated.**

Post-incident hardening disclosures from Veria Labs (April 2026):

| CVE | Severity | Description | Condition |
|-----|----------|-------------|-----------|
| CVE-2026-35030 | Critical | Auth bypass in JWT path | `enable_jwt_auth=true` only (off by default) |
| CVE-2026-35029 | High | Privilege escalation via valid API key | Requires authenticated access |
| GHSA-69x8-hrgq-fjj8 | High | Second authenticated escalation path | Same as above |

### OpenClaw / ClawHub — AI Agent Skill Marketplace Compromise (January–March 2026)

OpenClaw, an open-source AI agent platform, accumulated 25,000 GitHub stars in a single day in late January 2026. Its ClawHub skill marketplace became the largest confirmed AI agent supply chain compromise to date.

- Koi Security audited 2,857 ClawHub skills: **341 confirmed malicious** ("ClawHavoc" campaign), 335 traced to a single coordinated operation.
- By March 1, 2026: **1,184 confirmed malicious skills** out of 10,700+ total (~1 in 9 packages malicious).
- Attack method: Fake skills impersonating legitimate tools prompt users to paste terminal commands or download files. One skill masquerading as a Polymarket integration executed a hidden command opening an interactive reverse shell.
- Payloads: Atomic macOS Stealer, credential harvesters, remote access tools.

**Status: Ongoing remediation. Treat all ClawHub skills as untrusted until platform security model is revised.**

### Postmark-MCP — First Confirmed Malicious MCP Package on npm (September 2025)

A near-identical clone of the legitimate Postmark Labs MCP package was published to npm with a single added line that BCC'd every outgoing email to `phan@giftshop[.]club`. Across 15 incremental releases the actor mimicked official naming and versioning to build credibility.

- **Downloads:** 1,643
- **Estimated impact:** ~300 organizations; potentially 3,000–15,000 emails/day silently exfiltrated
- **Discovery:** September 25–29, 2025; package removed

**Status: Removed from npm. The first confirmed malicious MCP server in the wild; establishes that MCP package registries are a live attack surface.**

### Slopsquatting — AI Hallucinated Packages as Reliable Supply Chain Vector

No new in-the-wild incident as of April 9, 2026, but the structural risk continues to escalate as autonomous agents proliferate.

Key data points:
- **USENIX Security 2025 study (576,000 code samples, 16 LLMs):** ~19.7% of package suggestions were nonexistent. Of those, 43% were repeated across 10 queries — confirming the attack is statistically reliable, not random.
- Researcher Bar Lanyado registered the hallucinated package `huggingface-cli` on PyPI; it received 30,000+ authentic downloads in 3 months.
- **128 confirmed phantom packages** accumulated 121,539 downloads between July 2025 and January 2026 (~3,900/week).
- The hallucinated `react-codeshift` package propagated to 237 GitHub repositories via AI-generated agent skills that autonomously installed it.

The critical threat escalation in 2026: autonomous coding agents can `pip install` hallucinated packages in CI/CD environments **without human review**, at machine speed, with no lockfile or hash verification if the environment is not hardened.

**No systematic fix available. Mitigations: `--require-hashes` enforcement, private registries, lockfile pinning, and LLM output validation before package installation.**

### GlassWorm — 400+ Repositories via Unclaimed VSCode/OpenVSX Extensions (March 2026)

72 Open VSX extensions with unclaimed namespaces were registered by attackers. AI-focused IDEs built on VS Code forks (Cursor, Windsurf) source extensions from Open VSX rather than the Microsoft Marketplace, leaving recommended extensions claimable by anyone. Payloads landed across 400+ GitHub repositories, npm packages, and extension entries.

- Cursor patched December 2025; Google patched January 2026.
- **Windsurf: no vendor response to disclosure as of April 9, 2026. Users remain at risk.**

## Research & Advisories

### MCP Ecosystem: Structural Vulnerabilities in Authentication and Authorization

Large-scale audits of publicly exposed MCP servers (eSentire, Practical DevSecOps, Vulnerable MCP Project) reveal systemic weaknesses:

- **43%** have OAuth authentication flaws
- **43%** contain command injection vulnerabilities
- **33%** allow unrestricted network access
- **22%** allow file access outside intended data sources

The [Vulnerable MCP Project](https://vulnerablemcp.info/) now tracks 50+ disclosed MCP vulnerabilities. Notable additional CVEs beyond those covered in Incidents:

| CVE | Component | Type | Status |
|-----|-----------|------|--------|
| CVE-2026-25536 | MCP TypeScript SDK 1.10.0–1.25.3 | Cross-client data leak | Patched |
| CVE-2026-23744 | MCPJam Inspector ≤1.4.2 | Critical RCE | Patched |
| CVE-2026-0756 | GitHub Kanban MCP Server | Critical RCE | Patched |
| CVE-2026-0755 | gemini-mcp-tool | Command injection | Unpatched at disclosure |
| CVE-2025-66414/66416 | Official MCP SDKs | DNS rebinding | Patched |
| CVE-2025-67366 | filesystem-mcp | Path traversal | Patched |
| CVE-2025-53372 | node-code-sandbox-mcp | Docker sandbox escape | Patched |

**MCP Rug-Pull Design Gap (No CVE):** The MCP protocol has no mechanism to re-prompt user consent when a server silently modifies its tool definitions. A tool approved as a read-only browser can later be updated to include exfiltration capabilities with no client-side warning. No protocol-level fix in the MCP spec. **Mitigation: pin MCP server versions by commit hash; treat every server update as a new approval decision.**

### Tool Poisoning / "Line Jumping" — MCP Tool Description Exploits

Malicious content hidden in MCP server tool descriptions instructs the LLM to take actions invisible in the tool's visible name. The attack exploits the gap between what the user sees in the UI ("File Browser") and what the AI reads in the full tool schema (additional hidden instructions). Because LLMs follow instructions in tool descriptions with high fidelity, this is a reliable and low-noise attack vector requiring no code execution by the attacker — only control of a tool description field.

This attack class has no CVE assignment; it is a design-level risk inherent to the MCP architecture.

---

### Argument Injection in AI Agents — "Pre-Approved Command" Bypass (Trail of Bits, October 2025)

Trail of Bits demonstrated that AI agents which present a "human approval" step for shell commands can be bypassed by injecting malicious arguments into pre-approved command names. Agents approve the *command* (`go test`, `git show`, `ripgrep`) but not the full argument string. Injected flags (`-exec`, `--format`, `--output`, `--pre`) in code comments, file names, or log output redirect execution.

One-shot exploits work across all tested injection vectors. Three major AI agent platforms were disclosed under coordinated disclosure; CVE-2025-54795 (Claude Code) and a Cursor advisory were among assignments.

**The takeaway for defenders:** human-in-the-loop approval on shell commands is not a sufficient sandbox boundary when the agent is susceptible to prompt injection from file contents.

### Persistent Memory Poisoning in Agentic Workflows

A documented attack pattern: malicious instructions injected into an agent's persistent memory (via a document, email, or code file processed during a session) survive session restarts and are recalled days or weeks later. This enables attackers to establish persistent footholds in developer workflows — influencing future code generation, tool calls, or data access — without maintaining a continuous presence.

Combined with MCP tool use and autonomous agent execution, this forms the "Lethal Trifecta" identified by Adversa AI: **prompt injection + memory persistence + autonomous tool use**. When all three are present, a single injected instruction can persist across sessions, escalate through tool calls, and survive agent restarts.

## Key Mitigations

1. **Patch Flowise to v3.0.6+ immediately** — CVE-2025-59528 (CVSS 10.0) is actively exploited against 12,000+ exposed instances. Treat any unpatched internet-facing deployment as compromised.
2. **Isolate Langflow from the public internet** — CVE-2026-0768 requires no authentication; network access is sufficient for full RCE. Patch when available; restrict in the meantime.
3. **Upgrade LiteLLM to v1.83.0+** — Rotate all credentials on any host that ran v1.82.7 or v1.82.8.
4. **Treat repository cloning as code execution** — `.claude/settings.json`, `.cursor/mcp.json`, `.vscode/settings.json`, `.code-workspace`, and similar files are executed by AI coding agents on startup. Only open untrusted repositories in isolated environments.
5. **Pin MCP server versions by commit hash** — The MCP protocol offers no re-consent on tool definition changes. Treat every MCP server update as a new approval decision.
6. **Enforce `--require-hashes` in agent-accessible pip environments** — Prevents slopsquatting payloads from installing even when a hallucinated package name is registered.
7. **Vet MCP servers before enabling** — 43% of audited public MCP servers contain command injection vulnerabilities; 43% have OAuth flaws. Treat MCP servers as first-class dependencies requiring security review.
8. **Do not expose Flowise, Langflow, or PraisonAI without authentication** — All three have critical unauthenticated RCE vectors; none should be reachable from the public internet without a hardened auth layer.
9. **Audit CI/CD pipeline token scopes** — The LiteLLM/TeamPCP attack succeeded because a Trivy GitHub Action had access to a `PYPI_PUBLISH` token. Principle of least privilege on all CI/CD secrets; rotate any token that transited a compromised runner.
10. **Windsurf users: audit extensions** — No vendor response to GlassWorm disclosure; multiple unpatched Chromium CVEs; treat extension provenance as untrusted.
