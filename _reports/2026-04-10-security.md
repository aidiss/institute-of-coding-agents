# Security Brief — AI Coding Agents
**2026-04-10**

---

## Incidents & Vulnerabilities

### IDEsaster — 24 CVEs Across 10 AI IDEs (Ongoing)

Researcher Ari Marzouk's coordinated disclosure (dubbed "IDEsaster") covering six months of auditing AI-integrated developer environments reached its broadest documentation this week. **100% of tested tools were found vulnerable.** Affected tools: Cursor, Windsurf, GitHub Copilot, Roo Code, Cline, JetBrains Junie, Zed.dev, Kiro.dev, Gemini CLI, and Claude Code — with 24 CVEs assigned across the class.

The common attack chain: **Prompt Injection → Agent Tool Use → Base IDE Feature Weaponization**.

Three primary attack classes:

1. **Remote JSON Schema Injection** (VS Code, JetBrains, Zed.dev): Prompt injection causes the agent to write a file with a JSON schema reference pointing to an attacker-controlled URL. IDE validation sends an outbound GET request leaking workspace data.
   - *CVE-2025-58335 (JetBrains Junie), CVE-2025-55012 (Zed.dev)*

2. **IDE Settings Overwrite**: Agent overwrites `.vscode/settings.json` or `.idea/workspace.xml` with attacker-controlled values — custom task runners or terminal profiles — that execute arbitrary commands on the next workspace open.
   - *CVE-2025-49150, CVE-2025-54130 (Cursor), CVE-2025-53097, CVE-2025-58372 (Roo Code)*

3. **Multi-root Workspace Hijacking**: `.code-workspace` files reference attacker-controlled folders, silently overriding trusted project settings.

**Status: Most CVEs patched in vendor releases (Cursor 1.7, JetBrains Junie December 2025). Windsurf patches remain incomplete. Gemini CLI unresolved as of April 10, 2026.**

### Claude Code — Hook Injection RCE and API Key Exfiltration

Multiple CVEs in Anthropic's Claude Code CLI disclosed between mid-2025 and early 2026 share a root cause: malicious repository configuration files are processed by Claude Code on launch, before any user trust prompt.

| CVE | Type | Fixed Version | Disclosed |
|-----|------|--------------|-----------|
| CVE-2025-59536 (CVSS 8.7) | RCE via hooks in `.claude/settings.json` | 2.0.65 | Feb 2026 |
| CVE-2026-21852 | API key exfiltration via `ANTHROPIC_BASE_URL` redirect | 2.0.65 | Feb 2026 |
| CVE-2026-35020 / 35021 / 35022 | Command injection → credential exfiltration over HTTP (chained) | Patched | 2026 |
| CVE-2025-54794 / 54795 ("InversePrompt") | Prompt inversion techniques turning Claude against itself | Coordinated disclosure | Oct 2025 |

The Check Point Research disclosure (February 2026) demonstrated the full chain: a malicious `.claude/settings.json` embedded in a repository triggers arbitrary shell commands via the hooks mechanism the moment the developer opens the project directory. CVE-2025-59536 carries CVSS 8.7.

**Status: CVE-2025-59536 and CVE-2026-21852 patched in v2.0.65. Three additional command injection CVEs (CVE-2026-35020/21/22) patched. CVE-2025-54794/54795 in coordinated disclosure. Upgrade to latest Claude Code release. Treat repository cloning as equivalent to running untrusted code.**

### Cursor — Electron/Chromium Debt Under Active Exploitation (94+ CVEs)

OX Security research found that Cursor and Windsurf — both built on VS Code Electron forks last updated to Chromium 132 in March 2025 — carry **94+ unpatched inherited CVEs** from outdated browser engine versions. Of these:

- **CVE-2025-5419** (V8 OOB read/write) — CISA KEV; exploited in the wild
- **CVE-2025-6554** (V8 type confusion) — CISA KEV; exploited in the wild
- **CVE-2025-6558** (GPU input validation) — CISA KEV; exploited in the wild
- **CVE-2025-4609** (sandbox escape) — Exploited; earned a $250,000 Google VRP bounty

**Estimated exposure: 1.8 million developers.** Cursor and Windsurf have not updated their Chromium/Electron base since the affected versions.

Additional Cursor-specific CVEs:

- **CVE-2025-59944** (CVSS 8.0) — Case-sensitivity bypass on Windows/macOS: attacker creates `.cUrSoR/mcp.json` (mixed case), which the OS resolves to `.cursor/mcp.json`, bypassing file protection and enabling prompt injection → RCE. Fixed in Cursor 1.7+.
- **CVE-2025-54135** ("CurXecute", CVSS 8.6) — Prompt injection via malicious Slack message processed through Slack MCP Server. Cursor rewrites MCP config and executes arbitrary commands before the user can reject edits. Fixed in Cursor 1.3.
- **CVE-2025-54136** ("MCPoison", CVSS 7.2) — Once a malicious MCP server is approved, later changes to its config are auto-trusted (bound by name, not content), enabling silent execution of injected commands without re-approval. Fixed in Cursor 1.3.

**Status: Cursor-specific CVEs patched in v1.3 and v1.7. Inherited Chromium vulnerabilities remain unresolved; three are actively exploited (KEV). Windsurf shares the same Chromium exposure with no announced remediation timeline.**

### GitHub Copilot — CamoLeak Zero-Click Exfiltration (CVE-2025-59145, CVSS 9.6)

Malicious instructions hidden in pull request descriptions exploited GitHub's image proxy infrastructure as a CSP bypass. Copilot Chat silently exfiltrated source code, API keys, and secrets from private repositories — no user interaction required. The AI acted as its own exfiltration channel.

**Status: Patched August 14, 2025 (GitHub disabled image rendering in Copilot Chat). Self-hosted or API-integrated Copilot deployments should verify the patch is applied.**

Related CVEs:
- **CVE-2025-53773** (CVSS 7.8) — Prompt injection via source file comments causes Copilot to execute attacker-controlled shell commands locally via flags like `find -exec` or `go test -exec`. Patched in Microsoft's August 2025 Patch Tuesday.
- **CVE-2025-64660** — Code execution via network through improper access control in the VS Code extension. Patched November 2025.

---

### Amazon Q Developer — Supply Chain Compromise (CVE-2025-8217)

An attacker from the "CanisterWorm" campaign compromised the official Amazon Q Developer VS Code Extension GitHub repository via an improperly scoped CI/CD token, injecting malicious code into version 1.84.0. The payload directed the extension to wipe local files and disrupt AWS infrastructure. The extension passed Amazon's verification and was publicly available for two days. Execution failed only due to a syntax error in the payload.

**Status: Patched in version 1.85.0. Illustrates that vendor-signed extensions are not immune to supply chain compromise.**

## Supply Chain

### Slopsquatting — AI-Hallucinated Dependencies as Reliable Attack Vector

A landmark study analyzing **576,000 AI-generated Python and JavaScript code samples** across 16 LLMs found that approximately **20% of recommended packages did not exist**:

- Open-source models (CodeLlama, DeepSeek, WizardCoder, Mistral): ~21.7% hallucination rate
- Commercial models (GPT-4, GPT-4 Turbo): ~5.2% hallucination rate
- **43% of hallucinated package names reappeared consistently across 10 attempts** with the same prompt — confirming the attack is statistically reliable

Real-world validation: researcher Bar Lanyado (Lasso Security) registered `huggingface-cli` — a name AI models repeatedly hallucinate — as an empty PyPI package. It received **30,000+ genuine downloads in three months**. A separate hallucinated package, `react-codeshift`, spread autonomously across **237 GitHub repositories** via AI-generated agent skills.

The escalating risk in 2026: autonomous coding agents can `pip install` hallucinated packages in CI/CD pipelines without human review, at machine speed, with no hash verification unless the environment is explicitly hardened.

**No systematic fix available. Mitigations: `--require-hashes` enforcement, private registries, lockfile pinning, and LLM output validation before package installation.**

### Malicious VS Code AI Extensions — 1.5 Million Combined Installs ("MaliciousCorgi")

Two malicious VS Code extensions with **1.5 million combined installs** were found exfiltrating developer source code to China-based servers. The extensions monitored every opened file, Base64-encoded the content, and transmitted it to attacker servers. A second mechanism transmitted up to 50 workspace files per session.

**Status: Extensions removed. Illustrates that install count is not a reliable indicator of safety for AI coding extensions.**

---

### VSCode Fork Supply Chain — Unclaimed OpenVSX Namespaces ("GlassWorm")

Koi Security found that several VS Code forks (Cursor, Windsurf, Trae, Google Antigravity) hardcoded "recommended extension" lists pointing to **unclaimed namespaces** in the OpenVSX extension registry. Attackers registered 72 of these namespaces and published malicious extensions that were auto-recommended to millions of developers. Payloads landed across 400+ GitHub repositories.

- Cursor patched December 2025; Google patched January 2026.
- **Windsurf: no vendor response to disclosure as of April 10, 2026. Users remain at risk.**

---

### OpenClaw / ClawHub — AI Agent Skill Marketplace Compromise

OpenClaw (an open-source AI agent framework, 135,000+ GitHub stars) suffered what Antiy CERT described as "the first major AI agent security crisis of 2026." Its ClawHub skill marketplace was found to contain **1,184 confirmed malicious packages** — approximately 1 in 9 total packages (21,000+ exposed instances confirmed).

Attack method: fake skills impersonating legitimate tools prompt users to paste terminal commands or download files. One skill masquerading as a Polymarket integration opened a reverse shell. Payloads included Atomic macOS Stealer, credential harvesters, and remote access tools.

**Status: Ongoing remediation. Treat all ClawHub skills as untrusted until the platform's security model is revised.**

## Research & Advisories

### MCP Ecosystem — Structural Security Deficits

Large-scale audits of publicly exposed MCP servers reveal systemic weaknesses across the ecosystem:

- **43%** have OAuth authentication flaws
- **43%** contain command injection vulnerabilities
- **33%** allow unrestricted network access
- **22%** allow access to files outside intended data sources

The [Vulnerable MCP Project](https://vulnerablemcp.info/) tracks 50+ disclosed MCP CVEs. Additional CVEs beyond those in the Incidents section:

| CVE | Component | Type | Status |
|-----|-----------|------|--------|
| CVE-2026-25536 | MCP TypeScript SDK 1.10.0–1.25.3 | Cross-client data leak | Patched |
| CVE-2026-23744 | MCPJam Inspector ≤1.4.2 | Critical RCE | Patched |
| CVE-2026-0756 | GitHub Kanban MCP Server | Critical RCE | Patched |
| CVE-2026-0755 | gemini-mcp-tool | Command injection | Unpatched at disclosure |
| CVE-2025-66414 / 66416 | Official MCP SDKs | DNS rebinding | Patched |
| CVE-2025-67366 | filesystem-mcp | Path traversal | Patched |
| CVE-2025-53372 | node-code-sandbox-mcp | Docker sandbox escape | Patched |

**MCP Rug-Pull Design Gap (No CVE):** The MCP protocol has no mechanism to re-prompt user consent when a server silently updates its tool definitions. A tool approved as a read-only browser on Day 1 can include exfiltration capabilities by Day 7 with no client-side warning. No protocol-level fix exists in the current MCP spec. **Mitigation: pin MCP server versions by commit hash; treat every server update as a new approval decision.**

### Tool Poisoning — MCP Tool Description Injection

Invariant Labs formally documented the "tool poisoning" attack class: malicious instructions embedded in MCP tool *descriptions* — invisible to users in the UI but read and acted upon by the AI model — instruct the agent to exfiltrate data or execute commands while appearing to perform legitimate operations. Demonstrated against WhatsApp history exfiltration.

The MCPTox benchmark tested 20 AI agents against real-world poisoning across 45 MCP servers: **o1-mini was successfully attacked 72.8% of the time**; Claude 3.7-Sonnet refused fewer than 3% of attacks.

This is a design-level risk inherent to MCP architecture. No CVE is assignable; the vulnerability is structural.

---

### LSP Server Fuzzing — 51 Previously Unknown Bugs

Systematic fuzzing research across Language Server Protocol (LSP) implementations (clangd, sorbet, verible, solc) revealed **51 previously undisclosed bugs**: 42 confirmed, 26 fixed, 2 CVEs issued. The dominant bug class arose from coupled code/editor-operation inputs rather than isolated random inputs — meaning standard unit testing largely misses these defects.

This research suggests LSP servers represent an underexamined attack surface in AI-assisted development toolchains.

- 53% of AI-generated code ships with at least one vulnerability
- Privilege escalation paths in AI-generated code rose 322%; architectural design flaws rose 153% in 2025–2026

## Key Mitigations

1. **Treat repository cloning as code execution** — `.claude/settings.json`, `.cursor/mcp.json`, `.vscode/settings.json`, and `.code-workspace` files are executed by AI coding agents on startup. Open untrusted repositories only in isolated environments.
2. **Update Cursor to 1.7+** — Fixes CurXecute (CVE-2025-54135), MCPoison (CVE-2025-54136), and the case-sensitivity bypass (CVE-2025-59944). The inherited Chromium CVEs remain unresolved across all VS Code forks.
3. **Pin MCP server versions by commit hash** — The MCP protocol has no re-consent mechanism for tool definition changes. Treat every MCP server update as a new approval decision.
4. **Enforce `--require-hashes` in agent-accessible pip environments** — Prevents slopsquatting payloads from installing even when a hallucinated package name is registered in PyPI.
5. **Vet MCP servers before enabling** — 43% of audited public MCP servers contain command injection vulnerabilities; 43% have OAuth flaws. Treat MCP servers as first-class dependencies requiring security review.
6. **Windsurf users: audit extensions and restrict MCP surface** — No vendor response to GlassWorm disclosure; 94+ inherited Chromium CVEs including three on CISA's KEV list. Treat extension provenance as untrusted.
7. **Audit CI/CD pipeline token scopes** — The Amazon Q Developer compromise succeeded via an improperly scoped CI token. Apply least privilege to all CI/CD secrets; rotate any token that transited a compromised runner.
8. **Review AI-generated code for OWASP Top 10 patterns** — 45–53% of AI-generated code samples contain at least one security flaw; secret leak rates in AI-assisted commits are double the baseline.
9. **Upgrade mcp-remote to 0.1.16+** — CVE-2025-6514 (CVSS 9.6) enables OS command injection from a malicious MCP server response.
10. **Upgrade mcp-server-git to 2025.12.18+** — Three chained CVEs (CVE-2025-68143/44/45) enable full RCE via SSH config overwrite. The `git_init` tool was removed entirely.
