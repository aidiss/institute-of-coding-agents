## Incidents & Vulnerabilities

### IDEsaster — Full CVE Inventory (Disclosed December 2025)

Researcher Ari Marzouk (MaccariTA) concluded a coordinated disclosure covering **30+ vulnerabilities, 24 CVEs, and 10+ affected products**. AWS issued a companion advisory (AWS-2025-019). Every single tested AI IDE was found vulnerable to at least one critical chain.

The universal attack path: **Prompt Injection → AI Tools → Base IDE Features**.  A malicious rule file, MCP server, deeplink, or filename hijacks the agent's context, which then overwrites trusted IDE configuration files to achieve RCE or data theft — abusing mechanisms that predate AI by decades.

### GitHub Copilot — RCE & Data Theft (CVSS 9.6)

| CVE | CVSS | Status | Description |
|-----|------|--------|-------------|
| CVE-2025-53773 | 9.6 | **Patched** | Prompt injection → overwrite `.vscode/settings.json` → YOLO mode (auto-approve terminal) → RCE; no user interaction |
| CVE-2025-64660 | — | **Patched** | Multi-root workspace config abuse enabling RCE without reopening the workspace |

**Rules File Backdoor** (Pillar Security, March 2025): Zero-width Unicode characters hidden in `.github/copilot-instructions.md` are invisible to human reviewers but followed by the model, enabling silent steering of AI-generated code. GitHub added a UI warning for hidden Unicode on github.com.

**RoguePilot** (Orca Security): Documented privilege escalation via environment self-modification — AI agent modifies its own configuration to gain elevated tool permissions.

### Windsurf & Cursor — 93+ Unpatched Chromium CVEs (OX Security, October 2025)

Both AI IDEs ship outdated Electron/Chromium builds. OX Security weaponized **CVE-2025-7656** (a patched Chromium bug) against then-current releases and found **93 additional unpatched Chromium CVEs** in the bundled engines. Estimated exposure: **1.8 million developers**. Windsurf had not responded to the disclosure as of the report date.

Separately, a **Windsurf indirect prompt injection** (Embrace The Red, May 2025) demonstrated exfiltration of source code, environment variables, and secrets without user approval when analyzing a malicious file via Cascade. No fix ETA was provided at disclosure.

Sources: [OX Security](https://www.ox.security/blog/94-vulnerabilities-in-cursor-and-windsurf-put-1-8m-developers-at-risk/) · [Embrace The Red](https://embracethered.com/blog/posts/2025/windsurf-data-exfiltration-vulnerabilities/)

### Devin AI — Full Agent Compromise via Prompt Injection

Researcher Johann Rehberger demonstrated complete agent compromise: crafted prompts caused Devin to expose ports to the internet, leak access tokens, and install C2 malware. No CVE assigned. Demonstrates that autonomous agents with broad tool access multiply the impact of prompt injection to full system compromise.

## Supply Chain

### s1ngularity — AI CLI Tools Used as Active Malware Components (August 26, 2025)

The `nx` build system npm package was backdoored with data-stealing malware for approximately five hours. The malware **weaponized AI CLI tools as exfiltration agents**: Claude, Gemini, and Amazon Q (`q`) binaries were invoked to read and transmit stolen data. This is the **first documented supply chain attack using AI developer assistants as active attack tools**.

Stolen data included: 1,000+ GitHub tokens, valid cloud credentials, npm tokens, and ~20,000 files.

Source: [StepSecurity](https://www.stepsecurity.io/blog/supply-chain-security-alert-popular-nx-build-system-package-compromised-with-data-stealing-malware) · [InfoQ](https://www.infoq.com/news/2025/10/npm-s1ngularity-shai-hulud/)

### Shai-Hulud — AI-Generated Malware Payload, Self-Propagating (September 15, 2025)

40+ npm packages (including `@ctrl/tinycolor`) were trojanized. Unit 42 assessed with moderate confidence that the **malicious payload was AI-generated** (inferred from characteristic comment patterns and emoji use). Novel behavior: upon finding npm tokens, the malware auto-published malicious versions of every package accessible by that token.

Source: [Unit 42 / Palo Alto](https://unit42.paloaltonetworks.com/model-context-protocol-attack-vectors/) · [InfoQ](https://www.infoq.com/news/2025/10/npm-s1ngularity-shai-hulud/)

### Slopsquatting — AI Package Hallucination as Attack Vector

Research paper (576,000 generated code samples, 16 LLMs) quantified the attack surface:

| Metric | Finding |
|--------|---------|
| Hallucinated package rate (all models) | ~20% of generated samples |
| Commercial models (GPT-4 class) | ~5% hallucination rate |
| Open-source models (CodeLlama, DeepSeek, Mistral) | ~21.7% hallucination rate |
| Unique fake package names generated | 200,000+ |
| Hallucinated names that repeat across 10 runs | 43% — deterministic, squattable |
| Hallucinated names within 1–2 chars of real packages | 13.4% |
| LLM-assisted dep upgrade recommendations for non-existent versions | 28% (Sonatype 2026) |

The attack is passive: register commonly hallucinated names on PyPI/npm and wait for AI-assisted developers to install them. The `huggingface-cli` proof-of-concept was adopted by thousands of developers including Alibaba teams within days of registration.

Sources: [BleepingComputer](https://www.bleepingcomputer.com/news/security/ai-hallucinated-code-dependencies-become-new-supply-chain-risk/) · [Socket.dev](https://socket.dev/blog/slopsquatting-how-ai-hallucinations-are-fueling-a-new-class-of-supply-chain-attacks) · [Trend Micro](https://www.trendmicro.com/vinfo/us/security/news/cybercrime-and-digital-threats/slopsquatting-when-ai-agents-hallucinate-malicious-packages)

## Research & Advisories

### MCP (Model Context Protocol) — Expanding Attack Surface

Key vulnerabilities in the MCP ecosystem:

| CVE / Incident | CVSS | Status | Description |
|---|---|---|---|
| CVE-2025-6514 (mcp-remote) | 9.6 | **Patched** | OS command injection when MCP clients connect to untrusted servers; first confirmed real-world MCP RCE (JFrog) |
| MCP Inspector RCE | — | **Patched** | Anthropic's own MCP Inspector allowed unauthenticated RCE via inspector-proxy; attacker triggers by having a victim inspect a malicious server |
| Tool Poisoning (Invariant Labs) | — | Active risk | Malicious MCP server silently exfiltrated WhatsApp history by combining poisoned + legitimate tools in same agent context |
| Rug Pull / Silent Redefinition | — | Active risk | MCP tools can redefine themselves post-installation; an approved tool on Day 1 reroutes API keys by Day 7 |
| Session ID Hijacking | — | Design flaw | MCP mandates session IDs in URLs (e.g., `GET /messages/?sessionId=UUID`), exposing them in logs |
| Smithery path traversal | — | Patched | `dockerBuildPath: ".."` in smithery.yaml caused builder to read `~/.docker/config.json` and a Fly.io token controlling 3,000+ apps |
| Unit 42 MCP Sampling Attack | — | Active risk | Malicious server issues covert LLM sampling requests through the client using a legitimate-looking tool as cover |

Additional statistics: ~7,000 MCP servers exposed publicly; ~half have no authentication or encryption; 36.7% vulnerable to SSRF.

Sources: [JFrog CVE-2025-6514](https://www.cvedetails.com/cve/CVE-2025-6514/) · [AuthZed Timeline](https://authzed.com/blog/timeline-mcp-breaches) · [Unit 42](https://unit42.paloaltonetworks.com/model-context-protocol-attack-vectors/) · [Red Hat](https://www.redhat.com/en/blog/model-context-protocol-mcp-understanding-security-risks-and-controls)

### Academic Research

| Paper | Key Finding |
|-------|-------------|
| "Your AI, My Shell" (arXiv:2509.22040, Sep 2025) | First large-scale empirical study of project injection in agentic editors; attack success rates **41–84%** across Cursor and GitHub Copilot; rule files are the highest-impact vector |
| "Library Hallucinations in LLMs" (arXiv:2509.22202, Sep 2025) | Requesting a "2025" library → hallucinations in up to 84% of tasks; one-character misspellings → 26% hallucination; adversarial prompts → fake names used in up to 99% of tasks |
| "Prompt Injection in LLMs" (MDPI, Jan 2026) | OWASP ranks prompt injection #1 in LLM Top 10; present in >73% of audited production deployments |
| Trail of Bits (Oct 2025) | Prompt injection → RCE in three production agentic systems; argument injection in CLI agents via unvalidated flag parameters |

### Defensive Recommendations

| Layer | Recommendation |
|-------|---------------|
| Agent config | Audit `.claude/settings.json`, `.cursor/mcp.json`, `.vscode/settings.json` for unexpected hooks or MCP entries before opening untrusted repos |
| Dependencies | Verify all AI-recommended packages exist in the registry before installing; review lock files; sign SBOMs |
| MCP servers | Only connect to authenticated, encrypted servers; treat tool definitions as mutable post-approval |
| IDE settings | Enable human-in-the-loop approval for all file writes; do not auto-approve terminal commands |
| CI/CD | Run automated dep-scan / Safety CLI in pipeline; isolate AI-generated installs in ephemeral containers |
| Rule/instruction files | Scan `.cursorrules`, `.github/copilot-instructions.md` for hidden Unicode (zero-width joiners, BiDi marks) |
| Architecture | Sandbox agent operations; apply least-privilege to all LLM tool calls; assume prompt injection is always possible at any trust boundary |
