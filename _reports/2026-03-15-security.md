## Incidents & Vulnerabilities

### OpenClaw — "ClawJacked" & Multi-CVE Crisis (February–March 2026)

OpenClaw is an open-source AI agent (~135,000 GitHub stars) that connects to messaging platforms (WhatsApp, Telegram, Slack, Discord) and acts autonomously on the user's behalf — executing terminal commands, managing files, browsing the web, and sending messages. It is extensible via "skills" published to its **ClawHub** community marketplace.

On February 15, OpenClaw's creator Peter Steinberger joined OpenAI. That same week, coordinated security disclosures revealed systemic vulnerabilities across the platform.

#### CVE-2026-25253 — "ClawJacked": Full Gateway Takeover via WebSocket (CVSS 8.8)

| Field | Detail |
|-------|--------|
| CVE | CVE-2026-25253 |
| CVSS | 8.8 |
| Status | **Patched** in v2026.2.26 |
| User interaction | Victim visits one malicious webpage |

OpenClaw assumes localhost-originating connections are implicitly trusted without accounting for browser-initiated WebSocket connections. A malicious webpage can silently:
1. Open a WebSocket to the local OpenClaw gateway
2. Brute-force the gateway password at hundreds of attempts/second (rate limiter exempts localhost entirely)
3. Auto-register as a trusted device (no user prompt for localhost pairings)
4. Gain full administrative control — disable confirmation prompts, escape Docker sandbox, execute arbitrary host commands

The exploit chain takes milliseconds. SecurityScorecard's STRIKE team found **135,000+ publicly exposed OpenClaw instances** across 82 countries; more than **15,000 are directly vulnerable to RCE**.

### Cursor — Shell Built-in Bypass RCE (CVE-2026-22708)

| CVE | CVSS | Status | Affected Versions |
|-----|------|--------|-------------------|
| CVE-2026-22708 | High | **Patched** in v2.3 | Cursor < v2.3 |

When Cursor Agent runs in Auto-Run Mode with Allowlist mode enabled, shell built-ins (`export`, `unset`, `set`, `typeset`, etc.) bypass the allowlist entirely — they are not external binaries and receive no validation or user notification, even on an empty allowlist.

**Attack path:** Indirect prompt injection (e.g., via a malicious file or repository) instructs the agent to use `export PATH=/attacker/bin:$PATH`. Subsequent execution of trusted commands (`git`, `npm`, `python`) resolves to attacker-controlled binaries. Both zero-click and one-click exploitation chains are demonstrated.

Cursor now requires explicit approval for any commands the server-side parser cannot classify. Allowlists are no longer presented as a security boundary.

Sources: [SentinelOne](https://www.sentinelone.com/vulnerability-database/cve-2026-22708/) · [SC Media](https://www.scworld.com/news/cursor-vulnerability-enables-stealthy-rce-via-indirect-prompt-injection) · [DEV Community writeup](https://dev.to/cverports/cve-2026-22708-trust-issues-bypassing-cursor-ais-safe-mode-via-shell-built-ins-55ao) · [Pillar Security](https://www.pillar.security/blog/new-vulnerability-in-github-copilot-and-cursor-how-hackers-can-weaponize-code-agents)

### MCPJam Inspector — Zero-Interaction RCE (CVE-2026-23744)

| CVE | CVSS | Status | Affected Versions |
|-----|------|--------|-------------------|
| CVE-2026-23744 | **9.8 Critical** | **Patched** in v1.4.3 | MCPJam Inspector ≤ 1.4.2 |

MCPJam Inspector (a widely used tool for testing and debugging MCP servers) listens on `0.0.0.0` by default with no authentication on its critical endpoint. A crafted HTTP request can install an arbitrary MCP server and trigger remote code execution on the host — **no user interaction required**. This is an escalation of CVE-2025-49596 (which required a click); CVE-2026-23744 requires nothing from the victim.

The target is the developer building MCP infrastructure, not a production server, meaning blast radius includes proprietary codebases, stored credentials, and signing keys.

Sources: [GitLab Advisory](https://advisories.gitlab.com/pkg/npm/@mcpjam/inspector/CVE-2026-23744/) · [NVD](https://nvd.nist.gov/vuln/detail/CVE-2026-23744) · [Vulnerable MCP Project](https://vulnerablemcp.info/)

### Anthropic Git MCP Server — Three-CVE RCE Chain (CVE-2025-68145 / 68143 / 68144)

Three chained vulnerabilities in Anthropic's own `mcp-server-git` package, all patched:

| CVE | Description |
|-----|-------------|
| CVE-2025-68145 | Path validation bypass — allows traversal outside repo root |
| CVE-2025-68143 | Unrestricted `git_init` call can re-initialize arbitrary directories (including `~/.ssh`) as git repos |
| CVE-2025-68144 | Argument injection in `git_diff` command handler |

Chaining all three with the Filesystem MCP server achieves **full RCE via a malicious `.git/config` file** placed in a repository. A developer cloning or opening an attacker-controlled repository is sufficient to trigger exploitation.

Sources: [Vulnerable MCP Project](https://vulnerablemcp.info/) · [Practical DevSecOps](https://www.practical-devsecops.com/mcp-security-vulnerabilities/)

The OpenClaw skill ecosystem (ClawHub) has become the most actively abused AI agent package registry to date.

| Metric | Finding |

| Total ClawHub skills | ~10,700 |
| Malicious skills identified (mid-March 2026) | **820+** (up from 324 in early February) |

### MCP TypeScript SDK — Cross-Client Data Leak (CVE-2026-25536)

| CVE | CVSS | Status | Affected Versions |
|-----|------|--------|-------------------|
| CVE-2026-25536 | 7.1 | **Patched** in v1.26.0 | `@modelcontextprotocol/sdk` v1.10.0–1.25.3 |

When a single `McpServer` instance with `StreamableHTTPServerTransport` is reused across multiple clients, responses silently misroute — one authenticated client receives data intended for another. Authentication does not prevent exploitation; the leak happens within already-authenticated sessions.

Multi-client stateless deployments running the affected SDK versions are actively leaking data between clients right now to any connecting user.

**Fix:** Upgrade to `@modelcontextprotocol/sdk` v1.26.0. **Interim workaround:** Create fresh `McpServer` and transport instances per request (stateless) or per session (stateful).

Sources: [Vulnerable MCP Project](https://vulnerablemcp.info/vuln/cve-2026-25536-sdk-cross-client-data-leak.html) · [NVD](https://nvd.nist.gov/vuln/detail/CVE-2026-25536) · [DEV Community analysis](https://dev.to/kai_security_ai/the-mcp-sdk-is-now-its-own-attack-surface-2o4)

## Research & Advisories

### arXiv: "Taming OpenClaw" — Empirical Security Audit (March 2026)

A large-scale empirical security audit of OpenClaw's architecture found that **~26% of community-contributed tools contain security vulnerabilities**. The initialization phase is flagged as most susceptible to supply-chain and configuration attacks. The paper synthesizes findings across all five agent lifecycle stages and proposes a threat model for autonomous local AI agents.

Source: [arXiv:2603.11619](https://arxiv.org/html/2603.11619v1)

---

### MDPI: Prompt Injection in LLM Systems — Systematic Review (January 2026)

A meta-analysis synthesizing 78 studies (2021–2026) on prompt injection finds:

- Prompt injection ranked **#1 in OWASP LLM Top 10**; present in >73% of audited production deployments
- Attack success rates against state-of-the-art defenses **exceed 85%** when adaptive strategies are employed
- No complete fix exists — even frontier models (OpenAI, Google, Anthropic) remain vulnerable after applying best defenses
- OpenAI has publicly acknowledged prompt injection "may never be fully solved" at the model level

Source: [MDPI Information (Jan 2026)](https://www.mdpi.com/2078-2489/17/1/54)

### Unit 42: New Prompt Injection Attack Vectors via MCP Sampling

Palo Alto Unit 42 documents a novel attack vector: a malicious MCP server issues covert LLM sampling requests through the client, piggybacking on a legitimate-looking tool as cover. The client model completes attacker-authored instructions believing them to be part of legitimate tool output. This bypasses prompt-level content filtering and user-review workflows.

Source: [Unit 42](https://unit42.paloaltonetworks.com/model-context-protocol-attack-vectors/)

### Defensive Recommendations

| Layer | Action |
|-------|--------|
| OpenClaw | Upgrade to v2026.2.26+; rotate all stored credentials in `~/.clawdbot/.env` and `~/.openclaw/credentials/`; never run on corporate workstations without network isolation |
| Cursor | Upgrade to v2.3+; treat allowlist mode as a UX feature, not a security boundary; sandbox agent execution |
| MCPJam Inspector | Upgrade to v1.4.3+; bind to `127.0.0.1` explicitly; add authentication to the inspector endpoint |
| MCP SDK | Upgrade `@modelcontextprotocol/sdk` to v1.26.0; create per-request server instances in stateless deployments |
| MCP servers | Audit all server implementations against the Vulnerable MCP Project database; require auth + TLS on all endpoints; treat tool definitions as mutable post-approval |
| Agent marketplaces | Validate all skills/plugins against a vetted registry; scan for known-malicious actors; treat marketplace installs as untrusted code |
| Dependencies | Verify all AI-recommended packages exist before installing; use private registry proxies with age and reputation filters |
| Architecture | Apply least-privilege to all agent tool grants; assume prompt injection is always possible at any trust boundary; sandbox file-write operations to workspace |
