## Headlines

- **IDEsaster (Dec 2025):** Researcher Ari Marzouk disclosed 30+ vulnerabilities across 10+ AI IDEs — 24 assigned CVEs — confirming every tested product is exploitable via a three-stage chain: prompt injection → tool abuse → base IDE features. GitHub Copilot, Cursor, Claude Code, Windsurf, Gemini CLI, and others all affected.
- **MCP becomes the fastest-growing attack surface in AI infrastructure:** 30 CVEs in 60 days; a CVSS 10.0 RCE in `mcp-remote` (CVE-2025-6514); Anthropic's own MCP Inspector found to allow unauthenticated RCE; tool poisoning, rug-pull redefinition, and cross-server hijacking documented in production.
- **Supply chain:** AI-hallucinated "slopsquatting" packages exploited at scale in npm (s1ngularity, Shai-Hulud); Shai-Hulud payload confirmed AI-generated by Unit 42; 454,648 malicious packages published in 2025; 28% of LLM dependency upgrade suggestions in 2026 are hallucinations.

## Incidents & Vulnerabilities
_As of 2026-03-07_

### IDEsaster — Systemic Vulnerabilities Across All AI IDEs

**Researcher:** Ari Marzouk (MaccariTA) | **Disclosed:** December 2025 | **CVEs assigned:** 24 | **Status:** Partially patched (vendor-specific)

A six-month research effort produced the largest coordinated AI IDE vulnerability disclosure to date, affecting every major AI coding product. The attack model is a three-stage chain that exploits **base IDE features**, not just AI-specific code:

> **Prompt Injection → Agent Tools → Base IDE Features → RCE / Data Exfiltration**

Malicious payloads can enter via code comments, rule files (`.cursorrules`, `.clinerules`), MCP server tool descriptions, repository filenames, or log output.

### MCP (Model Context Protocol) Vulnerabilities

The MCP ecosystem — designed for functionality over security — has become one of the fastest-growing attack surfaces in AI infrastructure.

| CVE | CVSS | Description | Status |
|-----|------|-------------|--------|
| CVE-2025-6514 | **10.0** | OS command injection in `mcp-remote` (OAuth proxy for local MCP clients). Arbitrary OS command execution when clients connect to untrusted servers. First documented full RCE in production MCP. Disclosed by JFrog. | Patch available |

**Anthropic MCP Inspector RCE (no CVE assigned as of report date):**
Unauthenticated RCE in Anthropic's own MCP Inspector developer tool via its inspector-proxy architecture. Inspector runs with user privileges, lacks authentication, and listens on localhost/0.0.0.0 — exploitation requires only that the developer inspect a malicious MCP server. Full filesystem, API key, and environment secret exposure. *Status: patched in updated releases.*

**Ecosystem-wide statistics (as of early 2026):**
- 30 CVEs filed against MCP components in 60 days
- 38% of 500+ scanned MCP servers completely lack authentication
- 36.7% of 7,000+ scanned MCP servers vulnerable to SSRF

**Key attack patterns:**
- **Tool Poisoning:** Malicious tool definitions pass seamlessly into agent contexts; MCPTox benchmark documents high pass-through rates leading to unauthorized execution or data leakage.
- **Rug Pull / Silent Redefinition:** MCP tools mutate their own definitions after installation approval. Safe-looking tool on Day 1 silently reroutes API keys to attacker by Day 7.
- **Cross-Server Hijacking:** Malicious server in a multi-server setup overrides or intercepts calls to trusted servers.
- **Prompt Hijacking via Exposed Session IDs:** The MCP spec mandates session identifiers in URLs (`GET /messages/?sessionId=UUID`), violating security best practices; session IDs appear in logs and are guessable.
- **Smithery Path Traversal (Real-World):** `dockerBuildPath: ".."` in `smithery.yaml` leaked builder's `~/.docker/config.json` including a Fly.io API token granting control over 3,000+ apps.
- **WhatsApp MCP Exfiltration (Invariant Labs):** Malicious MCP server combined with legitimate `whatsapp-mcp` silently exfiltrated a user's entire WhatsApp history via tool poisoning.
- **Supabase/Cursor Agent Breach (mid-2025):** Privileged service-role Cursor agent processed support tickets containing user-supplied SQL instructions, exfiltrating sensitive integration tokens into a public support thread. Root cause: privileged access + untrusted input + external communication channel.

*Sources: [JFrog CVE-2025-6514](https://www.esentire.com/blog/model-context-protocol-security-critical-vulnerabilities-every-ciso-should-address-in-2025) | [Unit 42 MCP Sampling](https://unit42.paloaltonetworks.com/model-context-protocol-attack-vectors/) | [Pomerium June 2025 Round-Up](https://www.pomerium.com/blog/june-2025-mcp-content-round-up) | [authzed MCP Timeline](https://authzed.com/blog/timeline-mcp-breaches)*

### Devin AI — Total Compromise via Prompt Injection

Researcher spent $500 testing Devin AI's autonomous coding agent and found it completely defenseless. Via prompt injection, an attacker could:
- Expose ports to the public internet
- Leak access tokens
- Install command-and-control malware

All through crafted prompts with no other access required. No CVE assigned. Current patch status unknown.

## Supply Chain
_As of 2026-03-07_

### Slopsquatting — AI-Hallucinated Package Names as Attack Vector

Coined by researcher **Seth Larson**, slopsquatting weaponizes the tendency of LLMs to hallucinate plausible but non-existent package names. An attacker registers the hallucinated name in PyPI or npm with a malicious payload; subsequent AI-assisted installations pull and execute it.

**Scale:**
- ~20% of generated Python/JavaScript code samples recommend non-existent packages (research, March 2025; 576,000 samples)
- Open-source LLMs (CodeLlama, DeepSeek, WizardCoder, Mistral): ~21.7% average hallucination rate
- Commercial models (ChatGPT-4): ~5.2% average — still significant at scale
- 43% of hallucinated package names repeat across 10 queries; 58% repeat more than once — enabling reliable targeting
- 28% of LLM-assisted dependency upgrade suggestions in 2026 are hallucinations (Sonatype 2026 report)
- 200,000+ unique hallucinated package names documented in one study alone

### npm Supply Chain Attacks — s1ngularity & Shai-Hulud (2025)

| Attack | Date | Affected Packages | Payload | AI Involvement |
|--------|------|-------------------|---------|----------------|
| **s1ngularity** | Aug 26, 2025 | Multiple versions of **Nx build system** | Collects crypto wallets, GitHub/npm tokens, SSH keys | Not confirmed |
| **Shai-Hulud** | Sep 2025 | 40+ npm packages incl. `@ctrl/tinycolor` | `bundle.js` injects TruffleHog to hunt tokens and cloud credentials | Unit 42: "moderately confident attacker used AI to generate malicious script — evidenced by comments and emojis" |

**2025 ecosystem statistics (Sonatype):**
- 99%+ of all open-source malware occurred on npm
- 454,648 new malicious packages published in 2025 across major registries
- 9.8 trillion downloads processed across Maven Central, PyPI, npm, and NuGet

**2026 outlook:** Attackers expected to expand beyond npm to Maven, RubyGems, NuGet, and container/IaC registries. "Supply chain worms" — self-propagating attack patterns using compromised packages to poison downstream toolchains — are the projected next evolution (Dark Reading, 2026).

## Research & Advisories
_As of 2026-03-07_

### OWASP LLM Top 10 (2025 Edition)
- **#1: Prompt Injection** — present in 73%+ of production AI deployments assessed in security audits.
- Attack success rates against state-of-the-art defenses exceed **85%** when adaptive strategies are used (meta-analysis of 78 studies, 2021–2026).
- InjecAgent benchmark: ReAct-prompted GPT-4 vulnerable to injections **24% of the time**; near-doubles with enhanced adversarial prompts.

### NIST — AI Agent Hijacking Evaluations (Jan 2025)
NIST CAISI augmented the AgentDojo benchmark with new injection tasks to evaluate: remote code execution, database exfiltration, and automated phishing. Key finding: indirect prompt injection (attacker inserts instructions into data ingested by the agent) is a distinct and underrated threat class separate from direct prompt injection. Formal evaluation tooling is now available.

*Source: [NIST CAISI](https://www.nist.gov/news-events/news/2025/01/technical-blog-strengthening-ai-agent-hijacking-evaluations)*

### arXiv — Exploiting Web Search Tools for Data Exfiltration (Oct 2025)
Demonstrates real-world attack scenario: indirect prompt injection via poisoned web content exploits RAG/tool-calling to route sensitive corporate data retrieved by an AI agent to an attacker-controlled endpoint. Highlights enterprise deployments using AI web-search agents as a high-risk, underexplored surface.

*Source: [arXiv:2510.09093](https://arxiv.org/abs/2510.09093)*

### Trail of Bits — Prompt Injection to RCE in AI Agents (Oct 2025)
Full technical walkthrough of the prompt injection → RCE chain in agentic systems. Documents argument injection and workspace config injection as distinct exploitation paths. Notes that malicious payloads are effective across code comments, rule files, repository metadata, and log output — significantly expanding attack surface beyond interactive prompts.

*Source: [Trail of Bits Blog](https://blog.trailofbits.com/2025/10/22/prompt-injection-to-rce-in-ai-agents/)*

### Cisco — State of AI Security 2026
- **71% of organizations** planning to deploy agentic AI report not being prepared to secure those deployments.
- Many vendors have chosen **not to fix** reported vulnerabilities, citing functionality concerns — indicating systemic "insecure by design" patterns.

### Vendor Response Pattern
Across IDEsaster, MCP, and supply chain disclosures, a recurring finding is that vendors classify novel agent attack surfaces as outside their security responsibility boundary. Several CVEs required public pressure before patches were issued. Only Cursor (v1.3.9) and GitHub Copilot (CLI v0.0.423+) have issued explicit version-pinned security advisories tied to specific CVE mitigations as of this report.

## Analyst Note

The structural theme across every 2025–2026 AI coding security incident is the same: **trust without verification at the interface boundary**. Whether the vector is a prompt injection through a code comment, a hallucinated npm package, a poisoned MCP tool definition, or a crafted `.cursorrules` file — the agent (or developer) trusted unverified external content and treated it as authoritative.

Three converging forces make this worse in 2026:
1. **Agentic autonomy is expanding** (Claude Code, Devin, Cursor background agents) — more tools, more filesystem access, more network egress, less human review.
2. **MCP is being adopted faster than it is being secured** — 38% of servers have no authentication at all; the spec itself creates security anti-patterns (session IDs in URLs).
3. **AI-generated code is eliminating the developer as a supply chain checkpoint** — slopsquatting bypasses the human judgment that previously caught typosquatting; developers running AI-suggested `npm install` commands no longer read what they're installing.

The attack surface is not theoretical. CVE-2025-53773, CVE-2025-6514, the Supabase breach, and the Ethereum developer wallet drain are all confirmed incidents with real-world financial and credential consequences. The gap between disclosed vulnerabilities and patched deployments remains wide, and Cisco's data shows most organizations are not prepared to defend the agentic AI they are already deploying.

**Recommended minimum posture for teams using AI coding agents:**
- Pin to patched versions: Cursor ≥ 1.3.9, Copilot CLI ≥ 0.0.423; audit Claude Code hooks in all cloned repositories
- Treat every MCP server as an untrusted third-party dependency; audit tool definitions before and after installation
- Verify all AI-suggested package names independently before installing; do not `npm install` from AI output without cross-referencing the official registry
- Apply a 7-day cooldown on newly published package versions (would have blocked 8/10 major 2025 supply chain attacks)
- Enforce least-privilege for all agent tool access; require human approval for filesystem writes, network egress, and config changes
- Sandbox agent workspaces from host credentials (API keys, SSH keys, cloud tokens)
