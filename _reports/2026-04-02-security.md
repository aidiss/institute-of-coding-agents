# Security Brief — AI Coding Agents
**2026-04-02**

## Incidents & Vulnerabilities

### IDEsaster: 30+ CVEs Across 10+ AI Coding Products

A six-month research campaign by researcher Ari Marzouk (MaccariTA) catalogued **30+ vulnerabilities across 10+ AI IDEs** — GitHub Copilot, Cursor, Claude Code, Windsurf, Kiro.dev, Zed.dev, Roo Code, JetBrains Junie, Cline, and Gemini CLI — yielding **24 assigned CVEs** and AWS advisory AWS-2025-019. The attack chain targets the **base IDE layer** rather than AI-specific components; 100% of tested tools were affected. Attack flow: **Prompt Injection → Tool Abuse → Base IDE Feature Exploitation**.

Claude Code was rated lowest risk (mandatory tool confirmation, no auto-approve flag, sandboxed MCP servers). Cursor was rated Critical due to auto-approve availability and unsandboxed MCP servers.

#### CamoLeak — GitHub Copilot Chat (CVSS 9.6)

Disclosed Oct 8 2025; patched Aug 14 2025. Attacker embeds hidden instructions in a pull request description using GitHub invisible comments. When a developer queries Copilot Chat about the PR, the payload executes with the victim's full permissions. Exfiltration bypassed CSP by encoding secrets character-by-character into the ordering of ~100 one-pixel images served through GitHub's Camo proxy — secrets were reconstructable from Camo proxy request logs. **Status: Patched (image rendering in Copilot Chat disabled).**

### PromptPwnd — AI Agents in CI/CD Pipelines (Actively Exploited)

Discovered by Aikido Security. When AI agents (Gemini CLI, Claude Code, OpenAI Codex, GitHub AI Inference) are deployed in GitHub Actions/GitLab CI/CD, untrusted user-supplied strings (issue bodies, PR descriptions, commit messages) can be injected as prompts. The AI then uses high-privilege tools to exfiltrate `GITHUB_TOKEN`, API keys, and cloud credentials — or to poison the repository.

**Actively exploited against at least 5 Fortune 500 companies.** Google patched Gemini CLI within 4 days. Attack can be triggered by any external user filing a GitHub issue — no special permissions required.

### 94 Unpatched Chromium CVEs in Cursor and Windsurf (OX Security)

Responsible disclosure sent Oct 12 2025. Cursor and Windsurf are built on outdated Electron/Chromium/V8 versions. Researchers successfully weaponized CVE-2025-7656 against the latest releases of both IDEs, affecting ~**1.8 million developers**. A further **93 CVEs with existing PoCs** remain unweaponized but publicly available. **Windsurf never responded to disclosure. Cursor status unclear.**

### Vibe Security Radar — AI-Attributed CVEs Accelerating (Georgia Tech SSLab)

Monitoring CVE.org, NVD, GHSA, and OSV for AI tool commit signatures:

- **74 confirmed CVEs** attributable to AI-authored code (of 43,849 advisories analyzed) as of March 20 2026
- Monthly trend: **6 (Jan 2026) → 15 (Feb) → 35+ (Mar)** — roughly doubling each month
- By tool: Claude Code (27), GitHub Copilot (4), Devin (2), Aether (1), Cursor (1)
- Claude Code is over-represented because it always leaves co-author metadata; inline suggestion tools leave no trace
- Researchers estimate **true figure is 5–10× higher** (~400–700 cases in open-source ecosystem)

## Supply Chain

### LiteLLM PyPI Compromise — **Actively Exploited**, March 24 2026

**Threat actor:** TeamPCP  
**Affected versions:** litellm 1.82.7 and 1.82.8  
**Window:** ~5 hours, 2026-03-24 10:39–16:00 UTC  
**Downloads of compromised versions:** ~40,000 before PyPI quarantine

**Attack chain:**
1. TeamPCP compromised `aquasecurity/trivy-action` GitHub Action (version-tag pinned, not commit-pinned) to carry a credential harvester.
2. LiteLLM's CI pulled the poisoned action; the `PYPI_PUBLISH` token was exfiltrated.
3. Attackers published `litellm 1.82.7` then `1.82.8` to PyPI using the stolen token.
4. A `.pth` file (`litellm_init.pth`) auto-executed a multi-stage payload on every Python process startup.

**Payload capabilities:** SSH key theft, cloud provider token harvest (AWS/GCP/Azure), Kubernetes secret extraction, `.env` file exfiltration, Kubernetes lateral movement (creates privileged `alpine:latest` pods mounting host filesystem), persistent systemd backdoor polling C2 at `models.litellm[.]cloud`.

**Clean version:** v1.83.0. **Docker users unaffected** (pinned requirements.txt).

**Detection:** Check for `~/.config/sysmon/sysmon.py` and associated systemd unit; audit `kube-system` for `node-setup-*` pods.

**Broader campaign:** Same actor hit **Checkmarx KICS** (2026-03-23) and **Telnyx Python SDK** (2026-03-27) using credentials from the same pipeline compromise.

**Remediation:** Remove litellm 1.82.7/1.82.8 and purge pip caches. Enable **PyPI Trusted Publishers (OIDC)**. Pin all CI action dependencies to commit SHAs. Rotate all credentials on affected hosts.

### Slopsquatting — AI Hallucinated Package Names as Attack Surface

Coined by Python Software Foundation Developer-in-Residence Seth Larson. Key research data (UTSA / Virginia Tech / University of Oklahoma, 2025):

- Analyzed **576,000 code samples** from 16 LLMs
- ~**20%** of recommended packages did not exist in any public registry
- **43%** of hallucinated names repeated across 10 re-runs of the same prompt — reliably exploitable
- **58%** repeated more than once

Real example: `huggingface-cli` (non-existent PyPI package) is hallucinated repeatedly; Alibaba copy-pasted AI-generated instructions referencing it into their public documentation. Attacker registers the name; all developers following those instructions install the payload.

Autonomous agents that can `pip install` without human review are the highest-risk surface — a single hallucinated dependency can reach production at machine speed. No confirmed in-the-wild exploitation as of April 2026, but the attack is theoretically proven at scale.

### Malicious Extension Supply Chain — Cursor, Windsurf, Google Antigravity

VS Code forks use the Open VSX registry but inherit Microsoft's extension recommendation list. Many recommended extensions don't exist in Open VSX, leaving unclaimed namespaces that attackers can register with malicious payloads. IDEs actively push these "recommended extensions" to users.

**Status:** Cursor patched Dec 1 2025. Google patched Jan 1 2026. **Windsurf never responded.**

---

### MCP Ecosystem Supply Chain Risks

Analysis of 7,000+ public MCP servers found **36.7% vulnerable to SSRF**. The Vulnerable MCP Project ([vulnerablemcp.info](https://vulnerablemcp.info)) now tracks **50+ documented vulnerabilities, 13 critical**, researched by 32+ security researchers.

"Tool poisoning" / rug-pull: MCP servers can silently modify their tool definitions after initial approval — tools consented on Day 1 can be rerouted to exfiltrate API keys by Day 7 with no re-consent required.

## Research & Advisories

### MCP Remote OS Command Injection — CVE-2025-6514 (CVSS 9.6)

Discovered by JFrog Security Research. Arbitrary OS command execution when any MCP client connects to an untrusted server. No special privileges or user interaction required. **Affects mcp-remote 0.0.5–0.1.15. Patched. First documented real-world RCE in MCP deployments.**

### MCP Inspector CSRF → RCE — CVE-2025-49596

Anthropic's MCP Inspector bound to `0.0.0.0` with no authentication. Visiting a crafted webpage triggered a CSRF attack that installed arbitrary MCP servers and executed code with the user's full privileges. **Patched by Anthropic, June 2025.**

### Anthropic mcp-server-git RCE Chain — CVE-2025-68143, -68144, -68145

Three chained CVEs in Anthropic's own `mcp-server-git`: path validation bypass (CVE-2025-68145), unrestricted `git_init` that can convert `.ssh` into a git repo (CVE-2025-68143), and argument injection in `git_diff` (CVE-2025-68144). Combined with the Filesystem MCP server, these achieve full RCE via malicious `.git/config` files. **Affects Claude Desktop, Cursor, and Windsurf users of the package. Patched.**

### MCP TypeScript SDK Cross-Client Data Leak — CVE-2026-25536 (CVSS 7.1)

When a single `McpServer` instance with `StreamableHTTPServerTransport` is reused across multiple clients, responses can leak between clients. Relevant for any multi-tenant MCP deployment. **Affects v1.10.0–v1.25.3. Patched.**

### MCPJam Inspector RCE — CVE-2026-23744 (CVSS 9.8)

MCPJam inspector bound to `0.0.0.0` with no authentication. A crafted HTTP request installs an MCP server and executes arbitrary code on the host — no user interaction required. **Affects versions ≤ 1.4.2. Patched.**

### Prompt Injection SoK Paper (arXiv 2601.17548, January 2026)

Systematization of Knowledge paper analyzing **78 studies (2021–2026)** on prompt injection against agentic coding assistants. Key finding: **attack success rates against state-of-the-art defenses exceed 85%** when adaptive attack strategies are used. Proposes a three-dimensional taxonomy across delivery vectors, attack modalities, and propagation behaviors.

### Unit 42 — MCP Sampling Attack Vectors

Palo Alto Unit 42 documented new prompt injection vectors through MCP's sampling mechanism, where a malicious server influences the model's intermediate reasoning steps without triggering tool-call consent dialogs.

### AI-Generated Code Quality (January 2026)

Sonar data: **96% of developers distrust AI-generated code**, yet only **48% consistently verify it before committing**. A study of 15 AI-built apps across 5 platforms (Claude Code, Cursor, Windsurf, Replit, Devin) found **69 vulnerabilities across apps**, including 6 critical. All tools produced apps lacking CSRF protection. Every tool introduced SSRF vulnerabilities. Dominant failures: missing ownership checks (IDOR), admin endpoint bypass, negative pricing.

## Key Mitigations

1. **Sandbox agent execution** — isolate from host filesystem, network, and credentials.
2. **Pin CI/CD action dependencies to commit SHAs** — version-tag pinning is exploitable (LiteLLM).
3. **Enable PyPI Trusted Publishers (OIDC)** to eliminate long-lived publishing tokens from CI pipelines.
4. **Least-privilege MCP tool scoping** — restrict server access; treat tool definitions as untrusted after initial approval.
5. **Verify all AI-suggested package names** before install; use lockfiles and hash verification.
6. **Audit `.vscode/settings.json`, `.mcp.json`, and agent config files** in any repository before opening it.
7. **Never store live credentials in `.env` files** within AI-agent working directories.
8. **Treat MCP server tool definitions as untrusted after version changes** — re-verify on update.
9. **Update Windsurf** — multiple unpatched Chromium CVEs with public PoCs.
