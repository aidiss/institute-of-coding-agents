# Security Report — 2026-03-18

> Vulnerabilities, supply chain attacks, and safety incidents affecting AI coding agents.

---

## Incidents & Vulnerabilities

### MCPJam Inspector — Unauthenticated RCE
**CVE-2026-23744** | CVSS: 9.8 | **Status:** Patched (v1.4.3)

MCPJam Inspector (≤ v1.4.2) bound its critical endpoint to `0.0.0.0` with no authentication. A crafted HTTP request can install an arbitrary MCP server and execute code on the host. No user interaction required. Considered among the most severe MCP-class vulnerabilities disclosed to date.

---

### Azure MCP Server — "MCPwned" RCE
**Researcher:** Token Security | **Disclosure venue:** RSAC™ 2026 | **Status:** Patched

Research to be presented at RSAC 2026 demonstrates an RCE vulnerability in Microsoft's Azure MCP server capable of compromising cloud environments. The flaw follows the now-established MCP attack pattern of exploiting implicit tool trust to escalate from prompt injection to full cloud credential access.

---

### MCP TypeScript SDK — Cross-Client Data Leak
**CVE-2026-25536** | CVSS: 7.1 | **Affected:** MCP TypeScript SDK v1.10.0–v1.25.3 | **Status:** Patched

When a single `McpServer` instance with `StreamableHTTPServerTransport` is reused across multiple clients, responses leak across client boundaries — one client may receive data intended for another. Affects any multi-tenant or multi-session MCP deployment built on the TypeScript SDK before the patch.

---

### Cursor IDE — Triple CVE Chain
**CVE-2026-22708** (CVSS 9.8) / **CVE-2026-26268** / **CVE-2026-21523** | **Status:** Patched

Three distinct vulnerabilities in Cursor combine to enable RCE via indirect prompt injection:

| CVE | Description | CVSS |
|---|---|---|
| CVE-2026-22708 | Shell built-in bypass (`export`, `typeset`) — executed without user approval even on empty allowlists | 9.8 |
| CVE-2026-26268 | Git hook escape — attacker-controlled `.git/config` triggers execution via hook lifecycle | — |
| CVE-2026-21523 | TOCTOU race condition — file replacement between validation and execution | — |

Both "zero-click" and "one-click" RCE scenarios confirmed. The shell built-in bypass is notable because it circumvents Cursor's tool-approval model entirely through standard shell syntax manipulation.

### Claude Code — Configuration Injection (Hooks + API Key Exfiltration)
**CVE-2025-59536** (CVSS 8.7) / **CVE-2026-21852** (CVSS 5.3) | **Researcher:** Check Point Research | **Disclosed:** February 2026 | **Status:** Both patched

Two vulnerabilities triggered by opening a crafted repository:

| CVE | CVSS | Description | Fixed |
|---|---|---|---|
| CVE-2025-59536 | 8.7 | Malicious Hook in `.claude/settings.json` executes shell commands before trust dialog appears | v1.0.111 (Oct 2025) |
| CVE-2026-21852 | 5.3 | `ANTHROPIC_BASE_URL` override redirects API calls → developer API key exfiltrated to attacker server | v2.0.65 (Jan 2026) |

CVE-2025-59536 also included an MCP consent bypass: `autoApproveTools` in `.mcp.json` silently pre-authorizes all MCP servers.

### OpenAI Codex CLI — Command Injection
**CVE-2025-61260** | **Status:** Patched

Codex CLI implicitly trusts MCP server entries in `.env` and `./.codex/config.toml` and executes them at startup without user confirmation. Tampering with these repository-resident files enables arbitrary command execution on the developer's machine.

### MCP in Active Cyberattack Infrastructure
**Status:** Ongoing threat class | No CVE assigned

Security researchers identified an exposed server hosting active attack infrastructure leveraging MCP to connect LLMs directly to compromised systems. The server contained 1,000+ files including firewall configurations, credential dumps, and attack plans. Adversaries use MCP-connected agents to automate reconnaissance analysis, vulnerability assessment, and attack planning — dramatically accelerating intrusion timelines beyond traditional manual operations.

---

### Memory Poisoning — Persistent Prompt Injection
**Status:** Active threat class | No CVE assigned

Unlike standard prompt injections that end when a session closes, poisoned agent memory persists across sessions. A malicious instruction injected into an agent's memory store is recalled in future sessions — sometimes days or weeks later. This enables attackers to establish persistent footholds in developer workflows without ongoing access to the original attack vector.

---

### Zero-Click RCE via MCP + Document Trigger
**Status:** Active threat class | No CVE assigned

Researchers documented a fully zero-click RCE chain: a crafted Google Docs file triggers an IDE agent to fetch attacker-authored instructions from a malicious MCP server, execute a Python payload, and harvest secrets — all without any user interaction. Combines ambient document injection with MCP tool execution.

## Supply Chain

### Slopsquatting — Hallucinated Package Attacks (Updated Metrics)
**Term coined by:** Seth Larson | **Research:** USENIX Security 2025 / ongoing monitoring

LLMs hallucinate non-existent package names at scale. Attackers register those names with malicious payloads before developers install them.

**Updated scale figures (through January 2026):**
- 19.7% of generated code samples include hallucinated package names (756,000 samples, 16 models)
- 43% of hallucinated names repeat consistently across repeated prompts — predictable squatting targets
- 128 tracked "phantom packages" accumulated **121,539 downloads** (July 2025–January 2026), averaging ~4,000 downloads/week
- `openapi-generator-cli` alone: **48,356 downloads**
- RAG-grounded models reduce hallucination rate by up to 85% but degrade overall code quality

**Documented examples:**
- `huggingface-cli` PoC (Lasso Security): 30,000+ downloads in 3 months; Alibaba referenced it in official documentation
- `react-codeshift`: hallucinated npm package combining `jscodeshift` + `react-codemod`; AI agents observed attempting installation in the wild

---

### Automated Dependency Side-Loading via IDE Extensions
**Researcher:** Multiple | **Disclosed:** Early 2026 | **Status:** Extensions removed; no CVE

Researchers identified a campaign involving VS Code extensions with **1.5 million+ combined installs** that posed as AI coding helpers. A hidden profiling engine monitored active coding sessions; when an AI tool generated code, the extension silently modified suggestions to include attacker-controlled dependencies. Detected through behavioral analysis of extension network activity.

---

### PromptPwnd — CI/CD Pipeline Injection
**Status:** Active attack class | No CVE assigned

A new attack class named **PromptPwnd** targets AI agents connected to GitHub Actions or GitLab CI/CD pipelines. Prompt injections trick agents into invoking privileged built-in tools, leading to information leakage or code execution within the pipeline context. Builds on the Cline/OpenClaw incident (February 2026) in which ~4,000 developer machines were infected via a compromised npm package seeded through a similar workflow.

---

### Smithery Supply Chain — Path Traversal → Fly.io Token Compromise
**Status:** Incident resolved; no CVE assigned

A path-traversal bug in Smithery's `smithery.yaml` build config (`dockerBuildPath: ".."`) caused the MCP registry to build Docker images from the builder's home directory. The exploit leaked `~/.docker/config.json` including a Fly.io API token that granted administrative control over **3,000+ hosted MCP server applications**.

## Research & Advisories

### OWASP Top 10 for Agentic Applications 2026
**Published:** December 2025 | **Source:** [genai.owasp.org](https://genai.owasp.org/resource/owasp-top-10-for-agentic-applications-for-2026/)

OWASP released a dedicated framework for agentic AI security risks, developed by 100+ industry experts. Key risks (ASI01–ASI10) cover: Agent Goal Hijack, Cascading Failures, Unbounded Consumption, and Rogue Agents. Two core principles introduced:

1. **Least-Agency** — grant only minimum autonomy needed for the defined task
2. **Strong Observability** — comprehensive logging of goal state, tool-use patterns, and decision paths

The framework explicitly states that the existing OWASP Top 10 for LLM Applications is insufficient for agentic systems, where a minor prompt injection can cascade into system-wide compromise.

---

### IDEsaster: 30+ CVEs Across All Major AI IDEs
**Researcher:** Ari Marzouk (MaccariTA) | **Disclosed:** December 2025 | **Status:** Most patched

100% of tested AI IDEs vulnerable via shared attack chain: **Prompt Injection → Agent Tools → Base IDE Features**. 24 CVEs assigned; AWS advisory AWS-2025-019 issued. Cursor, Windsurf, GitHub Copilot, Roo Code, JetBrains Junie, Cline, Kiro.dev, Zed.dev, Gemini CLI, and Claude Code all affected.

Cursor and Windsurf are built on outdated Chromium/Electron versions, exposing ~1.8 million developers to 94+ known browser CVEs. CVE-2025-7656 (patched Chromium flaw) was successfully weaponized against current Cursor and Windsurf releases at time of disclosure.

---

### "Promptware" Kill Chain
**Researchers:** Tel Aviv University, Ben-Gurion University, Harvard | **Published:** February 2026

Formalizes AI-targeting malware as "promptware" with a seven-stage kill chain: Initial Access → Privilege Escalation → Reconnaissance → Lateral Movement → Persistence → C2 → Execution. By 2025–2026, 15 of 21 documented attacks covered 4+ stages; AI coding assistants were the most targeted category (7 of 21 incidents). No current defenses address the Reconnaissance stage.
