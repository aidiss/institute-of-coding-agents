# Security Brief — AI Coding Agents
**2026-04-11**

---

## Incidents & Vulnerabilities

### GitHub Copilot — Wormable RCE via Prompt Injection (CVE-2025-53773, CVSS 7.8)

Researcher Johann Rehberger ("Embrace the Red") demonstrated that malicious prompts embedded in source code comments, GitHub issues, or web content can instruct Copilot to write `"chat.tools.autoApprove": true` to `.vscode/settings.json`, disabling all execution confirmations ("YOLO mode"). Once set, the AI executes arbitrary shell commands without user intervention.

The attack is **wormable**: infected repositories automatically propagate the instruction to new clones, producing what the researcher termed "ZombAI" nodes.

**Status: Patched — Microsoft August 2025 Patch Tuesday. Affected versions prior to August 2025 release. Upgrade immediately. Any `.vscode/settings.json` containing `chat.tools.autoApprove: true` from an unknown source should be treated as compromised.**

Related: **CVE-2025-59145** (CVSS 9.6) — A separate GitHub Copilot critical allowing exfiltration of API keys and private source code through prompt manipulation alone, with no code execution on the developer machine required. Patched in August 2025.

---

### Claude Code — Repository Hook RCE and API Key Exfiltration

Two CVEs disclosed by Check Point Research (February 2026) share a root cause: project-level configuration files are processed by Claude Code on launch before any user trust dialog is shown.

| CVE | Type | CVSS | Fixed In | Disclosed |
|-----|------|------|----------|-----------|
| CVE-2025-59536 | RCE via hooks in `.claude/settings.json` | 8.7 | v1.0.111+ / v2.0.65 | Feb 2026 |
| CVE-2026-21852 | API key exfiltration via `ANTHROPIC_BASE_URL` redirect | 5.3 | v2.0.65 | Feb 2026 |

**CVE-2025-59536:** Malicious `.claude/settings.json` embedded in a cloned repository defines Hook commands that execute arbitrary shell commands on project open — before the trust prompt fires. A separate `enableAllProjectMcpServers` bypass auto-approved MCP servers before the user could read the consent dialog.

**CVE-2026-21852:** Project config overrides `ANTHROPIC_BASE_URL`, routing all API traffic to an attacker-controlled server and exposing the Anthropic API key in plaintext Authorization headers. A public PoC exploit was published. Timeline: reported July 2025 → first patch August 2025 → public disclosure February 2026.

**Status: Both CVEs patched in v2.0.65. Upgrade to latest Claude Code. Treat repository cloning as equivalent to running untrusted code.**

---

### Cursor — MCPoison and CurXecute (CVE-2025-54135 / CVE-2025-54136)

Check Point Research disclosed two chained vulnerabilities in Cursor v1.2.4 and below:

- **CVE-2025-54136 (MCPoison):** Once a developer trusts an MCP configuration file, an attacker can silently swap tool definitions to malicious commands without triggering any re-approval prompt. The swap takes effect immediately on next execution with no warning.
- **CVE-2025-54135 (CurXecute):** Companion flaw enabling initial payload delivery; demonstrated via a malicious Slack message processed by the Slack MCP Server.

Additionally, **CVE-2025-59944** (CVSS 8.0): Cursor's file-protection rules used case-sensitive path comparisons. Creating `.cUrSoR/mcp.json` (mixed case) bypassed all protections on case-insensitive filesystems (Windows/macOS), as the OS resolves the path to `.cursor/mcp.json`. Enabled persistent RCE via prompt injection.

**Status: CVE-2025-54135 and CVE-2025-54136 patched in Cursor v1.3. CVE-2025-59944 patched in Cursor v1.7. Verify running version.**

---

### Windsurf / Codeium — Path Traversal and Credential Exfiltration (CVE-2025-62353, CVSS 9.8)

A critical path traversal vulnerability in Windsurf with documented prompt injection attack chains that can exfiltrate `.env` files and other credential stores from developer workstations.

**Status: Disclosed. Fix availability unconfirmed. Windsurf also inherits 94+ unpatched Chromium CVEs from an outdated Electron base (last updated to Chromium 132, March 2025), including three on CISA's Known Exploited Vulnerabilities list: CVE-2025-5419, CVE-2025-6554, CVE-2025-6558.**

### OpenClaw / ClawHub — Local Agent WebSocket Hijacking ("ClawJacked")

Oasis Security (February 26, 2026) disclosed that any website can open a WebSocket to OpenClaw's localhost gateway, brute-force the password (rate limiter exempted localhost connections), and auto-register as a trusted device (the gateway auto-approved localhost pairings). No plugins, extensions, or user interaction required.

- Full agent takeover: interact with the AI, dump configs, enumerate devices, read logs.
- 30,000+ internet-exposed OpenClaw instances identified, many unauthenticated.

**Status: Patched in OpenClaw v2026.2.13 / v2026.2.25. Update immediately.**

---

### MCP Remote — OS Command Injection (CVE-2025-6514, CVSS 9.6)

Discovered by JFrog Security Research (July 9, 2025). During OAuth authorization, `mcp-remote` passes a malicious server's `authorization_endpoint` response directly to an OS command without sanitization. This is the **first documented full RCE from a remote MCP server to the client OS** — affecting 437,000+ npm downloads across versions 0.0.5–0.1.15.

**Status: Patched in v0.1.16. Upgrade immediately.**

---

### Anthropic mcp-server-git — Chained RCE (CVE-2025-68143 / 68144 / 68145)

Three chained CVEs in Anthropic's official MCP git server, patched in v2025.12.18 (December 2025). All three are triggerable via prompt injection — a malicious README, GitHub issue, or poisoned webpage is sufficient:

| CVE | Type | Details |
|-----|------|---------|
| CVE-2025-68145 | Path validation bypass | `--repository` flag restriction not enforced for subsequent `repo_path` arguments; allows access to any repo on the filesystem |
| CVE-2025-68143 | Unrestricted `git_init` | Accepted arbitrary filesystem paths without validation; Anthropic removed this tool entirely in the patch |
| CVE-2025-68144 | Argument injection | User-controlled args passed to GitPython unsanitized in `git_diff`/`git_checkout`; `--output=/path` overwrites arbitrary files |

Combined with the Filesystem MCP server, the chain achieves full RCE via malicious `.git/config` injection. Reported June 2025; fixed December 2025 — a six-month remediation window.

**Status: Patched in mcp-server-git v2025.12.18. Upgrade and re-audit any repos processed by the tool during the exposure window.**

### Zed IDE — LSP Binary and MCP Auto-Load ACE

Two GitHub Security Advisories patched in Zed v0.218.2-pre:

- **GHSA-29cp-2hmh-hcxj (CVSS 7.8):** Zed loads LSP configurations from `.zed/settings.json` within a project directory. A malicious `settings.json` can define LSP binary paths containing arbitrary shell commands that run with user privileges when any matching source file is opened. Published December 17, 2025.
- **GHSA-cv6g-cmxc-vw8j:** Zed auto-loaded MCP context server settings from workspace config without user confirmation, allowing malicious projects to define MCP tools that execute arbitrary code silently.

Additional Mindgard disclosures: **CVE-2025-68432, CVE-2025-68433** (Zed MCP auto-loading). Zed introduced "Worktree Trust" as a systemic response. **CVE-2025-55012** (Agent Panel RCE via permissions bypass) patched separately in v0.197.3.

**Status: Patched in Zed v0.218.2-pre / v0.197.3. Upgrade and enable Worktree Trust.**

## Supply Chain

### Slopsquatting — Hallucinated Package Names as Reliable Attack Vector

A study analyzing 756,000 AI-generated code samples across 16 LLMs (University of Texas at San Antonio, University of Oklahoma, Virginia Tech) found approximately **20% of recommended packages do not exist**:

- Open-source models (CodeLlama, DeepSeek, Mistral): ~21.7% hallucination rate
- Commercial models (GPT-4 Turbo): ~5.2% hallucination rate
- **43% of hallucinated names recur across 10 repeated queries with the same prompt** — making the attack statistically reliable for pre-registration
- Researchers catalogued over 200,000 unique hallucinated package names

Real-world validation: a researcher registered `huggingface-cli` (a consistently hallucinated name) as a blank PyPI package and received **30,000+ genuine downloads in three months**.

The risk amplifies in agentic pipelines where tools like `pip install` or `npm install` execute without human review at machine speed.

**No systematic fix. Mitigations: `--require-hashes` enforcement, private registries with allowlists, lockfile pinning, and LLM output validation before any package installation step.**

### ClawHavoc — OpenClaw Skill Marketplace Compromise

Beginning January 27, 2026, attackers flooded the ClawHub skill marketplace with malicious packages. Antiy CERT confirmed **1,184 malicious skills**; Bitdefender placed the count at ~900 — approximately 20% of the entire ClawHub ecosystem (10,700+ total skills).

Attack techniques:
1. Prompt injection in skill descriptor files
2. Hidden reverse shell scripts
3. Token exfiltration — silently uploading OpenClaw bot credentials from `~/.clawdbot/.env` to external webhooks

**Infrastructure failure:** ClawHub required only a one-week-old GitHub account to publish; no automated scanning, no code signing, no review process.

**Status: Ongoing remediation. Treat all ClawHub skills as untrusted until the platform's security model is revised.**

### Broader Registry Threat Landscape

Supply-chain attacks on package registries surged 73% in 2025–2026. Four significant supply-chain attacks were disclosed in Q1 2026 alone, targeting Trivy, LiteLLM (PyPI), Telnyx, and Axios. The emergence of autonomous coding agents that can invoke package managers in CI/CD pipelines without human review accelerates the risk — hallucinated or typosquatted packages can be installed and executed before any human sees the dependency.

## Research & Advisories

### IDEsaster — Universal Attack Chain Across All Tested AI IDEs

Researcher Ari Marzouk documented a three-stage attack chain that **100% of tested AI IDEs were vulnerable to**: **Prompt Injection → Agent Tool Use → Base IDE Feature Weaponization**. 24 CVEs assigned across Cursor, Windsurf, GitHub Copilot, Claude Code, Zed.dev, JetBrains Junie, Roo Code, Cline, Kiro.dev, and Gemini CLI.

The attack uses only legitimate IDE features — terminal execution, file I/O, settings APIs — as the execution primitive. No exploit of the AI model itself is required. Three primary sub-classes:

1. **Remote JSON Schema Injection** — prompt injection causes the agent to write a file with a JSON schema reference to an attacker URL; IDE validation sends an outbound GET leaking workspace data.
2. **IDE Settings Overwrite** — agent overwrites `.vscode/settings.json` or `.idea/workspace.xml` with task runners or terminal profiles that execute arbitrary commands on next workspace open.
3. **Multi-root Workspace Hijacking** — `.code-workspace` files reference attacker-controlled folders, silently overriding trusted project settings.

**Status: Most CVEs patched in vendor releases (Cursor 1.7, JetBrains Junie December 2025). Gemini CLI issues unresolved as of April 2026.**

### Trail of Bits — Argument Injection → RCE in AI Agents (October 2025)

Even when agents have restricted "safe command" whitelists, argument injection via flags like `go test -exec`, `git show`, or ripgrep achieves arbitrary command execution in a single prompt step. Attack payloads work when embedded in code comments, rule files, logging output, or GitHub repositories — no direct user interaction required.

This research establishes that **command whitelists without argument sanitization are not a sufficient security boundary for AI agents with shell access**.

---

### MCP Protocol — Structural Security Deficits

Large-scale audits of publicly exposed MCP servers reveal systemic weaknesses:
- **43%** have OAuth authentication flaws
- **43%** contain command injection vulnerabilities
- **33%** allow unrestricted network access
- **22%** allow access to files outside intended data sources

**MCP Rug-Pull (No CVE):** MCP has no re-consent mechanism for server-side tool definition changes. A tool approved as read-only on Day 1 can include exfiltration logic by Day 7 with no client-side warning. Palo Alto Unit 42 documented this class of "rug pull" attacks in detail.

Invariant Labs demonstrated exfiltration of a user's entire WhatsApp message history via tool poisoning — malicious instructions embedded in MCP tool *descriptions* (invisible to the user in the UI, but read and acted upon by the AI).

**Mitigation: Pin MCP server versions by commit hash; treat every update as a new approval decision.**

---

### Rule Files Backdoor — Hidden Unicode in AI Agent Instructions

Pillar Security (March 2025) found that hidden Unicode characters and zero-width characters can be embedded in AI agent rule files (`.cursorrules`, `.github/copilot-instructions.md`) to inject invisible instructions that persist across code reviews. Because rule files are shared via corporate templates and open-source repositories, a single compromised file propagates malicious instructions at scale while appearing entirely benign.

**GitHub added a hidden-Unicode warning as a partial mitigation. Users should audit all rule files from external sources.**

### AI-Generated Code — Security Debt Accelerating

The Cloud Security Alliance tracked CVEs directly attributable to AI-authored code:
- January 2026: 6 CVEs
- February 2026: 15 CVEs
- **March 2026: 35 CVEs** (estimated 5–10× higher in the untracked open-source ecosystem)

Key statistics:
- **45% of AI-generated code samples** introduce OWASP Top 10 vulnerabilities (Veracode)
- AI-assisted commits have a **3.2% secret-leak rate** vs. 1.5% baseline — double the non-AI rate
- **53% of AI-generated code** ships with at least one vulnerability
- Fortune 50 empirical study: AI-assisted developers commit 3–4× faster but introduce security findings at **10× the rate** of peers
- Privilege escalation paths in AI-generated code rose 322%; architectural design flaws rose 153% in 2025–2026

Notable attributed CVEs: **CVE-2025-55526** (CVSS 9.1, directory traversal in n8n-workflows); **CVE-2025-55182** (AI-generated Next.js vulnerability actively exploited across 766 hosts).

## Key Mitigations

1. **Treat repository cloning as code execution.** `.claude/settings.json`, `.cursor/mcp.json`, `.vscode/settings.json`, and `.code-workspace` files are processed by AI coding agents on startup. Open untrusted repositories only in isolated environments.
2. **Update Claude Code to v2.0.65+** — Fixes CVE-2025-59536 and CVE-2026-21852 (hook RCE and API key exfiltration).
3. **Update Cursor to v1.7+** — Fixes CurXecute (CVE-2025-54135), MCPoison (CVE-2025-54136), and the case-sensitivity bypass (CVE-2025-59944). Inherited Chromium CVEs (three on CISA KEV) remain unresolved.
4. **Upgrade mcp-remote to v0.1.16+** — Fixes CVE-2025-6514 (CVSS 9.6, remote OS command injection).
5. **Upgrade mcp-server-git to v2025.12.18+** — Fixes three chained CVEs (CVE-2025-68143/44/45) enabling full RCE.
6. **Pin MCP server versions by commit hash.** The MCP protocol has no re-consent mechanism for tool definition changes. Treat every server update as a new approval decision.
7. **Enforce `--require-hashes` in agent-accessible pip environments** — Prevents slopsquatting payloads from installing even when a hallucinated package name is registered on PyPI.
8. **Audit AI-generated code for OWASP Top 10 patterns** — 45–53% of AI-generated code samples contain at least one security flaw; secret-leak rates in AI-assisted commits are double the baseline.
9. **Audit all rule files (`.cursorrules`, Copilot instructions) from external sources** for hidden Unicode characters before enabling them.
10. **Review CI/CD token scopes.** Overly broad tokens are the primary supply chain attack vector for signed vendor extensions (cf. Amazon Q Developer CVE-2025-8217). Apply least privilege and rotate any token that transited a compromised runner.
