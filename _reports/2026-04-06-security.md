# Security Brief: AI Coding Agents
*2026-04-06*

A high-tempo quarter for AI coding agent security. March–April 2026 saw two confirmed PyPI/npm supply chain attacks, a wave of 30+ MCP CVEs in 60 days, a novel approval-dialog bypass technique affecting Claude Code and Copilot, and escalating evidence that AI-generated code ships vulnerabilities at scale.

---

## Incidents & Vulnerabilities

### Cursor IDE — Indirect Prompt Injection RCE
**CVE-2026-22708** (CVSS 9.8) | **CVE-2025-59944** (High) | Disclosed, patch status unconfirmed

Attackers exploit implicit trust in shell built-ins via indirect prompt injection, executing malicious commands through syntax manipulation in both zero-click and one-click scenarios. CVE-2025-59944 is a case-sensitivity bypass in Cursor's MCP configuration that enables persistence across IDE restarts (also tracked as MCPoison, CVE-2025-54136).

*Sources: SC Media, vulnerablemcp.info*

---

### GitHub Copilot — Silent Prompt Injection RCE
**CVE-2025-53773** (CVSS 9.6) | Disclosed; Microsoft initially classified outside threat model

Malicious instructions embedded in public repository code comments instruct Copilot to modify settings enabling code execution without user approval. Reported as actively exploited in production environments.

*Source: The Hacker News ThreatsDay Bulletin, Security Boulevard*

---

### VS Code — mcp.json Command Injection RCE
**CVE-2026-21518** | Patched — Microsoft March 2026 Patch Tuesday

Lack of validation on a user-supplied string in `mcp.json` handling allows remote attackers to execute arbitrary code via a crafted MCP configuration file. Addressed in Microsoft's March 2026 Patch Tuesday (84 flaws total).

*Source: SystemTek, The Hacker News*

---

### "Memory Jack" RCE — LangGraph / AutoGen
**CVE-2026-20805** | Disclosed, no patch

A second-order storage vulnerability: an attacker induces an agent to store a malicious serialized payload in long-term vector memory (Redis, Postgres, ChromaDB). When the framework later deserializes state using unsafe Python `pickle` or unrestricted YAML loading, arbitrary code executes server-side. Enables credential exfiltration, intranet pivoting, and supply chain poisoning. Affects LangGraph, AutoGen, and other frameworks using unsafe deserialization for state management.

*Source: Penligent AI*

---

### "Lies-in-the-Loop" (LITL) — Claude Code & GitHub Copilot
No CVE assigned | Disclosed; both vendors classified outside current threat models

Checkmarx Zero research demonstrating manipulation of the human approval dialog shown before an agent executes a risky action. Malicious instructions bury or obfuscate the dangerous action in the confirmation UI — every developer in the test group approved the action. Demonstrated on Claude Code and Microsoft Copilot Chat in VS Code.

*Source: Dark Reading, Checkmarx, Infosecurity Magazine*

---

### AI Coding Agent Code Quality Study
No CVE | Published March 2026 (DryRun Security)

38 scans across Claude Code, OpenAI Codex, and Google Gemini; 30 pull requests generated. **87% of PRs (26/30) contained at least one vulnerability; 143 total issues identified.** Top classes: broken access control, unauthenticated endpoints on destructive operations, OAuth missing state parameters, missing WebSocket authentication, hardcoded JWT secrets, client-side trust without server validation.

*Source: Help Net Security*

## Supply Chain

### LiteLLM PyPI Supply Chain Attack
**Confirmed incident** | March 24, 2026 | Attacker: "TeamPCP"

Malicious versions `litellm 1.82.7` and `1.82.8` published to PyPI. Payload extracted cloud credentials, API keys, and cryptocurrency wallets; installed persistent downloaders; contacted C2 servers on a 50-minute cycle to evade detection. With ~3 million daily downloads and presence in ~36% of cloud environments (Wiz Research), the packages were live for at least two hours before removal. Mercor publicly confirmed a security incident tied to this attack.

**Mitigation:** Remove affected versions; rotate all credentials in affected environments.

*Source: The Record / Recorded Future*

---

### Axios npm Supply Chain Attack
**Confirmed incident** | March 2026 | Attribution: Sapphire Sleet (North Korean state actor, per Microsoft Threat Intelligence)

Malicious injected dependencies in `axios 1.14.1` and `0.30.4` downloaded payloads from attacker-controlled C2 servers. Mitigated; Microsoft advisory published April 1, 2026.

*Source: Microsoft Security Blog*

---

### OpenClaw / ClawHub "ClawHavoc" Campaign
**Confirmed incident** | Ongoing through Q1 2026

1,184+ malicious skills identified in ClawHub (the third-party skill registry for OpenClaw, a framework with 325,000+ GitHub stars). A Koi Security audit of 2,857 skills found 341 malicious, 335 traced to a single coordinated operation. Malicious skills executed with full system permissions, exfiltrating private files, browser data, and API keys; some distributed the Atomic macOS Stealer. 135,000 OpenClaw instances were internet-exposed; 12,812 were exploitable via RCE. NVIDIA announced the NemoClaw enterprise security layer at GTC 2026 in response.

*Sources: eSecurity Planet, Trend Micro, HKCERT, AdminByRequest*

---

### Aqua Trivy VS Code Extension Compromise
**Confirmed incident** | February 27–28, 2026 | Attacker: Former employee with retained publishing rights

Malicious versions `aqua-trivy 1.8.12` and `1.8.13` on OpenVSX injected prompts weaponizing installed AI assistants (Copilot, Claude, Gemini, Codex, Kiro) to act as "forensic agents," locating and exfiltrating credentials and financial data via email, Slack, or GitHub. Available ~36 hours before removal by Socket researchers. Credential rotation advised.

*Source: NetCrook*

---

### TeamPCP GitHub Actions Campaign
**Confirmed incident** | March 19–27, 2026 | Attacker: TeamPCP (same group as LiteLLM)

Used retained credentials to inject malicious code into 75 of 76 `trivy-action` tags in Aqua Security's GitHub Actions — targeting the security toolchain itself.

*Source: Security Boulevard*

---

### Documentation Poisoning for AI Agent Context
Research published ~March 25, 2026 demonstrating that documentation portals can embed malicious instructions consumed by AI agents at runtime, simplifying supply chain attacks by poisoning agent context without touching package registries.

*Source: The Register*

### Slopsquatting — Hallucinated Package Dependency Attack

Attackers register package names that LLMs hallucinate. When a developer uses AI-generated code, the phantom dependency installs from an attacker-controlled package. Of 756,000 AI-generated code samples studied, ~20% recommended non-existent packages; 43% of hallucinated names were repeated across 10 separate queries — making them stable, predictable targets. A hallucinated npm package `react-codeshift` was found in active AI infrastructure in January 2026 with agents attempting to execute it. No mass confirmed in-the-wild slopsquatting attack has been publicly attributed yet, but active probing is documented.

*Sources: Aikido, Bleeping Computer, Infosecurity Magazine, Snyk, Mend.io, Trend Micro*

---

### AI-Generated Code CVE Trend

At least 35 new CVEs disclosed in March 2026 were directly attributed to AI-generated code, up from 6 in January and 15 in February — a steep acceleration. 28.65 million hardcoded secrets were detected in public GitHub commits in 2025, a 34% YoY increase and the largest single-year jump on record.

*Sources: The Register, Infosecurity Magazine, CrowdStrike*

---

### Trail of Bits — Prompt Injection to RCE Taxonomy
*(October 2025, still widely referenced)*

Foundational research establishing the technical pathway from prompt injection to full RCE in agentic systems with filesystem access, API credentials, and code execution capabilities. Remains the primary reference for the attack class underlying CVE-2026-22708, CVE-2025-53773, and the LITL findings.

*Source: Trail of Bits Blog*
