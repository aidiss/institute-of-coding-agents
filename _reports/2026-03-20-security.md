## Incidents & Vulnerabilities

### Claude Code — CVE-2025-59536 & CVE-2026-21852
**Status: Both Patched**

Disclosed by Check Point Research on February 25, 2026.

- **CVE-2025-59536** (CVSS 8.7): Code injection via malicious `.claude/settings.json`. Two vectors: (1) hooks — shell commands that fire at lifecycle events before the trust dialog appears; (2) MCP server entries auto-loaded from repo config, bypassing explicit consent. Patched August 26, 2025; CVE assigned October 3, 2025.
- **CVE-2026-21852**: API key exfiltration — attackers redirect Claude Code API traffic to an attacker-controlled server by modifying project config, harvesting the developer's Anthropic API key with zero user interaction. The key then grants access to shared Workspace files. Patched December 28, 2025; CVE assigned January 21, 2026.

Affected: All Claude Code versions prior to August 2025 and December 2025 patches. **Fix available — update to latest.**

Sources: [Check Point Research](https://research.checkpoint.com/2026/rce-and-api-token-exfiltration-through-claude-code-project-files-cve-2025-59536/) · [The Hacker News](https://thehackernews.com/2026/02/claude-code-flaws-allow-remote-code.html) · [NVD](https://nvd.nist.gov/vuln/detail/CVE-2025-59536)

### GitHub Copilot — CVE-2025-53773 & CVE-2025-64660
**Status: Patched (August 2025 Patch Tuesday, VS Code 17.14.12+)**

- **CVE-2025-53773** (CVSS 7.8): Copilot in Agent Mode can write to `.vscode/settings.json` without user approval. Prompt injection via malicious code comments, GitHub issues, or hidden Unicode characters enables `"chat.tools.autoApprove": true`, disabling all confirmation dialogs and enabling arbitrary shell command execution. **Wormable** — infected repos can propagate malicious instructions to new projects. Works against GPT-4.1, Claude Sonnet 4.0, and Gemini backends.
- **CVE-2025-64660**: Additional IDEsaster-class RCE; patched in the same release cycle.

Sources: [Embrace The Red](https://embracethered.com/blog/posts/2025/github-copilot-remote-code-execution-via-prompt-injection/) · [Persistent Security](https://www.persistent-security.net/post/part-iii-vscode-copilot-wormable-command-execution-via-prompt-injection) · [NVD](https://nvd.nist.gov/vuln/detail/CVE-2025-53773)

### Cursor IDE — CVE-2025-54135 "CurXecute", CVE-2025-54136 "MCPoison", CVE-2026-22708, and others
**Status: Mostly patched — update to v1.3.9+**

- **CVE-2025-54135 "CurXecute"**: Permission check inconsistency — creating a new `.cursor/mcp.json` doesn't require approval while editing one does. Attacker tricks agent into creating a malicious MCP config, triggering zero-click RCE.
- **CVE-2025-54136 "MCPoison"**: Once an MCP server is approved by name, Cursor trusts it unconditionally. Post-approval config modification results in a persistent silent backdoor.
- **CVE-2026-22708**: Indirect prompt injection RCE via shell built-ins (`export`, `typeset`), executable even with an empty allowlist. Fixed: Cursor now requires explicit user approval for commands the server-side parser cannot classify.
- Additional: CVE-2025-49150, CVE-2025-54130, CVE-2025-61590 (IDEsaster class).

Sources: [SC Media](https://www.scworld.com/news/cursor-vulnerability-enables-stealthy-rce-via-indirect-prompt-injection) · [Pillar Security](https://www.pillar.security/blog/new-vulnerability-in-github-copilot-and-cursor-how-hackers-can-weaponize-code-agents)

### IDEsaster — Systemic Vulnerability Class (~30 CVEs across 10+ products)
**Status: Partial patches by individual vendors; underlying class unresolved**

Disclosed December 2025 by researcher Ari Marzouk. Six-month research covering GitHub Copilot, Cursor, Windsurf, Kiro.dev, Zed.dev, Roo Code, JetBrains Junie, Cline, Gemini CLI, and Claude Code — **100% of tested tools were vulnerable**. AWS issued advisory AWS-2025-019.

Attack chain: Prompt Injection → Agent Tools → Base IDE config files (`.vscode/settings.json`, `.idea/workspace.xml`, Zed configs) → RCE or silent data exfiltration. Novel vector: agent reads `.env`/API key files and writes output to a JSON file with an attacker-controlled remote schema URL, causing the IDE to silently GET the attacker's server with embedded secrets.

Key CVEs: CVE-2025-53097, CVE-2025-58372 (Roo Code); CVE-2025-58335 (JetBrains Junie); CVE-2025-7656 (Chromium, re-weaponized against current Cursor/Windsurf).

Sources: [The Hacker News](https://thehackernews.com/2025/12/researchers-uncover-30-flaws-in-ai.html) · [Tom's Hardware](https://www.tomshardware.com/tech-industry/cyber-security/researchers-uncover-critical-ai-ide-flaws-exposing-developers-to-data-theft-and-rce) · [MaccariTA](https://maccarita.com/posts/idesaster/)

## Supply Chain

### "Slopsquatting" — AI Hallucinated Package Names
**Status: Active threat; systemic**

Named by Seth Larson (Python Software Foundation). USENIX Security 2025 research (576,000 code samples, 16 LLMs): ~20% of AI-recommended packages don't exist. Commercial LLMs hallucinate at ~5%; open-source models (CodeLlama, DeepSeek, WizardCoder, Mistral) at up to 38%. Critically, **43% of hallucinated names recur consistently**, making them reliable squat targets. Breakdown: 38% conflation of two real names, 51% pure fabrication, 13% typo variants.

Real-world precursor: malicious PyPI package `huggingface-cli` observed in early 2024 exploiting a name commonly hallucinated by LLMs.

Sources: [Trend Micro](https://www.trendmicro.com/vinfo/us/security/news/cybercrime-and-digital-threats/slopsquatting-when-ai-agents-hallucinate-malicious-packages) · [BleepingComputer](https://www.bleepingcomputer.com/news/security/ai-hallucinated-code-dependencies-become-new-supply-chain-risk/) · [Aikido](https://www.aikido.dev/blog/slopsquatting-ai-package-hallucination-attacks)

### Shai-Hulud npm Campaign — September 2025
**Status: Active attack — 25,000+ downstream projects compromised**

Trojanized 40+ npm packages including `@ctrl/tinycolor`. Injected controller uses TruffleHog to hunt for tokens and cloud credentials, validate npm/GitHub tokens, and abuse found credentials. Connected to prior "s1ngularity" campaign (August–November 2025), which harvested 2,349 credentials from 1,079 developer systems via compromised Nx packages. Attackers specifically targeted config files and auth tokens for Claude, Gemini, and Amazon Q.

Sources: [Dark Reading](https://www.darkreading.com/cyberattacks-data-breaches/supply-chain-worms-in-2026-what-shai-hulud-taught-attackers-and-how-to-prepare) · [Oligo Security](https://www.oligo.security/blog/the-hidden-risks-of-the-npm-supply-chain-attacks-ai-agents)

## Research & Advisories

### MCP Protocol Vulnerabilities

**mcp-server-git RCE Chain — CVE-2025-68145, CVE-2025-68143, CVE-2025-68144**
**Status: Disclosed — first documented real-world MCP RCE**

Three chained CVEs in Anthropic's `mcp-server-git`: path validation bypass (CVE-2025-68145), unrestricted `git_init` converting `.ssh/` into a git repo (CVE-2025-68143), and argument injection in `git_diff` (CVE-2025-68144). Combined with the Filesystem MCP server via malicious `.git/config`, achieves full RCE.

**MCP Inspector — CVE-2026-23744**
**Status: Patched in v0.14.1**

Unauthenticated proxy accepted connections without origin verification. Combined with DNS rebinding, attacker-controlled pages could issue commands to the victim's localhost Inspector, achieving RCE with user privileges.

**Additional MCP CVEs:**
- CVE-2026-27896: Official MCP Go SDK
- CVE-2026-26118: Azure MCP Server SSRF (CVSS 8.8)
- CVE-2026-0756: GitHub Kanban MCP Server RCE
- CVE-2026-0755: gemini-mcp-tool command injection
- Microsoft MarkItDown MCP Server: Unpatched SSRF enabling AWS EC2 metadata access (Microsoft classified as low risk)

**MCP Tool Poisoning / Rug Pull**
**Status: Actively exploited pattern**

Hidden instructions in MCP tool descriptions — invisible in UI but parsed by LLM. A tool need not even be called; being loaded into context is sufficient. Rug pull variant: clean definition served during approval, malicious version served thereafter. An OAuth flaw (related to CVE-2025-68144) in a Drift/Salesforce-integrated MCP server was actively exploited in early 2026 to steal OAuth tokens and access customer data.

AgentSeal scan of 1,808 MCP servers: 66% had security findings; 43% command injection; 33% unrestricted network access; 22% out-of-scope file access; 43% OAuth implementation flaws.

**Smithery Registry Path Traversal**
Path traversal in `smithery.yaml` (`dockerBuildPath: ".."`) let attackers build Docker images from the builder's home directory, leaking `~/.docker/config.json` including a Fly.io token — granting control over 3,000+ hosted MCP servers.

Sources: [Vulnerable MCP Project](https://vulnerablemcp.info/) · [Invariant Labs](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks) · [Palo Alto Unit42](https://unit42.paloaltonetworks.com/model-context-protocol-attack-vectors/) · [Elastic Security Labs](https://www.elastic.co/security-labs/mcp-tools-attack-defense-recommendations) · [CyberArk](https://www.cyberark.com/resources/threat-research-blog/poison-everywhere-no-output-from-your-mcp-server-is-safe) · [authzed](https://authzed.com/blog/timeline-mcp-breaches)
