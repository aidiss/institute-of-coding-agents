## Headlines

- **Flowise CVSS 10.0 now actively weaponized:** CVE-2025-59528, a code injection flaw in Flowise's `CustomMCP` node, moved from disclosed-but-unpatched to actively exploited on April 7, 2026. Despite patches being available since version 3.0.6 (disclosed 6+ months ago), scanning shows 12,000–15,000 internet-exposed instances. Exploitation observed from a Starlink IP, suggesting opportunistic scanning.
- **Axios npm compromised by Sapphire Sleet:** On March 31, 2026, North Korean state actor Sapphire Sleet hijacked the npm account of Axios's lead maintainer via a social engineering + RAT campaign, publishing two poisoned versions (1.14.1, 0.30.4) with a cross-platform second-stage RAT. The malicious versions were live for ~3 hours; 70M+/week download footprint made this the highest-reach supply chain incident of 2026.
- **OpenCode zero-auth RCE (CVE-2026-22812):** Network-reachable unauthenticated RCE in a 71k-star AI coding tool allows full credential sweep: AWS/GCP/Azure tokens, SSH private keys, GitHub tokens, and `.env` API keys. No authentication required.

## Incidents & Vulnerabilities
_As of 2026-04-15_

### Flowise — CVSS 10.0 RCE Under Active Exploitation

**CVE:** CVE-2025-59528 | **CVSS:** 10.0 | **Status:** Patched in v3.0.6 — exploitation ongoing against unpatched instances | **First exploitation observed:** April 7, 2026

Flowise's `CustomMCP` node accepts user-supplied JavaScript configuration and passes it to the Node.js runtime with no validation. The flaw allows access to dangerous modules (`child_process`, `fs`) through the full Node.js runtime privilege context.

**Exploitation detail:**
- Attacker supplies a crafted `mcpServerConfig` string that calls `require('child_process')` and executes arbitrary OS commands.
- Full system compromise: read/write filesystem, execute commands, exfiltrate credentials from environment.
- VulnCheck first observed active exploitation on April 7, 2026 from a single Starlink IP; broader scanning likely underway.
- Estimated 12,000–15,000 instances remain internet-exposed and unpatched.

**Affected versions:** Flowise < 3.0.6  
**Fix:** Upgrade to Flowise ≥ 3.0.6  
**Priority:** Critical — CVSS 10.0, actively exploited

*Sources: [The Hacker News](https://thehackernews.com/2026/04/flowise-ai-agent-builder-under-active.html) | [BleepingComputer](https://www.bleepingcomputer.com/news/security/max-severity-flowise-rce-vulnerability-now-exploited-in-attacks/) | [Security Affairs](https://securityaffairs.com/190471/security/attackers-exploit-critical-flowise-flaw-cve-2025-59528-for-remote-code-execution.html)*

### OpenCode — Zero-Auth RCE and Credential Sweep

**CVE:** CVE-2026-22812 | **CVSS:** Not yet scored | **Status:** Disclosed March 2026 — patch status unconfirmed | **Tool:** OpenCode (71k GitHub stars)

A researcher demonstrated unauthenticated RCE in OpenCode via the network or via a malicious website. Successful exploitation yields:
- AWS, GCP, and Azure credential files
- SSH private keys
- GitHub tokens
- `.env` files containing application API keys

No authentication, no user interaction required beyond the tool being open and reachable. The attack surface includes both local network adversaries and drive-by web exploitation.

*Source: [Medium — CVE-2026-22812 Writeup](https://medium.com/@dhxrxx/cve-2026-22812-how-i-got-rce-on-a-71k-star-ai-coding-tool-with-zero-authentication-7524fbc3317f) | [Dark Reading](https://www.darkreading.com/vulnerabilities-threats/rce-flaw-ai-coding-tool-supply-chain-risk)*

### mcp-server-git — Triple-CVE RCE Chain

**CVEs:** CVE-2025-68145, CVE-2025-68143, CVE-2025-68144 | **Product:** Anthropic `mcp-server-git` + Filesystem MCP server | **Status:** Patch availability unconfirmed as of report date

Three chained vulnerabilities in Anthropic's own Git MCP server combine with the standard Filesystem MCP server to achieve full remote code execution via a malicious `.git/config` file. The chain exploits:
1. Insufficient validation of repository-supplied configuration values
2. Unsafe passthrough of git config fields to shell invocations
3. Cross-server trust assumptions in the Filesystem MCP server

An attacker controlling a Git repository (e.g., via a submitted PR or a malicious clone URL) can trigger RCE when an AI coding agent performs standard git operations on the repo.

*Source: [MCP Security 2026 — 30 CVEs in 60 Days](https://www.heyuan110.com/posts/ai/2026-03-10-mcp-security-2026/)*

## Supply Chain
_As of 2026-04-15_

### Axios npm Supply Chain Compromise — Sapphire Sleet (DPRK)

**Date:** March 31, 2026, 00:21–03:15 UTC (~3 hours live)  
**Affected versions:** axios@1.14.1, axios@0.30.4  
**Attribution:** Sapphire Sleet (North Korean state actor) — Microsoft Threat Intelligence  
**Status:** Malicious versions unpublished; clean versions restored; post-mortems published

Axios — an HTTP client with **100M+ weekly downloads** (npm) — was compromised when Sapphire Sleet gained access to the lead maintainer's npm account via a targeted social engineering and RAT campaign.

**Attack chain:**
1. Maintainer's PC compromised via social engineering; RAT provides npm account credentials.
2. Two malicious versions published adding `plain-crypto-js@4.2.1` as a post-install dependency.
3. The fake dependency executes at install time via npm's `postinstall` hook — no import in application code required.
4. `plain-crypto-js` beacons to C2 and downloads a second-stage cross-platform RAT.
5. Three parallel RAT implementations (Windows, macOS, Linux) deployed based on OS — identical C2 protocol, command structure, and beacon behavior.

**Why this matters beyond Axios:** This is the first confirmed large-scale DPRK supply chain attack against a mainstream JavaScript package. Prior Lazarus/Sapphire Sleet npm attacks targeted crypto/DeFi developers specifically. Axios is general infrastructure — every Node.js project is a potential target.

**Detection and remediation:**
- Check install logs for `plain-crypto-js` as a dependency.
- Any system that ran `npm install` on the malicious versions between 00:21–03:15 UTC March 31, 2026 should be treated as compromised.
- Microsoft Security Blog provides full IOC list.
- Rotate all credentials (npm tokens, cloud keys, SSH keys) on affected systems.

*Sources: [Microsoft Security Blog](https://www.microsoft.com/en-us/security/blog/2026/04/01/mitigating-the-axios-npm-supply-chain-compromise/) | [The Hacker News](https://thehackernews.com/2026/03/axios-supply-chain-attack-pushes-cross.html) | [Elastic Security Labs](https://www.elastic.co/security-labs/axios-one-rat-to-rule-them-all) | [SANS Institute](https://www.sans.org/blog/axios-npm-supply-chain-compromise-malicious-packages-remote-access-trojan) | [Datadog Security Labs](https://securitylabs.datadoghq.com/articles/axios-npm-supply-chain-compromise/)*

### AI Agent Security — "The Lethal Trifecta" (2026 Framework)

The security research community has converged on a risk model called the **Lethal Trifecta** for agentic AI: an agent becomes a high-severity attack target when it simultaneously possesses:
1. **Access to private/sensitive data** (credentials, codebases, customer data)
2. **Ability to process untrusted external content** (web pages, user input, repo files)
3. **Capability to communicate externally** (API calls, email, Git pushes)

All three conditions are met by default in every major AI coding agent. The Trifecta explains why Copilot, Cursor, Claude Code, and Devin are all systematically vulnerable to the same underlying attack pattern despite different implementations.

*Source: [Airia — AI Security in 2026: Prompt Injection and the Lethal Trifecta](https://airia.com/ai-security-in-2026-prompt-injection-the-lethal-trifecta-and-how-to-defend/)*

## Analyst Note

The Axios compromise is the most significant new development since the March 7 brief and warrants immediate attention from any team using Node.js. Three aspects make it structurally different from prior npm attacks:

**1. Target selection has shifted upstream.** Prior Lazarus/Sapphire Sleet npm campaigns targeted niche crypto packages to hit a specific developer segment. Axios is general infrastructure — the malicious payload reached any Node.js developer who ran `npm install` during a 3-hour window. DPRK actors are demonstrating willingness to compromise universal dependencies for maximum breadth.

**2. The human is the weakest link, not the code.** Initial access came through social engineering the maintainer's machine, not a code vulnerability. Package signing, 2FA on npm accounts, and trusted publisher attestations would all have helped here — but none were in place. CISA's guidance on MFA for package registry accounts remains widely ignored.

**3. Post-install hooks are a blind spot in dependency review.** The malicious payload used `postinstall` — a standard npm lifecycle script. Most developers reviewing `package.json` diffs don't audit transitive dependency install hooks. Tooling like Socket.dev and StepSecurity flags this; adoption remains low.

For AI coding agent deployments specifically: every agent that can run `npm install` on its own behalf in response to AI-suggested dependency additions is one prompt injection away from installing a Flowise-style or Axios-style malicious package. Agents must not have unrestricted access to package registries without human review of dependency changes.

**Recommended immediate actions:**
- Audit all Node.js projects for `plain-crypto-js` in lock files; rotate credentials if found.
- Upgrade Flowise to ≥ 3.0.6 immediately — CVSS 10.0 and actively exploited.
- Confirm Claude Code is on a patched version; audit `.claude/settings.json` in all cloned repositories.
- Treat MCP server definitions as untrusted code: audit before installation and monitor for post-install mutations.
- Enable socket-level monitoring or equivalent on all AI coding agent environments to detect unexpected outbound connections.
- Apply npm `--ignore-scripts` in CI pipelines as a default; whitelist specific packages that require install scripts.
