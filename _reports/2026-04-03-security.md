# Security Report — 2026-04-03

Coverage period: approximately February 2025 – April 2026.

---

## Incidents & Vulnerabilities

### Claude Code (Anthropic)

**CVE-2025-59536** — RCE via Hook Injection in Project Settings
- CVSS: 8.7 (High)
- Disclosed: February 25, 2026 (Check Point Research)
- Status: **Patched** in version 1.0.111 (October 2025); publicly disclosed post-patch
- Affected: Claude Code < 1.0.111
- A flaw in the startup trust dialog allowed malicious `Hook` commands embedded in a repository's `.claude/settings.json` to execute arbitrary shell commands before the user accepted the trust prompt. The `enableAllProjectMcpServers` option in `.mcp.json` could silently activate untrusted MCP servers. Cloning a malicious repo and opening it in Claude Code was sufficient to trigger RCE.

**CVE-2026-21852** — API Key Exfiltration via Environment Variable Manipulation
- CVSS: 5.3 (Medium)
- Disclosed: February 25, 2026 (Check Point Research, same report)
- Status: **Patched** in version 2.0.65 (January 2026)
- A malicious repository could manipulate environment variables during project load to exfiltrate Anthropic API keys to an attacker-controlled endpoint.

**CVE-2026-25725** — Sandbox Bypass
- Disclosed: Early 2026
- Status: **Patched**
- Enabled unauthorized filesystem access by bypassing Claude Code's sandbox boundaries.

### GitHub Copilot

**CamoLeak** — Silent Repository Exfiltration via Prompt Injection in Copilot Chat
- CVSS: 9.6 (Critical); no CVE assigned at disclosure
- Disclosed: ~June–October 2025 (Legit Security / Omer Mayraz)
- Status: **Patched** — GitHub disabled image rendering in Copilot Chat
- Attackers injected hidden instructions into pull request descriptions using invisible Markdown comments. Copilot Chat parsed and acted on these, encoding stolen data character-by-character as unique pre-signed image URLs routed through GitHub's CDN to bypass CSP. Could leak entire private repository contents and secrets.

**CVE-2025-53773** — RCE via Prompt Injection ("ZombAI" worm vector)
- CVSS: 7.8 (High)
- Disclosed: June 29, 2025; Patched: August 2025
- Status: **Patched**
- Malicious instructions in source code comments, GitHub issues, or web pages directed Copilot to set `"chat.tools.autoApprove": true` in `.vscode/settings.json`, disabling approval gates. Subsequent shell commands executed without confirmation. Invisible Unicode characters were used to hide the payload from code review.

**RoguePilot** — GitHub Token Exfiltration via Codespaces
- No CVE assigned at time of reporting
- Disclosed: February 2026 (Orca Security)
- Status: **Patched** by Microsoft following responsible disclosure
- A hidden HTML comment in a GitHub Issue was automatically fed to Copilot when a Codespace was opened. The injected prompt directed Copilot to check out a crafted pull request containing a symbolic link, then exfiltrate the `GITHUB_TOKEN` via a remote JSON `$schema` fetch. The stolen token granted full read/write repository access.

**CVE-2025-62449** — Path Traversal
- CVSS: 6.8 (Medium)
- Status: **Addressed**
- Path traversal enabling unauthorized file access.

**CVE-2025-62453** — Improper AI Output Validation
- CVSS: 5.0
- Status: **Addressed**
- Allowed code injection via unvalidated AI outputs.

### Cursor IDE

**CVE-2025-59944** — Persistent RCE via MCP Trust Bypass ("MCPoison")
- Disclosed: ~August 2025 (Check Point Research)
- Status: **Patched** in Cursor v1.7
- Cursor's file protection checks were case-sensitive while Windows/macOS filesystems are not. Prompt injection could direct Cursor to write `.cUrSoR/mcp.json` — treated as a new unprotected file by Cursor, but identical to the real config file on disk. Persisted across IDE restarts, granting durable RCE.

**CVE-2025-54135 / CVE-2025-54136** — RCE via MCP Auto-Start ("CurXecute")
- Disclosed: 2025 (Cato Networks)
- Status: **Patched**
- Public prompt injection payloads triggered automatic MCP server launch in Cursor, achieving local shell execution without user approval.

Additional Cursor CVEs patched: CVE-2025-49150, CVE-2025-54130, CVE-2025-61590.

### IDEsaster — Broad AI IDE Vulnerability Class

- Disclosed: December 2025 (Ari Marzouk / MaccariTA)
- **30+ vulnerabilities, 24 CVEs assigned** across Cursor, Windsurf, Kiro.dev, GitHub Copilot, Zed.dev, Roo Code, Junie, Cline, and others
- **100% of tested AI IDEs were vulnerable**
- Attack classes:
  - Remote JSON Schema attacks — force IDE to fetch attacker-controlled schema, leaking data
  - IDE Settings Overwrite via prompt injection (`.vscode/settings.json`, `.cursor/mcp.json`)
  - Multi-Root Workspace attacks — load writable executable files; auto-run malicious code
  - Data exfiltration via image loading and Markdown rendering
- AWS issued immediate security advisories following disclosure.

### Devin AI (Cognition)

- Reported: April 6, 2025 (Johann Rehberger / Embrace the Red)
- Status: **Acknowledged; no confirmed public patch** — vendor unresponsive beyond 120 days
- Devin's asynchronous agent has no protection against prompt injection. Attackers can inject payloads via websites Devin browses, causing it to execute shell commands (`curl`, `wget`, Python), exfiltrate secrets via Markdown image fetches, or download and run malware within the DevBox sandbox.

### CrewAI (Agentic Framework)

- CERT/CC Advisory **VU#221883** — four chained CVEs (2026)
- **CVE-2026-2275** — Code execution via SandboxPython fallback
- **CVE-2026-2285** — Arbitrary local file read (JSON loader, no path validation)
- **CVE-2026-2286** — SSRF via RAG search tools (no URL validation)
- **CVE-2026-2287** — RCE when Docker is killed and sandbox falls back to unsafe mode
- Status: **Partially unpatched** — vendor committed to blocking `ctypes`; full remediation not yet released
- Exploitation chain: prompt injection → file read + SSRF for credential theft → RCE for full system compromise.

## Supply Chain

### LiteLLM — TeamPCP Supply Chain Campaign (March 24, 2026)

- Attacker group: **TeamPCP** (nation-state-linked or organized criminal)
- Affected: `litellm` on PyPI, versions **1.82.7** and **1.82.8**
- Exposure window: ~10:39–16:00 UTC on March 24, 2026 (~5.5 hours)
- Status: **Quarantined by PyPI**; malicious versions removed
- TeamPCP stole PyPI publish credentials by compromising a Trivy GitHub Action in LiteLLM's CI/CD pipeline. Malware in `proxy_server.py` exfiltrated SSH keys, cloud tokens, API keys, and Kubernetes secrets to `models.litellm[.]cloud`. LiteLLM proxies requests to OpenAI, Anthropic, and other LLM APIs, making developer and production AI stacks the primary targets. Developers using unpinned `pip install litellm` during the window were compromised.
- Broader campaign context: TeamPCP previously compromised Trivy, Checkmarx, and Telnyx npm packages in a cascading attack targeting the AI/security tooling ecosystem.

### Slopsquatting — AI-Hallucinated Package Dependencies

- **Research: "We Have a Package for You!" (USENIX Security 2025)**
  - 576,000 code samples across 16 LLMs
  - **19.7% of suggested packages were hallucinations** — 205,474 unique fake package names
  - Open-source models: ~21.7% hallucination rate; commercial models (GPT-4): ~5.2%
  - 43% of hallucinated names are consistent across repeated prompts — enabling pre-registration attacks
  - 8.7% of hallucinated Python packages are valid JavaScript packages — cross-ecosystem squatting opportunity
  - Term coined by Python Software Foundation Developer-in-Residence Seth Larson

- **Real-world validation (Bar Lanyado / Lasso Security):**
  - LLMs consistently hallucinated `huggingface-cli` as a Python package (does not exist on PyPI)
  - Lanyado registered an empty package as PoC; downloaded **30,000+ times within 3 months**
  - Alibaba and other major companies copy-pasted AI-generated instructions referencing the hallucinated package into public documentation, creating cascading downstream risk

- Attackers pre-register hallucinated package names; malicious packages execute on `pip install` / `npm install` via install hooks with no further user interaction.

### Rule Files Backdoor — Poisoned AI Configuration Files (March 2025)

- Disclosed: March 18, 2025 (Pillar Security)
- Affected: GitHub Copilot (`.github/copilot-instructions.md`) and Cursor (`.cursor/rules`)
- Status: **Both vendors declined to patch**, citing developer responsibility
- Configuration/rule files shared across organizations can be poisoned with hidden Unicode characters or covert instructions. When developers import poisoned rule files, the AI coding agent silently injects backdoors, weak crypto, or secret-leaking patterns into all generated code. A single poisoned file can propagate through an entire organization's codebase.

## Research & Advisories

### CVE-2025-6514 — mcp-remote OS Command Injection (Critical RCE)

- CVSS: 9.6 (Critical)
- Discovered: JFrog Security Research
- Affected: mcp-remote versions 0.0.5–0.1.15 (437,000+ downloads at disclosure)
- Fixed in: **version 0.1.16**
- Status: **Patched**
- `mcp-remote` is a widely used proxy enabling Claude Desktop and other LLM hosts to connect to remote MCP servers. When initiating OAuth flow with a malicious MCP server, the server can return a crafted `authorization_endpoint` URL containing shell metacharacters, causing arbitrary OS command execution on the developer's machine. No user interaction required beyond connecting to a malicious MCP server.

### CVE-2025-68143 / 68144 / 68145 — Anthropic mcp-server-git RCE Chain

- Disclosed: January 2026 (SecurityWeek, The Hacker News)
- Reported to Anthropic: June 2025; patched: December 2025 (silent, pre-public-disclosure)
- Status: **Patched** (version 2025.12.18 — `git_init` tool removed)
- Three-CVE chain:
  - **CVE-2025-68145** — Path validation bypass: `--repository` flag restriction not enforced on subsequent tool calls
  - **CVE-2025-68143** — Unrestricted `git_init`: any directory including `~/.ssh` could be initialized as a git repo
  - **CVE-2025-68144** — Argument injection in `git_diff`/`git_checkout`: user-controlled arguments passed unsanitized to GitPython
- RCE chain: bypass path check → init git repo in `~/.ssh` → write malicious `.git/config` with shell hook via Filesystem MCP server → trigger hook via `git_diff` injection → arbitrary code execution.

---

### CVE-2026-23744 — MCPJam Inspector Unauthenticated RCE

- CVSS: 9.8 (Critical)
- Affected: MCPJam Inspector ≤ 1.4.2
- Fixed in: **version 1.4.3** (PoC exploit publicly available)
- Status: **Patched**
- MCPJam Inspector bound its management API to `0.0.0.0` by default with no authentication. The `/api/mcp/connect` endpoint executed arbitrary commands via `child_process.spawn` with no auth check. Any network-adjacent attacker could achieve immediate RCE as the running user.

### MCPwned — Azure MCP Server RCE / Cloud Takeover

- Researcher: Ariel Simon (Token Security); to be presented at RSAC 2026
- Patch status: Disclosed to Microsoft; not fully public as of April 2026
- A vulnerability chain in Microsoft's official Azure MCP server allows unauthenticated network-adjacent attackers to achieve RCE, then harvest Azure credentials. Result: full compromise of the victim organization's Azure and Entra ID tenant.

### MCP Ecosystem-Wide Statistics (2025–2026)

Research from Invariant Labs, Palo Alto Unit 42, and Red Hat found:
- **43%** of surveyed MCP servers contain command injection vulnerabilities
- **43%** have OAuth authentication flow flaws
- **33%** allow unrestricted network access enabling data exfiltration or malware download
- Invariant Labs demonstrated silent exfiltration of entire WhatsApp conversation history via tool poisoning against a legitimate `whatsapp-mcp` server
- Real-world incident: A Cursor agent with privileged Supabase `service-role` access processed user-supplied support tickets containing SQL injection, leaking integration tokens into a public support thread

### OWASP Top 10 for Agentic Applications 2026

- Published: December 9, 2025
- Top risks for agentic/coding agent environments:
  1. Prompt injection (direct and indirect) — ranked #1
  2. Excessive agency — agents granted more permissions than necessary
  3. Memory poisoning — corrupting agent long-term memory/context
  4. Tool/plugin abuse
  5. Human-agent trust exploitation
  6. Rogue agents (misalignment, concealment, self-directed action)
  7. Sensitive information disclosure
  8. Inter-agent trust boundary failures in multi-agent systems
- Key finding: "When an agent is allowed to act 'as the user,' you effectively extend that user's blast radius to any system the agent can talk to."

### Trail of Bits — Prompt Injection to RCE Research

- Published: October 22, 2025
- Documented systematic escalation from prompt injection (model-level) to infrastructure-level RCE. Prompt injection now functions as an infrastructure attack, not just model output manipulation — injections trigger tool calls, file writes, and shell execution through the agent's legitimate capabilities.

### Tenzai / Pixee — 69 Vulnerabilities Across 5 AI Coding Platforms

- Published: January 21, 2026
- Identical coding prompts issued to 5 platforms (including Devin) yielded 69 total vulnerabilities, 6 critical. A significant proportion were missed by static analysis scanners because the vulnerabilities appear in semantically correct but logically unsafe patterns.
