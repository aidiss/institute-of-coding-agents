## Incidents & Vulnerabilities

### Cursor IDE — Triple CVE Chain (CVSS 9.8)

Pillar Security researchers disclosed a three-CVE chain enabling full RCE on developer machines via indirect prompt injection:

| CVE | Type | CVSS | Status |
|-----|------|------|--------|
| CVE-2026-22708 | Shell built-in bypass (`export`/`typeset` executed without user approval) | 9.8 | Patched |
| CVE-2026-26268 | Git hook escape | — | Patched |
| CVE-2026-21523 | TOCTOU race condition | — | Patched |

CVE-2026-22708 is exploitable in both zero-click and one-click scenarios: poisoning the shell execution environment via syntax manipulation causes commands to execute before the user approval dialog appears. Cursor is noted as one of the most-targeted AI coding tools — 7 of 21 documented multi-stage promptware attacks in Q1 2026 targeted this product category.

## Supply Chain

### LiteLLM Supply Chain Attack — TeamPCP (March 24, 2026)

**Actively exploited.** On March 24, 2026, LiteLLM versions **1.82.7** and **1.82.8** were published to PyPI containing a multi-stage credential stealer. Malicious window: **10:39 UTC to 16:00 UTC** (approximately 5.5 hours).

Payload targets: SSH keys, cloud provider tokens (AWS/GCP/Azure), Kubernetes secrets, cryptocurrency wallets, `.env` files. Carried out by **TeamPCP**, a threat group chaining open-source compromises since late 2025.

Many developers were not direct LiteLLM users — the package was pulled in as a **transitive dependency** through AI agent frameworks, MCP servers, and LLM orchestration tools. Any developer running `pip install litellm` without version pinning in the exposure window is considered affected.

---

### Slopsquatting — Weaponized Hallucinated Dependencies

AI models hallucinate plausible-sounding but nonexistent package names; attackers register these names with malicious payloads. Research across 756,000 AI-generated code samples found **~20% included hallucinated package names**. Sonatype's 2026 State of the Software Supply Chain found GPT-5 hallucinated **28% of dependency upgrade recommendations**.

Key statistics:
- 43% of the same hallucinated package names are suggested consistently across 10+ separate prompt runs — making targets predictable and pre-registerable
- July 2025–January 2026: 128 tracked phantom packages accumulated **121,539 downloads** (`openapi-generator-cli` alone: 48,356)
- Study of 117,062 dependency changes across 33,596 agent-authored PRs: AI agents modify dependency manifests far more frequently than humans (Copilot: 33.5%, Devin: 29.6%, OpenAI Codex: 23.6%, Cursor: 10.6%, Claude Code: 2.7%)
- Newer variant — **Automated Dependency Side-Loading**: a compromised IDE extension intercepts AI suggestions and silently substitutes attacker-controlled package names pre-commit, invisible to standard tooling

Prompt injection in Claude-powered GitHub Actions issue triage led to a compromised npm package that silently installed a persistent daemon on approximately **4,000 developer machines**, exposing credentials and SSH keys.

### Coding Agents and the Expanded Supply Chain Attack Surface (March 2026)

Security Boulevard analysis (March 2026) notes that autonomous coding agents — Cursor, Claude Code, GitHub Copilot Workspace — are no longer advisory tools but **autonomous participants** in the SDLC: generating code, selecting dependencies, executing build steps, and pushing changes at speeds no human reviewer can match in real time. A single hallucinated package name, poisoned tool call, or injected instruction interpreted as executable can propagate through the build and into production.

Amazon Q Developer near-miss (July 2025): attacker injected malicious instruction into the official Amazon Q Developer VS Code extension — **964,000 installations**, 2-day exposure — directing the AI to wipe systems and delete cloud resources. A syntax error in the payload prevented damage. Described as "a typo is the current margin of safety for AI coding tool supply chains."

## Research & Advisories

### MCP Ecosystem Attack Surface — Scale (Early 2026)

From vulnerablemcp.info (50 documented vulnerabilities, 13 critical, 32 researchers) and independent research across 67,057 MCP servers across 6 registries:
- **43%** of MCP servers contain command injection vulnerabilities
- **36.7%** vulnerable to SSRF (Server-Side Request Forgery)
- **33%** allow unrestricted network access
- **22%** allow filesystem access outside intended scope
- **5%** of open-source MCP servers seeded with tool poisoning attacks
- **1,862** MCP servers connected to the public internet, almost all without authentication
- **492** MCP servers with no client authentication or traffic encryption

MCP protocol design flaws: session identifiers mandated in URLs (violates security best practices); minimal authentication guidance; no required message signing or verification.

**Active attack categories:** Tool poisoning (malicious instructions in tool descriptions), "rug pull" attacks (tools modify their definitions post-approval), hidden Unicode injection (invisible characters containing commands), context poisoning, insecure transport.

### AI Security Readiness Gap (Cisco State of AI Security 2026)

While most organizations planned to deploy agentic AI in 2026, **only 29% reported being prepared to secure those deployments** — a 71% gap. The gap exists because agentic attacks don't resemble what existing security tools are designed to catch.

---

### Prompt Injection as Universal RCE Entry Point

OWASP assessment: prompt injection appears in over **73% of production AI deployments** audited. Research confirms that adding a generic security reminder in prompts improved secure and correct code generation from 56% to **66%** (Claude Opus 4.5 Thinking). Trail of Bits documented the full prompt injection → RCE chain in October 2025; the pattern has been replicated against every major AI IDE.

### Key Observations

1. **LiteLLM is the week's most acute threat** — an actively exploited supply chain attack affecting a widely used transitive dependency. Developers should audit `pip freeze` output for versions 1.82.7 and 1.82.8.
2. **MCP remains the dominant new attack surface** — nearly every serious disclosure involves MCP as either an amplifier for prompt injection or a direct RCE vector. 43% command injection prevalence across surveyed servers is not theoretical.
3. **AI-authored CVEs are accelerating** — 35 in March 2026 alone vs. 6 in January; detection methodology is immature and real numbers are likely 5–10× higher.
4. **Slopsquatting is operational at scale** — 121,000+ downloads of phantom packages in 6 months; automated side-loading is the next evolution of this attack class.
5. **The organizational readiness gap is large** — 71% of organizations deploying agentic AI are not prepared to secure it, per Cisco's 2026 assessment.
