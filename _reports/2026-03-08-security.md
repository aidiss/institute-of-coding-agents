## Headlines

- **WeKnora (Tencent) hit by a cluster of 5 CVEs** (March 6–7), including a CVSS 10.0 MCP stdio command injection (CVE-2026-30861) and a SQL injection RCE bypass (CVE-2026-30860). Patch available in v0.2.10.
- **MS-Agent (ModelScope) command injection** (CVE-2026-2256) publicly disclosed March 2 with no vendor fix; PoC is available, raising active exploitation risk for any ms-agent deployment using shell tools.
- **OpenCode** (Anomaly Co.) inadvertently exposed an unauthenticated HTTP server allowing arbitrary shell execution from any website (CVE-2026-22812); patched in v1.0.216.
- **OpenClaw/ClawHavoc campaign** continues to grow — latest scans confirm 824+ malicious skills; infostealers have evolved to harvest AI agent identity files, not just browser credentials.

## Incidents & Vulnerabilities
_As of 2026-03-08_

### WeKnora (Tencent) — Five CVEs Published March 6–7

WeKnora is Tencent's LLM-powered framework for deep document understanding and semantic retrieval. Five advisories were filed in two days.

| CVE | CVSS | Type | Affected Versions | Status |
|-----|------|------|-------------------|--------|
| **CVE-2026-30861** | **10.0** | MCP stdio command injection bypass | 0.2.5 – < 0.2.10 | **Patched in v0.2.10** |
| **CVE-2026-30860** | Critical | RCE via SQL injection bypass in AI Database Query Tool | — | Patched |
| **CVE-2026-30859** | High | Broken access control — cross-tenant data exposure | — | Patched |
| **CVE-2026-30858** | High | DNS rebinding / SSRF via `web_fetch` tool — exposes internal resources | — | Patched |
| **CVE-2026-30856** | Medium | Tool name collision enabling indirect prompt injection; malicious remote MCP server hijacks tool execution | < 0.3.0 | Patched in v0.3.0 |

**CVE-2026-30861 technical detail:** The application whitelists commands (`npx`, `uvx`) and blacklists dangerous arguments, but the validation can be bypassed using the `-p` flag with `npx node`. Any registered user (registration is unrestricted) can trigger the injection to execute arbitrary OS commands under the WeKnora application's privileges. No public PoC at time of writing.

**Analyst note:** The breadth of these CVEs — SSRF, tenant isolation failures, prompt injection, SQL injection, and command injection — suggests a security review was overdue. The DNS rebinding SSRF (CVE-2026-30858) in a web-fetching tool is a pattern that recurs across the MCP ecosystem (compare: Microsoft MarkItDown SSRF, previously unremediated).

*Sources: [TheHackerWire — CVE-2026-30861](https://www.thehackerwire.com/weknora-rce-cve-2026-30861-unauthenticated-command-injection/) | [Vulnerability-Lookup](https://vulnerability.circl.lu/vuln/cve-2026-30861) | [GitLab Advisory CVE-2026-30247 (SSRF)](https://advisories.gitlab.com/pkg/golang/github.com/tencent/weknora/CVE-2026-30247/)*

### CVE-2026-2256 — ModelScope MS-Agent Shell Tool Command Injection

**Disclosed:** March 2, 2026 | **CVSS:** 9.0 (Critical) | **Vendor fix:** None as of report date | **PoC:** Public

MS-Agent is ModelScope's open-source AI agent framework. CVE-2026-2256 affects the Shell tool, which executes shell commands at the agent's direction. The "safety checks" use a denylist-based pattern that is bypassable against real shell parsing semantics.

- **Affected versions:** ms-agent ≤ v1.6.0rc1
- **Exploitation:** Attacker-controlled input can influence which commands the Shell tool executes; bypassing the denylist achieves arbitrary OS command execution in the agent's runtime context.
- **Assigned:** Vulnerability Note VU#431821 (CERT/CC)
- **Status:** **No patch released.** Vendor was notified January 15, 2026; public disclosure March 2, 2026. With a public PoC available, active exploitation risk is elevated.

**Recommended mitigations (vendor):** Restrict MS-Agent to sandboxed/trusted environments; disable or limit shell command execution; replace denylist filters with strict allowlists; enforce least-privilege; isolate with containers or VMs.

*Sources: [SecQube — CVE-2026-2256](https://www.secqube.com/blog/ms-agent-vulnerability-cve-2026-2256-exposes-ai-agents-to-remote-hijacking-risks) | [CyberPress — MS-Agent Vulnerability](https://cyberpress.org/critical-ms-agent-vulnerability-allows-attackers-to-hijack-ai-agents-and-gain-full-system-control/)*

### CVE-2026-22812 — OpenCode Unauthenticated HTTP Server (Arbitrary Command Execution)

**Published:** January 12, 2026 | **CVSS:** 8.8 (High) | **CWE:** 306, 749, 942 | **Fix:** v1.0.216

OpenCode (Anomaly Co.) is an open-source AI coding agent distributed as an npm package. Prior to version 1.0.216, OpenCode automatically started an unauthenticated HTTP server on localhost with permissive CORS, exposing endpoints for shell command execution, terminal session creation, and arbitrary file reads.

- **Attack surface:** Any website the developer visits can silently send cross-origin requests to the local server and execute shell commands with the developer's full OS privileges. No user interaction beyond having OpenCode running is required.
- **Exploitation in wild:** No confirmed in-the-wild exploitation reported, but ease of exploitation (web page visit → arbitrary shell) warrants prompt patching.
- **Fix:** `npm install opencode-ai@1.0.216`

*Sources: [NVD — CVE-2026-22812](https://nvd.nist.gov/vuln/detail/CVE-2026-22812) | [GitLab Advisory](https://advisories.gitlab.com/pkg/npm/opencode-ai/CVE-2026-22812/) | [Sentinel One](https://www.sentinelone.com/vulnerability-database/cve-2026-22812/)*

### "ClawJacked" — Website-to-Local-Agent Hijacking in OpenClaw

**CVE:** Not assigned as of report date | **Disclosed:** February 2026 | **Fix:** OpenClaw v2026.2.25

Oasis Security identified that OpenClaw incorrectly trusted localhost connections without verifying their origin, failing to account for the fact that websites can also originate connections from localhost. Any malicious website visited by an OpenClaw user could hijack the local agent via WebSocket without authentication.

The OpenClaw team rated this high severity and patched within 24 hours. Users on earlier versions remain vulnerable.

*Source: [The Hacker News — ClawJacked](https://thehackernews.com/2026/02/clawjacked-flaw-lets-malicious-sites.html)*

## Supply Chain
_As of 2026-03-08_

### OpenClaw / ClawHavoc — Ongoing AI Agent Skills Campaign

The ClawHavoc supply chain campaign targeting OpenClaw's ClawHub marketplace has grown significantly since initial disclosure.

**OpenClaw background:** OpenClaw is an open-source AI agent framework (215,000+ GitHub stars) that allows users to install third-party "skills" from ClawHub. Skills execute within the agent's runtime, which has broad system access: shell execution, file read/write, email, web browsing, OAuth-connected services. The barrier to publishing a skill is a SKILL.md Markdown file and a week-old GitHub account — no code signing, no security review, no sandbox by default.

**Campaign scope (as of latest scan, February 16, 2026):**

| Source | Confirmed Malicious Skills | Total Scanned | Rate |
|--------|---------------------------|---------------|------|
| Koi Security (initial) | 335 (ClawHavoc campaign) + 341 total | 2,857 | ~12% |
| Latest scan | **824** | 10,700+ | **~8%** |
| Bitdefender estimate | ~900 | — | ~20% |

**Attack techniques used in ClawHavoc:**
1. **Prompt injection** embedded in SKILL.md descriptor files — treated as trusted instructions by the agent
2. **Hidden reverse shell scripts** providing persistent remote access
3. **Token exfiltration** via CVE-2026-25253 (CVSS 8.8, patched in OpenClaw v2026.1.29)

**Malware delivered:** Atomic macOS infostealer (distributed via 39 skills on ClawHub and SkillsMP, per Trend Micro). Payloads increasingly hosted on lookalike OpenClaw websites to bypass VirusTotal scanning.

**New development — Infostealer targeting AI agent identity:**
Hudson Rock disclosed a confirmed case of infostealer exfiltrating a victim's complete OpenClaw identity. Stolen files included:
- `openclaw.json` — gateway token for API access
- `device.json` — cryptographic keys for secure pairing and signing
- `soul.md` — the agent's operational principles and behavioral guidelines

> "This finding marks a significant milestone in the evolution of infostealer behavior: the transition from stealing browser credentials to harvesting the 'souls' and identities of personal AI agents." — Hudson Rock

**Exposed instances:** SecurityScorecard found 135,000 internet-exposed OpenClaw instances; over 12,800 directly exploitable via the now-patched RCE; 53,000+ correlated with prior breach activity (Bitdefender telemetry from enterprise environments).

**Microsoft advisory:** Microsoft Defender Security Research Team issued a warning that OpenClaw "should be treated as untrusted code execution with persistent credentials" and is "not appropriate to run on a standard personal or enterprise workstation."

**Broader applicability:** The Snyk ToxicSkills research (3,984 skills audited from ClawHub and skills.sh) found that prompt injection via skill descriptor files works against any AI agent platform processing third-party skill files as trusted instructions — including Claude Code, Cursor, and Windsurf.

## Research & Advisories
_As of 2026-03-08_

### Keysight — Command Injection as a New MCP Attack Vector (January 2026)

Keysight published research documenting command injection as a distinct, underexplored attack class in MCP server implementations — separate from prompt injection and tool poisoning. The research demonstrates that MCP stdio-based servers, when consuming user-controlled input for shell spawning, are prone to argument injection even with command whitelisting, as confirmed by CVE-2026-30861 in WeKnora. The paper recommends strict argument parsing (no shell=True equivalents), allowlist-not-denylist command validation, and privilege-separated subprocess execution.

*Source: [Keysight — MCP Command Injection New Attack Vector](https://www.keysight.com/blogs/en/tech/nwvs/2026/01/12/mcp-command-injection-new-attack-vector)*

### Adversa AI — MCP Security Resource Roundup (March 2026)

Adversa AI published a curated roundup of MCP security resources, noting that March 2026 marks the point at which MCP security research transitioned from proof-of-concept to real-world exploitation: CVE-2026-30861 (WeKnora), the Smithery path traversal, and the growing number of malicious skill registries all demonstrate production-grade attack chains. The roundup highlights the OWASP MCP Top 10 as the current best reference for teams implementing defenses.

*Source: [Adversa AI — Top MCP Security Resources March 2026](https://adversa.ai/blog/top-mcp-security-resources-march-2026/)*

### Pixee AppSec Weekly — 69 Vulnerabilities in 5 AI Coding Platforms

Analysis of five major AI coding platforms found 69 vulnerabilities, with the majority attributable to trust-model failures rather than classic application security bugs. The most common finding was that agent auto-approval settings — designed as convenience features — function as a single configuration change away from full RCE when paired with any prompt injection vector.

## Analyst Note

Three themes stand out from this reporting cycle:

**1. Frameworks are now in scope.** The WeKnora cluster and MS-Agent CVE-2026-2256 signal that the attack surface has expanded from user-facing AI IDEs (IDEsaster) into underlying LLM framework infrastructure. Organizations building on open-source agent frameworks — not just using AI coding tools — must now treat those frameworks as security-sensitive dependencies requiring active patching and isolation.

**2. Skills/plugin registries are the new npm.** ClawHavoc represents a maturation of supply chain attacks against AI agent ecosystems: coordinated campaigns, evasion techniques, and payloads specifically tuned to steal agent identity (not just credentials). The ClawHub model — no code signing, no sandbox, one-week-old GitHub account to publish — mirrors the conditions that produced the npm s1ngularity and Shai-Hulud attacks. Until registries enforce minimum security requirements, treating all third-party agent skills as untrusted code is the only defensible posture.

**3. AI agent identity is a new high-value target.** Hudson Rock's OpenClaw infostealer finding marks a qualitative shift: attackers are harvesting `soul.md`, gateway tokens, and cryptographic device keys — the persistent identity layer of AI agents — not just session tokens. As agents gain persistent memory and long-running OAuth authorizations, the exfiltration value of agent identity files will rival (and eventually exceed) browser credential stores.

**Immediate actions:**
- **WeKnora users:** Upgrade to v0.2.10 immediately (CVE-2026-30861, CVSS 10.0)
- **MS-Agent users:** No patch available — sandbox all deployments, disable shell tool, or remove until vendor issues a fix (CVE-2026-2256, PoC public)
- **OpenCode users:** `npm install opencode-ai@1.0.216` (CVE-2026-22812)
- **OpenClaw users:** Upgrade to v2026.2.25+; audit all installed skills; treat any skill installed before February 2026 as potentially compromised
- **All agent platforms:** Audit skill/plugin registries as untrusted third-party code; restrict agent filesystem and network access to least-privilege; isolate agent credential stores from host environment
