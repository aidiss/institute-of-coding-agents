## Incidents & Vulnerabilities

### Marimo Pre-Auth RCE — CVE-2026-39987 — Actively Exploited
**CVE:** CVE-2026-39987 | **CVSS v4.0:** 9.3 | **Status:** Patched — Fix in v0.23.0 | **Exploitation:** Confirmed in the wild

The highest-urgency item this cycle. Marimo, an open-source reactive Python notebook tool widely used in AI/ML development, contains a pre-authentication remote code execution flaw in its terminal WebSocket endpoint (`/terminal/ws`). The endpoint accepts unauthenticated connections and spawns a full PTY shell, allowing any network-reachable attacker to execute arbitrary commands on the host — no credentials required. All other WebSocket endpoints in Marimo correctly call `validate_auth()`; only `/terminal/ws` was missing the check.

**Exploitation timeline:** Within **9 hours and 41 minutes** of the advisory's publication, Sysdig Threat Research Team observed the first exploitation attempt in the wild — with no public proof-of-concept available. The attacker built a working exploit directly from the advisory text.

**Impact on AI development environments:** Marimo integrates natively with OpenAI, Anthropic, and Google AI APIs. Successful exploitation functions as a **credential harvesting operation** — LLM provider API keys, cloud credentials, and training infrastructure tokens are routinely present in active notebook environments. CSA Research classified this as the highest-severity AI toolchain compromise since the LiteLLM incident.

**Fix:** Upgrade to `marimo>=0.23.0`. Operators running Marimo with the `--host 0.0.0.0` flag (common in remote Jupyter-style deployments) are highest risk.

### Claude Code — API Key Exfiltration and RCE via Project Hooks
**CVE-2026-21852** (info disclosure, CVSS 5.3) | **CVE-2025-59536** (config override) | **Status:** Patched

Two separate vulnerabilities disclosed against Claude Code (Anthropic):

**CVE-2026-21852** — An information disclosure flaw in Claude Code's project-load flow allows a malicious repository to exfiltrate data — including Anthropic API keys — during the initial project load. Affects developers who `cd` into untrusted repositories before running `claude`.

**Project hook injection** — A user-consent bypass in Claude Code allowed malicious `.claude/settings.json` files embedded in repositories to define and execute arbitrary hooks without explicit user approval. An attacker-controlled project opened for the first time could trigger code execution before the user had any chance to review hook definitions.

**CVE-2025-59536** — Repository-defined configurations via `.mcp.json` and `.claude/settings.json` could be exploited to override explicit user permissions that had been set in personal or system-level configuration.

**Context:** Georgia Tech's Vibe Security Radar tracked Claude Code as the most-named tool in AI-generated CVEs during March 2026 (27 out of 35 directly attributed). Researchers noted this is partly because Claude Code "always leaves a signature" in generated code, making attribution easier — the true distribution across tools is likely more even.

### MCPJam Inspector — Unauthenticated RCE
**CVE:** CVE-2026-23744 | **CVSS:** 9.8 | **Status:** Patched — Fix in v1.4.3 | **Affected:** MCPJam Inspector ≤ 1.4.2

MCPJam Inspector, a popular developer tool for testing and debugging MCP servers, bound its HTTP listener on `0.0.0.0` with no authentication by default. A crafted HTTP request to the unauthenticated endpoint could install an arbitrary MCP server and execute code on the host. A documented real-world exploit chained this vulnerability with a Claude MCP integration to activate a physical pump at an industrial facility via a crafted PDF — illustrating the physical-world blast radius of MCP-adjacent vulnerabilities.

**Fix:** Upgrade to MCPJam Inspector v1.4.3.

### MCP TypeScript SDK — Cross-Client Response Leak
**CVE:** CVE-2026-25536 | **CVSS:** High | **Status:** Patched | **Affected:** MCP TypeScript SDK versions 1.10.0–1.25.3

A high-severity vulnerability in the official MCP TypeScript SDK allows responses to leak across client session boundaries, meaning one client can inadvertently receive data (including tool outputs, secrets, or file contents) intended for a different client. Affects multi-tenant or multi-session MCP server deployments.

**Fix:** Upgrade to MCP TypeScript SDK ≥ 1.25.4.

## Supply Chain

### LiteLLM Compromise — Downstream Fallout Continues (Mercor Confirmation)
**Incident date:** 2026-03-24 | **Status:** Ongoing downstream impact assessment

The March 24 LiteLLM PyPI supply chain compromise (versions 1.82.7 and 1.82.8, live ~40 minutes) continues to produce downstream disclosures. On **2026-04-01**, Mercor confirmed a security incident directly linked to the LiteLLM compromise. The `litellm_init.pth` payload ran as a multi-stage credential stealer: harvesting LLM API keys and cloud credentials on first stage, attempting Kubernetes lateral movement on second stage, and installing a persistent `systemd` backdoor polling for additional payloads on third stage.

**Current action items:** Rotate all API keys and cloud credentials in environments where litellm 1.82.7 or 1.82.8 were ever installed, even briefly. Audit `systemd` unit files for unexpected entries. Safe version: `litellm>=1.82.9`.

### Malicious VS Code AI Extension Campaign — 1.5M Combined Installs
**Status:** Ongoing investigation | **Disclosed:** Early 2026

Researchers identified a campaign in which VS Code extensions marketed as AI coding helpers accumulated over **1.5 million combined installs** before a hidden profiling engine was discovered inside them. The extensions presented a legitimate AI-assistant surface while silently fingerprinting developer environments — collecting IDE configuration, open project metadata, and installed extension lists. No CVE assigned at time of writing; scope and data destinations under investigation.

### Slopsquatting — Autonomous Agent Risk Escalation
**Status:** Ongoing threat | No CVE

Updated metrics on AI-hallucinated package name squatting reinforce the elevated risk posed by autonomous coding agents:

- ~20% of generated code samples (across 756,000 tested) reference at least one hallucinated package name
- 58% of hallucinated names are **consistent across repeated model runs** — providing reliable squatting targets for attackers who automate LLM output monitoring
- 128 tracked phantom packages accumulated **121,539 downloads** between July 2025 and January 2026
- The attack is amplified by autonomous agents (Cursor, Claude Code, GitHub Copilot Workspace): these tools can install packages without waiting for developer review, collapsing the window between hallucination and execution

### Trivy / Checkmarx / LiteLLM Trojanization Campaign — Ongoing Attribution
**Actor:** TeamPCP | **Status:** Active investigation (Kaspersky, Snyk, Sonatype)

Kaspersky, Snyk, and Sonatype have each published analyses confirming that the LiteLLM attack was one vector in a coordinated campaign also targeting Trivy (open-source vulnerability scanner) and Checkmarx (SAST platform). The common attack vector across all three: CI/CD pipeline compromise via GitHub Actions. The deliberate selection of security scanning tooling as the initial pivot is notable — Trivy and Checkmarx run with elevated trust and broad access in developer pipelines, making them high-value stepping stones.

## Research & Advisories

### Prompt Injection Now Exceeds 85% Adaptive Attack Success Rate
**Source:** arXiv:2601.17548 | Published: January 2026

A systematic analysis of 78 studies found that every tested AI coding agent — including Claude Code, GitHub Copilot, and Cursor — is vulnerable to prompt injection. Adaptive attack success rates exceed **85%** across all tested agents. The study classifies indirect injection (malicious content embedded in code comments, README files, dependency descriptions, issue bodies) as the dominant real-world vector, accounting for over 80% of documented enterprise-facing attempts.

---

### "Lethal Trifecta" — Three Agent Properties That Enable Full Compromise
**Source:** Airia Research / OWASP LLM Top 10 2026

Security researchers have formalized the "Lethal Trifecta" — the three agent properties whose combination transforms prompt injection into full system compromise:

1. **External content access** — agent reads attacker-controlled data (web pages, files, PR descriptions, issue comments)
2. **Persistent memory or state** — injected instruction survives across agent sessions
3. **Action execution capability** — agent can write files, call APIs, commit code, or execute shell commands

Claude Code, Cursor, GitHub Copilot Workspace, and Devin all satisfy all three conditions by design. Defense strategies focus on breaking at least one leg: sandboxing execution environments, disabling cross-session memory for external-facing agents, or requiring explicit human-in-the-loop approval for write operations.
