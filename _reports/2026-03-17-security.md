## Incidents & Vulnerabilities

### IDEsaster: 30+ CVEs Across All Major AI IDEs
**Researcher:** Ari Marzouk (MaccariTA) | **Disclosed:** December 2025 | **Status:** Most patched

100% of tested AI IDEs were found vulnerable to a shared attack chain: **Prompt Injection → Agent Tools → Base IDE Features**, enabling data exfiltration and RCE. Malicious payloads are delivered via rule files, MCP servers, deeplinks, or file names. Over 24 CVEs were assigned; AWS issued advisory AWS-2025-019. Affected tools include Cursor, Windsurf, GitHub Copilot, Roo Code, JetBrains Junie, Cline, Kiro.dev, Zed.dev, Gemini CLI, and Claude Code.

Key CVEs from this research:

| CVE | Tool | Description | Status |
|---|---|---|---|
| CVE-2025-49150 | Cursor | Prompt injection → sensitive file reads | Patched |
| CVE-2025-54135 ("CurXecute") | Cursor | Permission bypass via `.cursor/mcp.json` → RCE | Patched (v1.3.9) |
| CVE-2025-54136 ("MCPoison") | Cursor | Post-approval MCP config mutation → silent RCE | Patched (v1.3.9) |
| CVE-2025-61590 | Cursor | Workspace config override via prompt injection | Patched |
| CVE-2025-64660 | GitHub Copilot | Prompt injection → workspace config edit → code execution | Patched (Dec 2025) |
| CVE-2025-64671 | GitHub Copilot (JetBrains) | Malicious repo/PR review → arbitrary command execution, token theft | Patched |
| CVE-2025-53097 | Roo Code | Prompt injection → sensitive file access | Patched |
| CVE-2025-58372 | Roo Code | Workspace settings override → RCE | Patched |
| CVE-2025-58335 | JetBrains Junie | Prompt injection → file read | Patched |

Additionally, Cursor and Windsurf are built on outdated Chromium/Electron versions, exposing ~1.8 million developers to 94+ known browser CVEs. CVE-2025-7656 (a patched Chromium flaw) was successfully weaponized against current Cursor and Windsurf releases at time of disclosure.

### GitHub Copilot — RCE via Prompt Injection
**CVE-2025-53773** | CVSS: 9.6 | **Disclosed/Patched:** August 2025 | **Status:** Patched

### Amazon Q Developer VS Code Extension — Supply Chain Attack
**CVE-2025-8217** | **Patched:** July 24, 2025 (v1.85.0) | **Status:** Patched; not actively exploited (attack failed due to syntax error)

An attacker exploited an insufficiently scoped GitHub token in the CodeBuild pipeline to commit malicious code into the open-source extension. The payload in v1.84.0 was designed to delete the filesystem, clear config files, enumerate AWS profiles, and delete S3 buckets, EC2 instances, and IAM users. Approximately **1 million developers** downloaded the compromised version; execution was prevented only by a syntax error in the injected code.

Separately, researcher Johann Rehberger demonstrated prompt injection leading to data exfiltration via DNS and RCE in Amazon Q Developer. AWS fixed these but issued no public CVE or advisory for the prompt injection issues, citing CNA criteria.

## Supply Chain

### Slopsquatting — AI-Hallucinated Package Names as Attack Vectors
**Term coined by:** Seth Larson (Python Software Foundation) | **Key research:** March–April 2025 (USENIX Security 2025)

LLMs suggest non-existent package names at significant rates. Attackers register those names on PyPI/npm with malicious payloads.

**Scale (576,000 code samples, 16 LLMs):**
- 19.7% of all suggested dependencies were hallucinated (440,445 packages)
- Commercial models (GPT-4): ~5.2% hallucination rate
- Open-source models (CodeLlama, DeepSeek, WizardCoder, Mistral): up to 21.7%
- 43% of hallucinated names repeated consistently across queries — making them predictable and squattable

**Documented incidents:**
- **`huggingface-cli` PoC (Lasso Security):** Phantom PyPI package received 30,000+ downloads in 3 months after registration; Alibaba referenced it in official documentation
- **`react-codeshift` (January 2026):** npm package hallucinated by LLMs combining `jscodeshift` + `react-codemod`; real AI agents observed attempting to install it
- **Slopsquatting + Dependency Confusion hybrid:** LLM training data may include leaked internal package names; attacker registers the name publicly; package manager resolves the malicious public version over the legitimate private one

No individual CVEs assigned; this is a class-level threat.

## Research & Advisories

### MCP — CVE-2025-6514: `mcp-remote` OS Command Injection
**Researcher:** JFrog Security Research | **Disclosed:** July 9, 2025 | CVSS: 9.6 | **Status:** Patched (v0.1.16)

The `mcp-remote` OAuth proxy passed a server-controlled `authorization_endpoint` URL to an OS command without sanitization, enabling a malicious MCP server to trigger arbitrary command execution on the connecting client. First publicly documented full RCE in a real-world MCP deployment.

- **Affected:** `mcp-remote` npm v0.0.5–0.1.15 (~437,000 downloads)
- **Fixed:** v0.1.16

### MCP — CVE-2025-68143 / 68144 / 68145: Anthropic Git MCP Server RCE Chain
**Researcher:** Cyata Security | **Reported:** June 2025 | **Disclosed:** January 20, 2026 | **Status:** Patched (v2025.12.18)

Three chained vulnerabilities in Anthropic's official `mcp-server-git`:

| CVE | CVSS | Description |
|---|---|---|
| CVE-2025-68145 | 7.1 | Path validation bypass in `--repository` flag → access any repo on system |
| CVE-2025-68143 | 8.8 | `git_init` accepts arbitrary filesystem paths without validation |
| CVE-2025-68144 | 8.1 | Argument injection in `git_diff`/`git_checkout` → file deletion/overwrite |

**Full RCE chain:** Path bypass → `git_init` in `~/.ssh` → write malicious hook via Filesystem MCP server → trigger via `git_diff` → arbitrary code execution. Exploitable via prompt injection from a malicious README or issue description; no direct system access required.

### "Rules File Backdoor" — Hidden Unicode in AI Instruction Files
**Researcher:** Pillar Security | **Published:** March 2025

Attackers embed malicious instructions in `.cursorrules` or GitHub Copilot instruction files using zero-width Unicode characters, bidirectional text markers, and other invisible techniques. The AI reads and executes hidden instructions; generated code appears legitimate to human reviewers. Malicious instructions propagate silently through repo forks and clones with no trace in chat logs.

GitHub responded by adding warnings for hidden Unicode characters on github.com. Both GitHub and Cursor placed root responsibility on developers reviewing AI suggestions.

### "Promptware" Kill Chain
**Researchers:** Tel Aviv University, Ben-Gurion University, Harvard | **Published:** February 2026

Formalizes AI-targeting malware as "promptware" with a seven-stage kill chain: Initial Access → Privilege Escalation (jailbreaking) → Reconnaissance → Lateral Movement → Persistence → Command & Control → Execution.

Key finding: attacks documented in 2023 typically covered 2–3 stages; by 2025–2026, 15 of 21 documented incidents covered 4+ stages. AI coding assistants were the most-targeted category (7 of 21 incidents). No current defenses address the Reconnaissance stage. Covered by Bruce Schneier and Lawfare Media.
