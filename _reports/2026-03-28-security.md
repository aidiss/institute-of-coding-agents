### Claude Code — RCE & API Token Exfiltration

**CVE-2025-59536** (CVSS 8.7) and **CVE-2026-21852** (CVSS 5.3), discovered by Check Point Research. Status: **Both patched.**

Three attack mechanisms:
1. **Malicious Hooks** — `.claude/settings.json` in a cloned repo executes shell commands at lifecycle events before the trust dialog appears.
2. **MCP consent bypass** — `.mcp.json` with `enableAllProjectMcpServers` executes arbitrary commands before trust confirmation.
3. **API key interception** — `ANTHROPIC_BASE_URL` in project config redirects traffic to an attacker-controlled server, capturing the API key in plaintext.

Disclosure timeline: July 21, 2025 (reported) → Aug 26, 2025 (first fix) → Oct 3, 2025 (CVE-2025-59536 published) → Jan 21, 2026 (CVE-2026-21852 published) → Feb 25, 2026 (public disclosure).

**CVE-2025-54795** (Trail of Bits) — argument injection via allowlisted "safe" commands (`go test`, `git`, `ripgrep`) weaponized for file creation and execution. **Patched.**

### GitHub Copilot — RCE via Prompt Injection

**CVE-2025-53773** (CVSS 9.6), disclosed by Pillar Security / Embrace The Red. **Patched Aug 2025.**

Prompt injection embedded in source code, web pages, or GitHub issues tricks Copilot into modifying `.vscode/settings.json` to enable `chat.tools.autoApprove: true` ("YOLO mode"), then executing arbitrary terminal commands. Cross-platform: Windows, macOS, Linux. GitHub added a secondary mitigation: warnings for hidden Unicode content on github.com.

### Persistent Agent Memory Poisoning & Zero-Click RCE Chains

Researchers documented a fully zero-click RCE chain: crafted Google Docs → agent fetches instructions from malicious MCP server → Python payload executes → secrets harvested. No user interaction required. A related technique poisons agent long-term memory stores, allowing injected instructions to persist and execute in future sessions days or weeks later.

## Supply Chain

### s1ngularity — npm Supply Chain Attack (August 2025)

Malicious versions of `@nrwl/nx`, `@nx/devkit`, and related packages published to npm. Payload in `telemetry.js` searched for `.env` files, SSH keys, crypto wallets, and GitHub/npm tokens.

**AI-specific angle:** Attackers invoked installed AI CLIs (Claude, Gemini, Amazon Q) with dangerous flags (`--dangerously-skip-permissions`, `--yolo`, `--trust-all-tools`) to use victims' own AI tools for filesystem reconnaissance.

Impact: 1,079 developer systems, 2,349 credentials, 1,000+ GitHub tokens, 20,000+ files, 400+ orgs, 5,500+ private repos made public. 8-hour exposure window. Led to npm mandating Granular Access Tokens (Dec 9, 2025).

---

### Shai-Hulud Worm — Self-Replicating npm Malware (September–November 2025)

Self-replicating worm that steals credentials then uses them to infect every package the victim maintains. Initial vector: AI-generated phishing email mimicking official npm correspondence.

**526 packages compromised**, including `@crowdstrike/commitlint`, `@crowdstrike/falcon-shoelace`, and high-impact packages (`chalk`, `debug`, `ansi-styles` — 2+ billion downloads/week). CISA issued emergency alert September 23, 2025. Malicious script assessed as AI-generated based on code style and comments.

### OpenClaw / ClawHub — "ClawHavoc" AI Agent Skills Attack (January–March 2026)

The largest confirmed supply chain attack on AI agent infrastructure. OpenClaw, an open-source local AI agent that reached 180,000 GitHub stars in January 2026, had its community skills marketplace **ClawHub** compromised:

- **1,184 malicious skills** out of ~10,700 total (~1 in 9)
- **135,000+ publicly exposed instances** (SecurityScorecard); 50,000+ exploitable via RCE
- Skills run with full agent permissions: terminal, disk, OAuth tokens
- Primary payload: **Atomic macOS Stealer (AMOS)** targeting credentials, browser data, crypto wallets, Telegram chats, VPN profiles, Apple Keychain
- Secondary lures: fake crypto trading tools targeting exchange API keys and wallet private keys

### Slopsquatting — Weaponized AI Package Hallucinations

LLMs hallucinate plausible-sounding but nonexistent package names; attackers register these names with malicious payloads. Unlike typosquatting, the names only appear in AI suggestions — never in organic developer usage.

**Key data points:**
- 5–38% hallucination rates across leading LLMs (2024); GitHub Copilot hallucinates dependencies ~15% of the time
- 58%+ of hallucinated package names reappear across multiple prompt runs; 43% appear consistently in 10+ runs (predictable attack targets)
- July 2025–January 2026: 128 tracked phantom packages accumulated **121,539 total downloads**; `openapi-generator-cli` alone had 48,356 downloads
- **2026 study** of 117,062 dependency changes across 33,596 agent-authored PRs: AI agents modify dependency manifests far more frequently than humans (Copilot 33.5%, Devin 29.6%, OpenAI Codex 23.6%, Cursor 10.6%, Claude Code 2.7%)
- **Automated Dependency Side-Loading**: a newer variant where a compromised IDE extension intercepts AI suggestions and silently substitutes attacker-controlled package names pre-commit, invisible to standard tooling

### LSP Security — LSPFUZZ Research (2025)

LSPFUZZ, a grey-box hybrid fuzzer for LSP servers, found **51 previously unknown bugs** across clangd, sorbet, verible, and solc; 42 confirmed, 26 fixed, 2 CVEs granted. Notably, clangd bugs include buffer-overflow and use-after-free conditions exploitable via malicious source files — the LLVM team responded by **disabling clangd in their VS Code extension for untrusted workspaces**.

Historical reference: CVE-2022-23853 (KDE Kate LSP plugin) — executes LSP server binary from the directory of the opened file if not in PATH, enabling untrusted binary execution. Still cited as a design pattern risk.

### Key Observations

1. **Prompt injection is the universal entry point** — reliably chained to RCE in every major AI IDE via the same pattern: injection → auto-approved tool action → IDE feature weaponization.
2. **MCP is the dominant new attack surface** — nearly every serious disclosure from mid-2025 involves MCP as either an amplifier for prompt injection or a direct RCE vector.
3. **Some vendors chose documentation warnings over patches** — Claude Code notably issued security warnings rather than code fixes for several IDEsaster-class vulnerabilities.
4. **AI tools are being weaponized against themselves** — s1ngularity explicitly invoked AI CLIs with dangerous flags for filesystem reconnaissance, a novel compound threat category.
5. **Slopsquatting is operational at scale** — 121,000+ downloads of phantom AI-hallucinated packages documented in a six-month window.
6. **OpenClaw/ClawHub is the most severe AI agent registry incident to date** — 1 in 9 packages malicious, 135,000 exposed instances, AMOS stealer delivered via AI-native SKILL.md files.
