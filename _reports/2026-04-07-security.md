# Security Report: AI Coding Agents
**Date:** 2026-04-07

## Executive Summary

The first quarter of 2026 marks a significant escalation in the security threat landscape for AI coding agents. Prompt injection–to–RCE chains are now routine, MCP has emerged as a broad new attack surface, and supply chain risks are compounding through both slopsquatting and direct package registry compromise. AI-generated code is now directly responsible for an accelerating number of CVEs (35 in March 2026 alone, up 5× from January).

---

## Incidents & Vulnerabilities

### IDEsaster — Cross-IDE Vulnerability Class
- **Researcher:** Ari Marzouk (MaccariTA)
- **Disclosed:** December 2025; patches ongoing
- **Affected tools:** Cursor, Windsurf, GitHub Copilot (VS Code), Kiro.dev, Zed.dev, Roo Code, Junie, Cline — 100% of tested products
- **CVEs:** 24 assigned across vendors
- **Status:** Rolling patches by individual vendors
- Chains prompt injection with legitimate IDE features (tool calls, schema fetching, file writes) to achieve data exfiltration and RCE. Attack vectors include hidden Unicode characters in pasted URLs or code comments poisoning LLM context, MCP tool poisoning, and rug pulls.

### Claude Code — Malicious Hook RCE
- **CVE:** CVE-2025-59536 (CVSS 8.7)
- **Status:** Disclosed; patched
- Injecting a malicious hook into `.claude/settings.json` inside a project grants RCE the moment a developer opens that project. Configuration-file-based attack surface; not version-pinned.

### Claude Code — API Key Exfiltration
- **CVE:** CVE-2026-21852 (CVSS 5.3)
- **Status:** Disclosed; patched
- Environment variable manipulation allows exfiltration of API keys from Claude Code sessions.

### Claude Code — Sandbox Bypass / Unauthorized File System Access
- **CVE:** CVE-2026-25725
- **Status:** Disclosed; patched

### Claude Code — npm Source Leak & Trojanized HTTP Client (March 31, 2026)
- **CVE:** None assigned
- **Status:** Contained; new packaging controls deployed
- A 59.8 MB JavaScript source map bundled in `@anthropic-ai/claude-code` v2.1.88 exposed ~513,000 lines of unobfuscated TypeScript (1,906 files). A trojanized HTTP client component was also present; users who installed between 00:21–03:29 UTC on March 31 may have received a cross-platform RAT. Anthropic attributed to human packaging error, not an intrusion. No customer credentials exposed. Secondary impact: attackers distributed Vidar stealer and GhostSocks proxy malware through fake "leaked Claude Code" GitHub repositories capitalizing on the publicity.

### Cursor — RCE Chain
- **CVE:** CVE-2026-22708 (CVSS 9.8) + CVE-2025-49150, CVE-2025-54130, CVE-2025-61590
- **Status:** Patched
- Three vulnerabilities combined to enable RCE via indirect prompt injection in Cursor. A separate case-sensitivity bypass in MCP configuration (CVE-2025-59944) allowed persistent RCE surviving IDE restarts.

### GitHub Copilot — "RoguePilot" GITHUB_TOKEN Exfiltration
- **CVE:** None assigned
- **Status:** Patched (February 2026)
- Passive prompt injection via a crafted GitHub Issue. When a developer launches a Codespace from the issue, Copilot executes hidden attacker instructions (in an HTML comment), exfiltrating `GITHUB_TOKEN` via VS Code's JSON schema download feature, enabling full repository takeover.

### Trail of Bits — Prompt Injection to RCE Research (October 2025)
- **Status:** Research/disclosure; no single CVE
- Demonstrated one-shot RCE across three agent platforms using argument injection. Attack surfaces: code comments, agentic rule files, GitHub repos, log output. Exploits pre-approved commands (`go test -exec`, `git show`, `ripgrep`) to escape sandboxes. Recommendations: sandboxing, argument separator (`--`) enforcement, disabling shell execution.

### EchoLeak — First Zero-Click Production LLM Prompt Injection (September 2025)
- **Status:** Research disclosure
- Demonstrated the first zero-click, real-world prompt injection exploit in a production LLM system requiring no user interaction.

## Supply Chain

### LiteLLM PyPI Supply Chain Attack — TeamPCP Campaign
- **Date:** March 19–24, 2026
- **Status:** Contained; safe version `v1.83.0` released; packages quarantined by PyPI after ~40 minutes
- Threat actor group "TeamPCP" compromised LiteLLM's CI/CD by first rewriting Git tags in the `trivy-action` GitHub Action (March 19) to harvest PyPI credentials. Five days later they uploaded backdoored versions `litellm==1.82.7` and `1.82.8`. Three-stage payload: (1) credential harvesting (SSH keys, cloud credentials, Kubernetes secrets), (2) Kubernetes lateral movement, (3) persistent backdoor RAT via a malicious `.pth` file executing on every Python process startup. LiteLLM has ~3.4M daily downloads. Telnyx was also affected in the same campaign.

### Slopsquatting — AI Hallucinated Package Names
- **Researcher:** Seth Larson (PSF); term coined March 2025; ongoing
- **Status:** Systemic/ongoing risk; no single CVE
- ~20% of AI-generated Python/JavaScript code samples recommend non-existent packages. 43% of hallucinated packages are suggested consistently across 10 re-runs (high repeatability = high exploitability). Researcher Bar Lanyado registered one hallucinated package (`huggingface-cli` on PyPI) — it received 30,000+ authentic downloads in three months. The `sloppy-joe` tool is available as a mitigation.

### Dependency Confusion via Claude Code npm Leak
- **Date:** Late March / early April 2026
- **Status:** Active exploitation
- After the Claude Code source leak, attackers registered typosquatted internal npm package names visible in the leaked source to target developers attempting to compile it. Classic dependency confusion attack seeded by AI-generated internal package names.

### AI-Generated CVEs from Vibe Coding (2026 trend)
- 35 CVEs in March 2026 attributed to AI-generated code (27 from Claude Code, 4 from GitHub Copilot, 2 from Devin, 1 each from Aether and Cursor). Up from 6 in January and 15 in February 2026. Indicates AI-generated code is introducing novel vulnerabilities at an accelerating rate.

## Research & Advisories

### MCP Tool Poisoning
- **Researcher:** Invariant Labs
- **Status:** Systemic/architectural risk; no single CVE. Detection: `mcp-scan` tool.
- Malicious MCP servers embed hidden instructions in tool descriptions. The poisoned tool does not need to be called — loading it into context is sufficient to cause the model to follow attacker instructions.

### MCP Rug Pull Attacks
- **Status:** Architectural risk. ETDI (Enhanced Tool Definition Interface) with OAuth proposed as mitigation.
- A tool's description or behavior is silently altered after initial user approval, turning a trusted tool malicious without triggering a new approval flow.

### MCPJam Inspector — Unauthenticated RCE
- **CVE:** CVE-2026-23744 (CVSS 9.8)
- **Affected versions:** <= 1.4.2
- **Status:** Patched in v1.4.3+
- Binds a critical endpoint to `0.0.0.0` with no authentication. A single crafted HTTP request installs an arbitrary MCP server and executes code on the host with no user interaction.

### Anthropic Git MCP Server — Chained Path/Injection Vulnerabilities
| CVE | Description | Status |
|-----|-------------|--------|
| CVE-2025-68145 | Path validation bypass | Patched |
| CVE-2025-68143 | Unrestricted `git_init` | Patched |
| CVE-2025-68144 | Argument injection in `git_diff` | Patched |

### MCP TypeScript SDK — Cross-Client Data Leak
- **CVE:** CVE-2026-25536
- **Affected versions:** v1.10.0–v1.25.3
- **Status:** Patched in v1.25.4+
- When a single `McpServer` instance with `StreamableHTTPServerTransport` is reused across multiple clients, responses leak across client boundaries, exposing one user's session data to another.

### "MCPwned" — Azure MCP Server RCE / Cloud Takeover
- **Researcher:** Token Security; presented at RSAC 2026
- **Status:** Patched by Microsoft (removed vulnerable SSE transport, added authentication to Streamable HTTP transport)
- Unauthenticated RCE in the official Azure MCP server. Attacker with network access can steal Entra ID credentials, compromise the entire Azure/Entra ID tenant, and inject spoofed data to other MCP clients for downstream prompt injection.

### MCP Sampling Attack Vectors
- **Researcher:** Palo Alto Unit 42
- **Status:** Research disclosure
- Malicious MCP servers exploit the "sampling" feature (servers requesting LLM completions through the client) to hijack model reasoning and exfiltrate data.

### Vibe Coding / "Slop Code" Security Risk
- **Researcher:** Repello AI
- **Status:** Research; no CVE. Mitigations published by FOSSA, Contrast Security.
- AI agents in vibe coding workflows auto-approve insecure dependencies, skip input validation, and introduce vulnerabilities without review.

## Key Themes

1. **Prompt injection is now the #1 AI vulnerability** — appearing in 73%+ of production assessments (OWASP LLM Top 10). The path from prompt injection to full RCE is documented and repeatable across all major platforms.
2. **MCP is a critical new attack surface** — every major attack category (RCE, SSRF, cross-tenant data leakage, rug pulls, tool poisoning) has a live published example against MCP infrastructure.
3. **Supply chain risk is compounding** — slopsquatting, direct PyPI compromise (LiteLLM/TeamPCP), and AI-authored CVEs are all accelerating simultaneously and interact: leaked source enables dependency confusion, hallucinated packages enable slopsquatting.
4. **AI-generated code is producing vulnerabilities at scale** — 35 AI-authored CVEs in March 2026, up 5× from January; this is an emerging measurement category that will likely grow.
5. **Trust gap remains** — only ~5% of organizations have AI agents in broad production despite 85% experimenting, largely due to unresolved security concerns.
