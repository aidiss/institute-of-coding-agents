**2026-04-17**

## Incidents & Vulnerabilities

### MCPJam Inspector RCE — CVE-2026-23744 (CVSS 9.8)

**Status:** Disclosed; patch available  
**Affected versions:** MCPJam Inspector ≤ 1.4.2  
**Fix:** Upgrade to 1.4.3+

MCPJam Inspector listens on `0.0.0.0` by default with no authentication. A crafted HTTP request can install an arbitrary MCP server and execute code on the host. The vulnerability is remotely triggerable without any user interaction on local developer machines that expose the inspector port to the network or LAN.

Sources: [IronPlate Weekly Threat Intel](https://ironplate.ai/blog/weekly-agentic-ai-threat-intel-april-7-2026) · [AI Agent Security Risks 2026](https://blog.cyberdesserts.com/ai-agent-security-risks/)

### MCP TypeScript SDK Cross-Client Data Leak — CVE-2026-25536

**Status:** Disclosed; patch status unclear  
**Affected versions:** MCP TypeScript SDK (specific versions TBD)

When a single `McpServer` instance is reused across multiple client connections, responses can leak across client boundaries. Affects multi-tenant deployments of MCP-based tooling where a shared server instance handles multiple independent agent sessions.

Sources: [Vulnerable MCP Project](https://vulnerablemcp.info/) · [MCP Security Vulnerabilities — Practical DevSecOps](https://www.practical-devsecops.com/mcp-security-vulnerabilities/)

### Anthropic mcp-server-git Chained RCE — CVE-2025-68143, CVE-2025-68144, CVE-2025-68145

**Status:** Disclosed  
**Affected component:** Anthropic's official `mcp-server-git` + Filesystem MCP server

Three chained vulnerabilities in Anthropic's own `mcp-server-git` MCP server, which when combined with the Filesystem MCP server achieve full remote code execution via malicious `.git/config` files in cloned repositories. An attacker controlling a repository can trigger code execution in the developer's environment when the agent clones or inspects it.

Sources: [Vulnerable MCP Project](https://vulnerablemcp.info/) · [MCP Security Complete Guide 2026](https://aembit.io/blog/the-ultimate-guide-to-mcp-security-vulnerabilities/)

### Systemic MCP Architecture Flaw — Arbitrary Command Execution

**Status:** Disclosed April 15, 2026; no patch available (architectural)  
**Affected:** Anthropic's official MCP SDKs (Python, TypeScript, Java, Rust)  
**Exposure:** 200+ open-source projects, ~150 million downloads, up to 200,000 vulnerable instances

A critical systemic vulnerability rooted in an architectural design decision in the official MCP SDKs across all supported languages enables arbitrary command execution. Because the flaw is baked into the protocol's reference implementations rather than application code, remediation requires SDK-level changes and coordinated downstream patching across the ecosystem.

Sources: [Infosecurity Magazine](https://www.infosecurity-magazine.com/news/systemic-flaw-mcp-expose-150/) · [Red Hat MCP Security](https://www.redhat.com/en/blog/model-context-protocol-mcp-understanding-security-risks-and-controls)

### Cursor RCE via Indirect Prompt Injection — CVE-2026-22708, CVE-2026-26268, CVE-2026-21523

**Status:** Disclosed; fix status varies per CVE  
**Affected:** Cursor IDE

Three distinct vulnerabilities enable RCE through indirect prompt injection in the Cursor IDE. Both zero-click and one-click attack scenarios have been confirmed. A malicious repository, document, or web page referenced during an agent session can embed instructions that cause the agent to execute attacker-controlled commands on the developer's machine.

Additionally, CVE-2025-59944 (case-sensitivity bypass) and CVE-2025-54135 (CVSS 8.6) enable RCE through MCP configuration manipulation in Cursor.

Sources: [New Vulnerability in GitHub Copilot and Cursor — Pillar Security](https://www.pillar.security/blog/new-vulnerability-in-github-copilot-and-cursor-how-hackers-can-weaponize-code-agents) · [GBHackers](https://gbhackers.com/ai-developer-tools/)

### "Comment and Control" Attacks on GitHub Actions Agents

**Status:** Disclosed; no universal fix (design-level issue)  
**Affected:** Claude Code Security Review action, Google Gemini CLI Action, GitHub Copilot Agent

Three widely used AI agent GitHub Actions are vulnerable to "Comment and Control" attacks, where attackers steal API keys and access tokens by injecting malicious content into PR titles, issue bodies, and comments. ## Supply Chain

### LiteLLM PyPI Supply Chain Compromise (March 24, 2026)

**Affected versions:** litellm 1.82.7, 1.82.8 on PyPI  
**Status:** Quarantined by PyPI; packages removed ~40 minutes after publication  
**Attack vector:** Poisoned Trivy security scanner in CI/CD pipeline

The most significant AI tooling supply chain incident of 2026. The attack chain:

1. **March 19:** Attackers rewrote Git tags in the `trivy-action` GitHub Action repository to point to a malicious release (`v0.69.4`) carrying a credential-harvesting payload.
2. **March 24:** LiteLLM's CI/CD pipeline executed the compromised Trivy action, which exfiltrated the `PYPI_PUBLISH` token from the GitHub Actions runner.
3. Attackers used the stolen token to publish backdoored versions 1.82.7 and 1.82.8 to PyPI, delivering a multi-stage credential stealer targeting OpenAI, Anthropic, Google, and cloud provider API keys.

LiteLLM underpins a large fraction of agentic AI deployments as an API gateway. Any environment that pip-installed or upgraded during the 40-minute window may have had all configured credentials exfiltrated.

Sources: [LiteLLM Security Update](https://docs.litellm.ai/blog/security-update-march-2026) · [Trend Micro](https://www.trendmicro.com/en_us/research/26/c/inside-litellm-supply-chain-compromise.html) · [Snyk](https://snyk.io/blog/poisoned-security-scanner-backdooring-litellm/) · [Sonatype](https://www.sonatype.com/blog/compromised-litellm-pypi-package-delivers-multi-stage-credential-stealer) · [ARMO](https://www.armosec.io/blog/litellm-supply-chain-attack-backdoor-analysis/)

### OpenClaw AI Agent Incident

**Status:** Ongoing; millions in user losses reported

The OpenClaw open-source AI agent framework (135,000+ GitHub stars) suffered the first major AI agent supply chain incident of 2026, compounded by its use of LiteLLM as a routing layer. Users whose OpenClaw instances routed through a pip-installed LiteLLM Proxy during the compromise window are considered exposed. AI agent deployments are exceptionally high-value supply chain targets because they typically hold credentials for multiple LLM providers, cloud services, and messaging platforms in a single environment.

### Slopsquatting: Hallucinated Dependencies as Attack Surface

**Status:** Active, ongoing threat class

AI coding tools hallucinate non-existent package names in approximately **20% of code generation cases** (from a study of 576,000 Python and JavaScript samples). Security researcher Seth Larson coined the term *slopsquatting* for the resulting attack vector: adversaries register the hallucinated package names on PyPI/npm before legitimate developers notice, then publish malicious payloads.

Key data points:
- **58%** of hallucinated package names are repeated consistently across multiple model runs, making them reliable and predictable targets.
- **38%** of hallucinated names closely resemble real packages; **13%** are typos of real packages; **51%** are wholly fabricated.
- Researcher Bar Lanyado registered a hallucinated package (`huggingface-cli`) and observed **30,000+ downloads** within three months.
- 128 tracked "phantom packages" accumulated **121,539 downloads** between July 2025 and January 2026 (~4,000/week).

Sources: [Bleeping Computer](https://www.bleepingcomputer.com/news/security/ai-hallucinated-code-dependencies-become-new-supply-chain-risk/) · [Check Point Blog](https://blog.checkpoint.com/research/the-hidden-ai-threat-to-your-software-supply-chain/) · [Infosecurity Magazine — Slopsquatting](https://www.infosecurity-magazine.com/news/ai-hallucinations-slopsquatting/) · [Capitol Technology University](https://www.captechu.edu/blog/ai-driven-threats-in-software-supply-chains) · [Security Boulevard](https://securityboulevard.com/2026/03/coding-agents-widen-your-supply-chain-attack-surface/)

## Research & Advisories

### All Major Coding Agents Vulnerable to Prompt Injection (Meta-Analysis)

A January 2026 systematic analysis of 78 studies confirmed that every tested coding agent — Claude Code, GitHub Copilot, Cursor — is vulnerable to prompt injection. Adaptive attack success rates exceed **85%**. Indirect injection (via documents, repositories, web content) now accounts for **>80%** of documented enterprise attack attempts, up from a minority share in 2025. Overall prompt injection attack volume surged **340%** in 2026.

Sources: [MDPI Prompt Injection Review](https://www.mdpi.com/2078-2489/17/1/54) · [Airia — AI Security 2026](https://airia.com/ai-security-in-2026-prompt-injection-the-lethal-trifecta-and-how-to-defend/) · [Markaicode](https://markaicode.com/prompt-injection-attacks-ai-security-2026/)

### AI-Generated CVE Surge — "Vibe Coding's Security Debt"

Georgia Tech's Vibe Security Radar project confirmed **74 CVEs** directly attributable to AI coding tools over its tracking lifetime, with Claude Code accounting for 27 cases. Monthly rates are accelerating: 6 in January 2026, 15 in February, **35 in March 2026**. Researchers estimate the true count is 5–10× higher across the broader open-source ecosystem, as AI attribution is often not recorded in CVE metadata.

Sources: [CSA Research Note](https://labs.cloudsecurityalliance.org/research/csa-research-note-ai-generated-code-vulnerability-surge-2026/) · [Infosecurity Magazine — AI-Generated Code](https://www.infosecurity-magazine.com/news/ai-generated-code-vulnerabilities/)
