## Incidents & Vulnerabilities

### Axios npm Supply Chain Compromise — Sapphire Sleet / UNC1069

- **Attacker:** Sapphire Sleet (aka CryptoCore / CageyChameleon) — North Korean state-sponsored, an offshoot of BlueNoroff
- **Disclosed:** March 31, 2026 (Help Net Security, Microsoft Threat Intelligence)
- **Attribution confirmed:** April 1–3, 2026 (Microsoft Security Blog, Google Cloud / Mandiant, SANS Institute)
- **Status:** Malicious packages removed from npm; downstream exposure ongoing
- **Affected packages:** `axios` versions **1.14.1** and **0.30.4** — both were newly published typosquats/impersonation packages, not modifications to the legitimate `axios` repo
- **No CVE assigned** as of April 4, 2026

**Technical detail:** The two malicious packages injected a dependency that downloaded a second-stage RAT payload from a North Korean-controlled C2. The RAT varied by OS: macOS, Windows, and Linux variants were all served. Google / Mandiant is tracking the threat actor cluster as **UNC1069**; the attack infrastructure overlaps with prior Sapphire Sleet campaigns targeting cryptocurrency firms and developer toolchains.

**Relevance to AI coding agents:** Axios is one of the most-hallucinated npm packages in AI-generated JavaScript code. The attack is consistent with the Sapphire Sleet pattern of targeting packages that developers are directed to install — whether through legitimate documentation or AI-generated code suggestions. Agents running automated `npm install` on generated code are a viable propagation vector.

**Recommended action:** Pin `axios` to a verified digest; audit `package-lock.json` for versions 1.14.1 or 0.30.4; check outbound connections for the Sapphire Sleet C2 indicators published by Microsoft and Elastic.

### TeamPCP Cascading Campaign — Ongoing Attribution and Scope

- **Status update (April 2026):** ReversingLabs and Datadog Security Labs have published extended post-incident analysis of the March 19–27 TeamPCP campaign that compromised Trivy → KICS → LiteLLM → Telnyx
- **New finding:** TeamPCP's malware in the LiteLLM versions (1.82.7, 1.82.8) also harvested `~/.kube/config` Kubernetes credentials in addition to the previously reported SSH keys, cloud tokens, and API keys — broadening the scope of potential blast radius for affected organisations
- **Affected versions:** `litellm` PyPI 1.82.7, 1.82.8 (exposure window: ~10:39–16:00 UTC, March 24, 2026)
- **Status:** PyPI quarantine stands; no new malicious versions detected post-quarantine

### AI-Generated Code CVE Trend — Continued Acceleration

As of early April 2026, CVEs directly attributed to AI-generated code have surpassed **40** (up from 35 reported on March 26, 2026 by The Register). The shared-prompt propagation model — where one insecure coding suggestion is issued identically to millions of developers — continues to compress the time between a vulnerable pattern appearing and being exploited at scale.

## Supply Chain

### Slopsquatting — Attack Surface Remains Open

- No new mitigation announcements from PyPI or npm registries as of April 4, 2026
- The `huggingface-cli` hallucinated package PoC registered by Lasso Security continues to accumulate downloads, confirming that developers are still copy-pasting AI-generated install instructions without verification
- Microsoft's Sapphire Sleet campaign (Axios, see above) is the first confirmed nation-state attack whose targeting selection aligns closely with slopsquatting methodology: the actor registered plausible-looking Axios versions rather than waiting for developers to mistype

### Checkmarx VS Code Extension — Post-Incident Status

- The Checkmarx VS Code extension compromised in the TeamPCP cascading campaign has been restored; affected versions removed from the VS Code Marketplace
- Developers who installed the extension between March 20–23, 2026 should rotate credentials stored in VS Code's secret storage

## Research & Advisories

### Adversa AI — Top Agentic AI Security Resources (April 2026)

- Adversa AI published a curated digest of agentic AI security research for April 2026, consolidating findings from the last 90 days
- Key themes: indirect prompt injection remains the dominant attack class (>80% of documented enterprise incidents); MCP tool poisoning is now considered a solved attack class with multiple public PoCs; the research community is shifting focus to **multi-agent trust boundary failures** — attacks that propagate horizontally between cooperating agents with shared context

### RSAC 2026 Preview — MCPwned (Azure MCP RCE)

- Ariel Simon (Token Security) confirmed the MCPwned research (Azure MCP server leading to full cloud tenant compromise) will be presented at RSAC 2026
- Patch status: still not fully public as of April 4, 2026; Microsoft is working on remediation

### Elastic Security Labs — Axios Supply Chain Forensics

- Elastic published a detailed forensic breakdown of how the Axios npm attack was detected via behavioral signals in Elastic Agent, including the characteristic outbound connection pattern to the Sapphire Sleet C2 infrastructure
- The detection relied on process ancestry anomalies (Node.js spawning unexpected child processes) rather than signature matching — underscoring the importance of behavioral EDR for supply chain attack detection in developer environments
