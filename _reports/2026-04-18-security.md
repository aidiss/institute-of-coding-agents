## Incidents & Vulnerabilities

### Flowise AI Builder Active RCE Exploitation — CVE-2025-59528 (CVSS 10.0)

**Status:** Actively exploited (confirmed April 2026)
**Affected versions:** Flowise AI agent builder (all versions before patch)
**Fix:** Patch available; upgrade immediately

A critical code injection vulnerability in Flowise's CustomMCP node allows arbitrary JavaScript execution without input validation, granting access to `child_process` (command execution) and `fs` (filesystem) modules at full Node.js runtime privilege. Over 12,000 publicly exposed Flowise instances were identified at time of disclosure (September 2025). Active exploitation campaigns were confirmed in April 2026 by VulnCheck and SonicWall. Any publicly reachable Flowise deployment should be treated as compromised until patched and audited.

Sources: [VulnCheck](https://vulncheck.com) · [SonicWall Threat Intelligence](https://www.sonicwall.com/blog/threat-intelligence) · [The Hacker News](https://thehackernews.com/2026/04/flowise-ai-agent-builder-under-active.html)

### Cursor IDE "CurXecute" MCP Auto-Start RCE — CVE-2025-54135 (CVSS 8.6)

**Status:** Disclosed; patched in Cursor 1.3 (released July 29, 2025)
**Affected versions:** Cursor IDE < 1.3

A permission-check inconsistency in Cursor's MCP integration allows attackers to achieve RCE before the user is prompted for review. Creating a new `.cursor/mcp.json` file does not require approval, while editing an existing one does — an attacker can drop a fresh config via a malicious prompt or repository file, causing a crafted MCP server to auto-start and execute arbitrary code. The attack requires no elevated privileges and can be triggered from a poisoned prompt embedded in a document or repository the agent references during a session.

Cursor is in use by over 50% of Fortune 500 companies (including NVIDIA, Uber, and Adobe). The same research batch (IDEsaster) attributed additional CVEs — CVE-2025-49150, CVE-2025-54130, CVE-2025-61590 — to Cursor for related MCP-path injection issues.

Sources: [Tenable FAQ — CurXecute](https://www.tenable.com/blog/faq-cve-2025-54135-cve-2025-54136-vulnerabilities-in-cursor-curxecute-mcpoison) · [AIM Security](https://www.aim.security/post/when-public-prompts-turn-into-local-shells-rce-in-cursor-via-mcp-auto-start) · [IDEsaster — GBHackers](https://gbhackers.com/ai-developer-tools/)

### mcp-remote OS Command Injection — CVE-2025-6514 (CVSS 9.6)

**Status:** Disclosed; patched in mcp-remote 0.1.16
**Affected versions:** mcp-remote 0.0.5 – 0.1.15
**Fix:** Upgrade to mcp-remote ≥ 0.1.16

JFrog Security Research identified an OS command injection in the `mcp-remote` bridge package, which is widely used to connect local MCP clients to remote servers. The vulnerability is triggered by a crafted `authorization_endpoint` URL returned by a malicious MCP server during the OAuth handshake. An attacker who controls (or can MITM) a remote MCP server can achieve full OS-level code execution on the connecting client without any additional user interaction. This is the first confirmed RCE on a client machine achieved purely by connecting to an untrusted remote MCP server.

Sources: [JFrog — CVE-2025-6514 Analysis](https://jfrog.com/blog/2025-6514-critical-mcp-remote-rce-vulnerability/)

### Claude Code Config-Injection RCE and Key Exfiltration — CVE-2025-59536 / CVE-2026-21852

**Status:** Both patched before public disclosure (February 2026)
**Affected versions:** Claude Code (versions prior to February 2026 patch)

Check Point Research disclosed two related vulnerabilities in Claude Code:

- **CVE-2025-59536** (CVSS 8.7): Malicious hooks, MCP server registrations, and environment variable overrides embedded in a repository's `.claude/` config can trigger arbitrary shell commands and exfiltrate the `ANTHROPIC_API_KEY` when a developer clones and opens the repository.
- **CVE-2026-21852** (CVSS 5.3): Setting `ANTHROPIC_BASE_URL` to an attacker-controlled endpoint in a project settings file causes Claude Code to issue API requests — including auth headers — before the user-trust prompt appears, leaking API keys silently.

Check Point attributed 75% of all RCE activity observed in late December 2025 through mid-February 2026 to exploitation of these Claude Code flaws.

Sources: [Check Point Research](https://research.checkpoint.com/2026/rce-and-api-token-exfiltration-through-claude-code-project-files-cve-2025-59536/) · [The Hacker News](https://thehackernews.com/2026/02/claude-code-flaws-allow-remote-code.html)

### IDEsaster: 30+ Vulnerabilities Across All Major AI IDEs

**Status:** Disclosed; patch status varies per vendor and CVE
**Affected:** Cursor, GitHub Copilot, Windsurf, Kiro.dev, Zed.dev, Roo Code, JetBrains Junie, Cline, Gemini CLI (24 CVEs assigned)

A coordinated research disclosure found that 100% of tested AI-augmented IDEs are vulnerable to at least one data theft or RCE attack path. Common attack classes include indirect prompt injection via referenced documents, MCP server manipulation, configuration file injection, and LSP-layer exploitation. The breadth of the finding suggests the root cause is an ecosystem-wide assumption that content processed by an LLM within a developer tool can be trusted — an assumption consistently invalidated in practice.

## Supply Chain

### Rules File Backdoor: Hidden Instructions in Cursor and Copilot Config

**Status:** Disclosed (March 2025); GitHub added Unicode warning (May 2025); no universal fix

Pillar Security demonstrated that `.cursor/rules` and `.github/copilot-instructions.md` files — which govern agent behavior at the project level — can be weaponized via hidden Unicode characters (invisible control sequences and zero-width joiners). The resulting files look clean to human reviewers and pass most linters, but contain model-facing instruction payloads that redirect agent behavior: suppressing security warnings, adding backdoor code, or exfiltrating secrets to attacker endpoints.

Propagation vectors include developer forums posting "useful" rules files, pull requests to popular open-source repos, and poisoned project templates distributed via GitHub template repositories. GitHub responded by adding a visible warning for hidden Unicode text in May 2025, but the underlying model behavior is not changed by the warning.

Sources: [Pillar Security — Rules File Backdoor](https://www.pillar.security/blog/new-vulnerability-in-github-copilot-and-cursor-how-hackers-can-weaponize-code-agents/)

### Slopsquatting Scale Update: 121,539 Downloads of Phantom Packages

**Status:** Active, ongoing threat class

Updated tracking data adds specificity to the slopsquatting threat: researcher Bar Lanyado registered `huggingface-cli` (a name hallucinated by multiple LLMs) and observed **30,000+ downloads** within three months, with no legitimate package by that name existing. Across 128 monitored phantom package names, total downloads reached **121,539** between July 2025 and January 2026 (~4,000 per week), indicating that developers are actively installing hallucinated packages at scale.

LLM hallucination rates for package names:
- ChatGPT-4: ~5%
- Open-source models (CodeLlama, DeepSeek, WizardCoder, Mistral): significantly higher
- **58%** of hallucinated names reappear consistently across multiple queries — making them predictable and high-value targets for pre-registration by attackers

Sources: [Bleeping Computer](https://www.bleepingcomputer.com/news/security/ai-hallucinated-code-dependencies-become-new-supply-chain-risk/) · [Infosecurity Magazine — Slopsquatting](https://www.infosecurity-magazine.com/news/ai-hallucinations-slopsquatting/) · [Check Point Blog](https://blog.checkpoint.com/research/the-hidden-ai-threat-to-your-software-supply-chain/) · [Security Boulevard](https://securityboulevard.com/2026/03/coding-agents-widen-your-supply-chain-attack-surface/)

### Sonatype: 34% of AI-Suggested Dependencies Don't Exist

**Status:** Ongoing research finding (June–August 2025)

Sonatype's analysis of 258,000 dependency recommendations across 36,870 unique upgrade suggestions found:
- **34%** of suggested dependencies did not exist in any public registry (hallucinations)
- **44–49%** of AI-imported dependency versions had known vulnerabilities even when the package existed
- 7 models tested from Anthropic, OpenAI, and Google all exhibited the behavior

The finding implies that AI-assisted dependency management without a verification step is reliably worse than no AI assistance for supply chain safety.

Sources: [The Register — AI code suggestions sabotage supply chain](https://www.theregister.com/2025/04/12/ai_code_suggestions_sabotage_supply_chain/) · [Sonatype State of Dependency Management](https://www.sonatype.com/blog/compromised-litellm-pypi-package-delivers-multi-stage-credential-stealer)

The list prioritizes prompt injection (direct and indirect), insecure tool execution, insufficient sandboxing, and supply chain vulnerabilities in agent dependencies and MCP servers. ### AI-Generated Code Vulnerability Rate: 25% Across LLMs

Multiple independent studies converge on a consistent finding: roughly **25% of AI-generated code contains confirmed security vulnerabilities**:

- Veracode (100+ LLMs tested): 45% of samples introduced OWASP Top 10 vulnerabilities
- AppSec Santa 2026 (534 samples, 6 LLMs): 25.1% contained confirmed OWASP Top 10 flaws
- Georgia Tech Vibe Security Radar: 74 CVEs directly attributable to AI coding tools tracked to date

Model variance is significant — GPT-5.2 scored best (~19% flaw rate); DeepSeek V3, Claude Opus 4.6, and Llama 4 Maverick scored worst (~29%). The consistent finding across methodologies suggests the vulnerability rate is a property of current LLM code generation, not of specific models or prompting strategies.

Sources: [Veracode GenAI Code Security Report](https://www.veracode.com/blog/genai-code-security-report/) · [CSA Research Note — AI-Generated CVE Surge](https://labs.cloudsecurityalliance.org/research/csa-research-note-ai-generated-code-vulnerability-surge-2026/) · [SQ Magazine — AI Coding Vulnerability Statistics 2026](https://sqmagazine.co.uk/ai-coding-security-vulnerability-statistics/)

### MCP Sampling Feature as Privilege Escalation Vector

Palo Alto Unit 42 published research on attack paths through MCP's *sampling* capability, which allows MCP servers to request LLM completions from the host client. Without rate limiting, scope restrictions, or output validation on the server side, a malicious MCP server can use repeated sampling requests to extract full conversation context, enumerate connected tool capabilities, and escalate privileges within an active agent session — all without any user-visible action.

Sources: [Unit 42 — MCP Attack Vectors](https://unit42.paloaltonetworks.com/model-context-protocol-attack-vectors/) · [Aembit — MCP Security Guide 2026](https://aembit.io/blog/the-ultimate-guide-to-mcp-security-vulnerabilities/)

### 492 MCP Servers Exposed to Internet With No Authentication

Trend Micro scanned internet-facing MCP servers and found **492 instances** with no authentication whatsoever, accepting arbitrary tool calls from any client. A significant subset of these are production deployments providing filesystem, shell, database, or cloud-provider access. Unauthenticated MCP servers represent a new class of exposed internal tooling comparable to unauthenticated Redis or Elasticsearch instances in prior attack waves.

Sources: [Trend Micro — Slopsquatting and MCP Exposure](https://www.trendmicro.com/vinfo/us/security/news/cybercrime-and-digital-threats/slopsquatting-when-ai-agents-hallucinate-malicious-packages) · [AI Agent Security Risks 2026](https://blog.cyberdesserts.com/ai-agent-security-risks/)
