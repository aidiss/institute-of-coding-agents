# Security Brief — 2026-03-19

Track of vulnerabilities, supply chain attacks, and safety incidents affecting AI coding agents.

## Incidents & Vulnerabilities

### IDEsaster — 30+ CVEs Across AI IDEs (December 2025)

Researcher Ari Marzouk disclosed 30+ vulnerabilities across 10+ AI coding tools over six months of research. 24 CVEs were assigned; AWS issued advisory AWS-2025-019. Affected tools include GitHub Copilot, Cursor, Windsurf, Kiro.dev, Zed.dev, Roo Code, JetBrains Junie, Cline, Gemini CLI, and Claude Code.

The attack chain is three-stage: **Prompt Injection → IDE Tools → Base IDE Features**. Two dominant exploit classes emerged:

**Workspace config override → RCE** (patched in all affected tools):

| CVE | Tool | Status |
|---|---|---|
| CVE-2025-64660 | GitHub Copilot | Patched |
| CVE-2025-61590 | Cursor | Patched (v1.3.9+) |
| CVE-2025-58372 | Roo Code | Patched |

**JSON schema poisoning → data exfiltration**: Injected prompt causes the AI to read a sensitive file and write a JSON document referencing a remote attacker-controlled schema. When the IDE fetches the schema, file contents leak via the GET request.

| CVE | Tool | Status |
|---|---|---|
| CVE-2025-49150 | Cursor | Patched |
| CVE-2025-53097 | Roo Code | Patched |
| CVE-2025-58335 | JetBrains Junie | Patched |
| — | GitHub Copilot | Security warning added |
| — | Claude Code | Security warning added |

### CVE-2025-53773 — GitHub Copilot / Visual Studio RCE (CVSS 7.8 HIGH)

Prompt injection (via source files, web pages, GitHub issues, or hidden Unicode characters) tricks Copilot into writing `"chat.tools.autoApprove": true` to `.vscode/settings.json`, enabling YOLO mode — full auto-execution including shell commands, file writes, and network access.

- **Disclosed:** June 29, 2025; publicly disclosed August 12, 2025
- **Patched:** August 2025 Patch Tuesday; Visual Studio 2022 v17.14.12
- **Status:** 7 public PoC repos on GitHub; mass exploitation not confirmed

### CVE-2025-61260 — OpenAI Codex CLI Command Injection

Codex CLI implicitly trusted MCP server entries and executed configured commands at startup without user confirmation. Exploitable if an attacker could modify `.env` or `.codex/config.toml` in the repository.

- **Patched:** Codex CLI 0.23.0 (less than two weeks after report)

### PromptPwnd — CI/CD Pipeline Attacks

Prompt injection in GitHub Actions and GitLab CI/CD workflows exploiting Gemini CLI, Claude Code, and OpenAI Codex — enabling secret exfiltration, RCE, and GitHub token theft. At least 5 Fortune 500 companies confirmed impacted. Google patched Gemini CLI within 4 days of disclosure.

- **Status:** Actively exploited in the wild

### EchoLeak — Microsoft 365 Copilot Zero-Click Exfiltration

Zero-click prompt injection attack: a crafted email causes Copilot to autonomously extract data from OneDrive, SharePoint, and Teams and exfiltrate it via trusted Microsoft domains — no user interaction required.

- **Status:** Patched by Microsoft before evidence of mass exploitation

### Rules File Backdoor — Cursor / GitHub Copilot (Pillar Security)

Malicious instructions injected into `.cursorrules` or equivalent config files via hidden Unicode characters cause the AI to silently embed backdoors or data exfiltration code in generated output.

- **Status:** Demonstrated and weaponized; GitHub added hidden Unicode warnings (May 2025)

### OpenClaw AI Agent — Multiple Critical Vulnerabilities (Early 2026)

Open-source AI agent (135,000+ GitHub stars, 21,000+ exposed instances) disclosed with multiple critical vulnerabilities and malicious marketplace exploits. CNCERT issued a warning; China restricted use on government systems.

## Supply Chain

### Slopsquatting — Hallucinated Dependencies as Attack Surface

A landmark March 2025 study of 576,000+ generated Python and JavaScript code samples found **~20% referenced non-existent packages**:

- GPT-4 class models: ~5.2% hallucination rate
- Open-source models (CodeLlama, DeepSeek, WizardCoder, Mistral): ~21.7%
- 43% of hallucinated packages recurred across 10 queries — hallucinations are consistent and systematically exploitable

Sonatype's 2026 report found **28% of LLM-assisted dependency upgrade suggestions recommended non-existent package versions**. Aikido Security (2026) attributed 1 in 5 breaches to AI-generated code.

**Real-world exploitation:**
- Threat actor "_Iain" automated creation of thousands of typosquatted packages targeting crypto libraries using ChatGPT to generate realistic package name variants; shared public tutorial videos
- Google AI Overview suggested malicious npm package `@async-mutex/mutex` (typosquatting `async-mutex`)
- **Shai-Hulud npm campaign (November 24, 2025):** Wave of compromised npm packages designed to leak GitHub tokens, CI/CD credentials, and cloud credentials; Unit 42 assessed AI was used to generate the malicious payload based on code style, comments, and emoji use

### Smithery MCP Registry Path Traversal (GitGuardian)

Path traversal in `smithery.yaml` build config (`dockerBuildPath: ".."`) leaked the builder's `~/.docker/config.json`, including a Fly.io API token controlling 3,000+ apps (mostly hosted MCP servers). Supply chain risk: compromise of the registry propagates to all hosted servers.

## Research & Advisories

### MCP Protocol Vulnerabilities

MCP (Model Context Protocol) has emerged as a high-density attack surface. Analysis of 67,057 MCP servers across 6 public registries found substantial numbers hijackable due to absent vetting. Approximately 7,000 MCP servers are open on the public internet.

| CVE | Component | Type | Status |
|---|---|---|---|
| CVE-2025-68145 | Anthropic mcp-server-git | Path validation bypass | — |
| CVE-2025-68143 | Anthropic mcp-server-git | Unrestricted `git_init` | — |
| CVE-2025-68144 | Anthropic mcp-server-git | Argument injection in `git_diff` | — |
| CVE-2025-6514 | mcp-remote (JFrog) | Critical OS command injection | — |
| CVE-2025-64106 | Cursor IDE MCP | Deep-link RCE (CVSS 8.8) | Patched (v1.3.9+) |
| CVE-2025-54135 | Cursor IDE MCP | Trust abuse → RCE | Patched (v1.3.9+) |
| CVE-2025-54136 | Cursor IDE MCP | Trust abuse → RCE | Patched (v1.3.9+) |

**CVE-2025-68143/44/45 — mcp-server-git chain:** Three vulnerabilities chainable with the Filesystem MCP server to achieve full RCE via malicious `.git/config` files.

**CVE-2025-6514 — mcp-remote:** Critical OS command injection in a popular OAuth proxy for local MCP clients. Windows: arbitrary command execution with full parameter control; macOS/Linux: arbitrary executable execution with limited parameter control.

**Anthropic MCP Inspector RCE:** The developer tool allowed unauthenticated RCE via its inspector-proxy architecture. A malicious MCP server could trigger arbitrary command execution on the developer's machine; inspector ran with user privileges, no auth, listening on localhost/0.0.0.0.

**MCP Tool Poisoning / Rug Pull:** MCP tool definitions can be mutated silently after installation. A tool approved as safe on Day 1 can redirect API keys by Day 7; not monitored by traditional security tooling. WhatsApp MCP exfiltration demonstrated: malicious server combined with legitimate `whatsapp-mcp` server causes silent exfiltration of complete message history.

**MCP Sampling Exploitation (Unit 42 / Palo Alto):** Three attack classes using MCP's sampling feature — (1) covert tool invocation for hidden file/system operations, (2) conversation hijacking to inject persistent cross-session instructions, (3) resource theft to drain AI compute quotas.

**Microsoft MarkItDown MCP Server — SSRF (Unpatched):** SSRF vulnerability allows the server to fetch arbitrary URLs without validation; AWS EC2 metadata endpoint exploitation demonstrated for cloud credential theft. Microsoft classified as low-risk; unpatched as of reporting.

### Academic Research

- **arXiv 2601.17548 (January 2026):** Systematic analysis of prompt injection across MCP, tool ecosystems, and agentic coding assistants. Meta-analysis of 78 studies (2021–2026): attack success rates against state-of-the-art defenses exceed **85%** with adaptive strategies. Risk ratings: Claude Code (Low — mandatory confirmation, sandboxed MCP, permission prompts) vs. Cursor (Critical — auto-approve available, MCP unsandboxed, `.cursorrules` unvalidated, no egress controls).
- **arXiv 2510.26103 (October 2025):** Large-scale GitHub analysis; 87.9% of AI-generated code does not contain identifiable CWE-mapped vulnerabilities, though language-specific patterns vary significantly by tool.
- **arXiv 2501.19012:** "Importing Phantoms: Measuring LLM Package Hallucination Vulnerabilities" — comprehensive analysis of the slopsquatting attack surface.
- **Veracode 2025 GenAI Code Security Report:** 45% of AI-generated code contains security flaws. Yet 76% of developers believe AI tools produce more secure code than humans.
- **Tenzai IDEsaster Broader Findings:** 69 vulnerabilities across 15 test applications in 5 AI coding platforms; 6 critical. Dominant issue class: business logic and authorization failures (IDOR, missing admin checks, negative pricing), not classic SQLi/XSS.

### Microsoft Security Blog (March 12, 2026)

Published "Detecting and analyzing prompt abuse in AI tools" — guidance on identifying and responding to prompt injection campaigns targeting enterprise AI deployments.
