**2026-04-16**

## Incidents & Vulnerabilities

### Flowise CVE-2025-59528 (CVSS 10.0) — Actively Exploited

**Status:** Actively exploited in the wild  
**Affected versions:** Flowise >= 2.2.7-patch.1 and < 3.0.6  
**Fix:** Upgrade to 3.1.1 or 3.0.6+  
**Exposed instances:** 12,000–15,000 internet-facing

Flowise, an open-source platform for building LLM applications and AI agent pipelines, contains a maximum-severity code injection flaw in its **CustomMCP node**. The node passes user-supplied `mcpServerConfig` directly to JavaScript's `Function()` constructor — functionally equivalent to `eval()` — without sanitisation. Successful exploitation grants access to Node.js `child_process` and `fs` modules, enabling arbitrary command execution, filesystem access, and exfiltration of API keys, credentials, and configuration secrets.

VulnCheck detected first in-the-wild exploitation on **April 7, 2026**, originating from a single Starlink IP. CSA confirmed the flaw provides a direct bridge from MCP configuration input to full host compromise.

Sources: [The Hacker News](https://thehackernews.com/2026/04/flowise-ai-agent-builder-under-active.html) · [Bleeping Computer](https://www.bleepingcomputer.com/news/security/max-severity-flowise-rce-vulnerability-now-exploited-in-attacks/) · [Security Affairs](https://securityaffairs.com/190471/security/attackers-exploit-critical-flowise-flaw-cve-2025-59528-for-remote-code-execution.html) · [SonicWall](https://www.sonicwall.com/blog/flowiseai-custom-mcp-node-remote-code-execution-) · [CSA Labs](https://labs.cloudsecurityalliance.org/research/csa-research-note-flowise-mcp-rce-exploitation-20260409-csa/)

### Langflow CVE-2026-33017 — CISA KEV, Federal Deadline Passed April 8

**Status:** Actively exploited; CISA KEV listed  
**CVSS:** Critical  
**Fix:** Upgrade to Langflow 1.9.0+; federal remediation deadline was April 8, 2026

Langflow, one of the most widely deployed open-source AI agent workflow frameworks, contains an unauthenticated code injection flaw in its `POST /api/v1/build_public_tmp/{flow_id}/flow` endpoint. The endpoint accepts attacker-supplied flow data containing arbitrary Python in node definitions, which executes server-side without sandboxing.

Weaponisation was exceptionally fast: the Sysdig Threat Research Team observed the first exploitation attempts **within 20 hours of advisory publication**, before any public PoC existed. Attackers derived working exploits directly from the advisory text and began mass-scanning.

CISA added the flaw to its Known Exploited Vulnerabilities catalogue and directed federal agencies to patch or stop using the product by **April 8, 2026**.

Sources: [Sysdig](https://www.sysdig.com/blog/cve-2026-33017-how-attackers-compromised-langflow-ai-pipelines-in-20-hours) · [Bleeping Computer](https://www.bleepingcomputer.com/news/security/cisa-new-langflow-flaw-actively-exploited-to-hijack-ai-workflows/) · [The Hacker News](https://thehackernews.com/2026/03/critical-langflow-flaw-cve-2026-33017.html) · [Qualys ThreatPROTECT](https://threatprotect.qualys.com/2026/03/26/cisa-added-langflow-vulnerability-to-its-known-exploited-vulnerabilities-catalog-cve-2026-33017/)

### Microsoft Azure DevOps MCP — CVE-2026-32211 (CVSS 9.1)

**Status:** Disclosed April 3, 2026  
**Affected:** `@azure-devops/mcp` — all versions prior to patch  
**Type:** Missing authentication on sensitive endpoints

The Microsoft Azure DevOps MCP server exposed privileged endpoints without authentication, allowing unauthenticated callers to interact with DevOps project resources. Combined with prompt injection, this creates a path from a malicious repository or issue description to full Azure DevOps project compromise without user interaction.

### mcp-server-kubernetes Argument Injection — CVE-2026-39884

**Status:** Patched in v3.5.0  
**Affected:** Flux159 `mcp-server-kubernetes` < 3.5.0  
**Type:** CWE-88 Argument injection

The `port_forward` tool concatenates user-controlled fields (`namespace`, `resourceType`, `resourceName`, `localPort`, `targetPort`) into a `kubectl` command string and splits on spaces before calling `spawn()`, unlike other tools that use safe array-based argument passing with `execFileSync()`. This allows injection of arbitrary `kubectl` flags.

Impact includes exposure of internal Kubernetes services to the network and cross-namespace access. The vulnerability is additionally exploitable via prompt injection against any AI agent connected to the MCP server, meaning a malicious repository or document could pivot into cluster access.

Sources: [OffSeq Radar](https://radar.offseq.com/threat/cve-2026-39884-cwe-88-improper-neutralization-of-a-0e46c7ee) · [THREATINT](https://cve.threatint.eu/CVE/CVE-2026-39884)

### OpenCode AI — CVE-2026-22813 (XSS via AI-Generated Markdown)

**Status:** Disclosed; patch status unclear  
**Affected:** OpenCode web interface  
**Type:** Stored XSS via unsafe markdown rendering

The OpenCode browser-based AI coding agent UI renders model output without sanitisation, allowing attacker-controlled content in AI responses to execute JavaScript in the localhost web interface. Because the UI communicates with backend automation features, XSS here can be a stepping stone to local file access or command execution via the agent's backend tooling.

Sources: [PointGuard AI](https://www.pointguardai.com/ai-security-incidents/opencode-ai-ui-turns-chat-output-into-code-cve-2026-22813)

## Supply Chain

### OpenClaw Framework — ClawHavoc Campaign (Ongoing)

**Status:** Active supply chain campaign  
**CVEs:** 138 total (7 critical, 49 high); anchor: CVE-2026-25253 (CVSS 8.8, one-click RCE)  
**Exposed instances:** 21,000–40,000+ internet-facing

OpenClaw, an open-source AI agent framework with 135,000+ GitHub stars, is the centre of a multi-vector supply chain crisis:

- **CVE-2026-25253** (CVSS 8.8): RCE chain exploitable against localhost-bound instances; nine follow-on CVEs within four days.
- **ClawHavoc campaign:** Koi Security identified 341 malicious entries in ClawHub (the skills marketplace), 335 traced to a single coordinated threat operation. As of February 16, 2026, confirmed malicious skills exceeded **824** across 10,700+ total entries.
- **Npm pivot:** A compromised npm package was observed silently installing OpenClaw on developer machines without consent.
- A September 2025 BlueRock Security scan of 7,000+ MCP servers found **36.7% potentially vulnerable to SSRF**.

### LiteLLM PyPI Compromise — Ongoing Remediation Context

**CVE:** CVE-2026-33634 (CVSS 9.4)  
**Threat actor:** TeamPCP  
**Affected:** litellm 1.82.7 and 1.82.8 (on PyPI March 24, 2026 for ~40 minutes)  
**Status:** Packages quarantined; remediation ongoing in affected environments

Although the malicious PyPI packages were quarantined within 40 minutes of publication, with LiteLLM downloaded ~3.4 million times per day the blast radius is significant. The flaw was discovered not by any automated security tooling but accidentally by a researcher testing a **Cursor MCP plugin** that pulled in LiteLLM as a transitive dependency.

Indicators of compromise:
- File `~/.config/sysmon/sysmon.py` and associated systemd unit
- `kube-system` namespace pods named `node-setup-*`
- `.pth` file `litellm_init.pth` in Python environment `site-packages`

Sources: [Trend Micro](https://www.trendmicro.com/en_us/research/26/c/inside-litellm-supply-chain-compromise.html) · [FutureSearch](https://futuresearch.ai/blog/litellm-pypi-supply-chain-attack/) · [Snyk](https://snyk.io/blog/poisoned-security-scanner-backdooring-litellm/) · [ARMO](https://www.armosec.io/blog/litellm-supply-chain-attack-backdoor-analysis/) · [Kaspersky](https://www.kaspersky.com/blog/critical-supply-chain-attack-trivy-litellm-checkmarx-teampcp/55510/)

### Automated Dependency Side-Loading — New Attack Class

A newly documented attack class targets the intersection of AI coding tools and browser/IDE extensions. A compromised or malicious extension monitors active coding sessions; when an AI tool generates code, the extension silently modifies the suggestion to substitute an attacker-controlled dependency for the legitimate one. No package hallucination required — the AI suggests a real package; the extension swaps it.

This bypass entirely circumvents slopsquatting defences focused on hallucinated names and operates below the visibility of most dependency scanning tools.

Sources: [Medium / InstaTunnel](https://medium.com/@instatunnel/automated-dependency-side-loading-the-invisible-supply-chain-attack-via-ai-extensions-fe615eb03f19) · [Security Boulevard](https://securityboulevard.com/2026/03/coding-agents-widen-your-supply-chain-attack-surface/)

### Slopsquatting / Phantom Packages — Growing Exploitation Window

128 tracked "phantom packages" (hallucinated names registered by attackers on PyPI/npm) accumulated **121,539 downloads** between July 2025 and January 2026, averaging ~4,000 downloads per week. The consistent repeatability of hallucinations across re-runs (58% of hallucinated names repeated across 10 identical prompt runs) makes pre-registration viable and scalable.

Autonomous coding agents that execute `pip install` without human review are the highest-risk surface: a single hallucinated transitive dependency can reach production at machine speed.

Sources: [Bleeping Computer](https://www.bleepingcomputer.com/news/security/ai-hallucinated-code-dependencies-become-new-supply-chain-risk/) · [InstaTunnel](https://instatunnel.my/blog/ai-hallucination-squatting-the-new-frontier-of-supply-chain-attacks) · [Capitol Technology University](https://www.captechu.edu/blog/ai-driven-threats-in-software-supply-chains)

### MCP Ecosystem — 30 CVEs in 60 Days

A Hacker News–linked post and associated analysis documented that the MCP ecosystem accumulated **30 CVEs in 60 days** as researchers turned coordinated attention to the protocol. The [Vulnerable MCP Project](https://vulnerablemcp.info/) now tracks 50+ documented MCP vulnerabilities (13 critical) across 32 researchers — functioning as an evolving CVE registry specific to the MCP ecosystem.

Key systemic findings:
- Session IDs mandated in URLs — violates security best practices and enables session hijacking.
- "Tool poisoning" (silent post-approval modification of server tool definitions) requires no re-consent.
- MCP sampling mechanism allows malicious servers to influence intermediate model reasoning steps without triggering tool-call consent dialogs (Palo Alto Unit 42).

## Key Mitigations (Cross-Cutting)

1. **Flowise / Langflow operators:** Upgrade immediately; disable unauthenticated public endpoints and do not expose instances to the internet without authentication.
2. **Kubernetes MCP users:** Upgrade `mcp-server-kubernetes` to 3.5.0+; restrict port-forward tool to trusted namespaces only.
3. **Azure DevOps MCP users:** Apply available patches for CVE-2026-32211; audit endpoint authentication configurations.
4. **LiteLLM users:** Confirm installed version is not 1.82.7 or 1.82.8; scan for IOCs listed above.
5. **All CI/CD pipelines:** Pin GitHub Action dependencies to commit SHAs; enable PyPI Trusted Publishers (OIDC) to eliminate long-lived publish tokens.
6. **AI agent package installs:** Verify every AI-suggested package name before install; use lockfiles, hash pinning, and a dependency scanner. Treat any extension that modifies AI suggestions as a potential side-loader.
7. **MCP deployments:** Treat MCP server tool definitions as untrusted; re-verify on every version change; monitor for tool poisoning (post-approval definition changes).
8. **Repository hygiene:** Audit `.claude/settings.json`, `.mcp.json`, and `.cursor/` configs before opening any new repository.
