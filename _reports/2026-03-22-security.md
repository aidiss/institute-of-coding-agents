## Incidents & Vulnerabilities

### IDEsaster — Systemic Vulnerability Class Across AI IDEs

**Status: Disclosed. Partially patched (vendor-dependent). No confirmed mass exploitation.**

Security researcher Ari Marzouk (MaccariTA) documented "IDEsaster," a systemic vulnerability class spanning 10+ AI coding products. All tested products were found vulnerable. **24 CVEs assigned**; AWS issued advisory AWS-2025-019.

Root cause: AI IDEs have no mechanism to distinguish attacker-controlled content from developer instructions. The attack chain is: **Prompt Injection → Agent Tools → Base IDE Features**.

Primary attack vectors:
- Remote JSON Schema attacks — automatic GET requests to attacker-controlled domains leak `.env` files and credential stores.
- IDE Settings Overwrite — prompt injection writes malicious config to `.vscode/settings.json` or `.idea/workspace.xml`, achieving RCE.
- Multi-Root Workspace manipulation — VS Code workspace configs abused to bypass security controls.

Affected products: GitHub Copilot, Cursor, Windsurf, Kiro.dev, Zed.dev, Roo Code, JetBrains Junie, Cline, Gemini CLI, Claude Code.

**Additional finding:** Cursor and Windsurf inherit 94+ known vulnerabilities from outdated Chromium/Electron builds. CVE-2025-7656 (patched upstream, unpatched in both IDEs) was successfully weaponized against ~1.8 million developers.

### Claude Code — CVE-2025-59536 & CVE-2026-21852 (Check Point Research)

**Status: Disclosed Feb–Mar 2026. Both CVEs fully patched. No confirmed active exploitation.**

**CVE-2025-59536** (CVSS 8.7) — MCP consent bypass / shell command execution. Two repository-controlled configuration settings could override Claude Code's consent safeguards and auto-approve all MCP servers, allowing arbitrary shell commands to execute when a developer opened an untrusted repository. Reported 2025-09-03; patched 2025-09-22; CVE published 2025-10-03.

**CVE-2026-21852** (CVSS 5.3) — API key exfiltration. Setting `ANTHROPIC_BASE_URL` in a repo's settings file to an attacker-controlled endpoint caused Claude Code to issue API requests — including the full `Authorization` header with the API key — before the trust prompt appeared. Reported 2025-10-28; patched 2025-12-28; CVE published 2026-01-21.

A third flaw (no CVE, CVSS 8.7): malicious Hooks in `.claude/settings.json` executed shell commands before any trust dialog. Published as GHSA-ph6w-f82w-28w6 on 2025-08-29.

**Fix:** Update to the latest Claude Code release. Anthropic has announced additional hardening.

### GitHub Copilot — CVE-2025-53773 (Detail)

**Status: Disclosed Aug 2025. Patched. 7 public PoC exploits available. Not on CISA KEV.**

Prompt injection in source files, web pages, or GitHub issues could manipulate Copilot into writing `"chat.tools.autoApprove": true` to `.vscode/settings.json` (YOLO mode), disabling all user confirmations. From there: shell command execution, repository propagation ("AI viruses"), and botnet recruitment ("ZombAI"). CVSS 7.8 HIGH. Affected versions: Visual Studio prior to Aug 2025 Patch Tuesday.

### Devin AI — Prompt Injection / ZombAI / Secret Leakage

**Status: Disclosed Apr 2025. Vendor did not respond after 120+ days. Patch status unknown — treat as unpatched.**

Research ($500 in API credits) demonstrated prompt injection via a website or GitHub issue could cause Devin to download and execute malware, turning it into a remotely controlled "ZombAI." Additional vectors: image rendering in chat leaks data to third-party servers; `expose_port` tool can be triggered by prompt injection to open attacker-accessible ports. No fix confirmed.

### AI Code Quality: Persistent Security Flaws in Generated Code

A Dec 2025 study (Tenzai) across Claude Code, Cursor, Windsurf, Replit, and Devin found all five generated 69 vulnerabilities (6 critical) — primarily business logic flaws: IDOR, authorization failures, negative pricing. Veracode's 2025 GenAI Code Security Report: ~45% of AI-generated code contains security flaws, with no improvement observed in newer or larger models across 100+ LLMs and 80 coding tasks.

## Supply Chain

### Slopsquatting — Hallucinated Package Names as Attack Vectors

**Status: Actively exploited. No CVEs. No central patch mechanism.**

LLMs hallucinate package names at scale. Attackers register malicious packages under predicted hallucinated names on npm/PyPI and wait for installs.

**Scale (USENIX Security 2025, Spracklen et al., UTSA/Virginia Tech/Oklahoma):**
- 576,000 code samples; 16 LLMs analyzed.
- 19.7% of all recommended packages were hallucinated (440,445 unique names).
- Open-source models (CodeLlama, DeepSeek, WizardCoder, Mistral): ~21.7% hallucination rate.
- Commercial models (GPT-4): ~5.2% hallucination rate.
- 43% of hallucinated names reappear predictably across similar prompts — enabling pre-registration attacks.

**Confirmed incidents:**
- `huggingface-cli` (PyPI, early 2024 / ongoing): Researcher registered hallucinated name; 30,000+ authentic installs in three months. Alibaba copy-pasted the hallucinated install command into a public repo README, creating downstream supply chain risk.
- `react-codeshift` (npm, Jan 2026): Hallucinated conflation of `jscodeshift` + `react-codemod`. Found spreading through real AI infrastructure with actual agents attempting installation. No human review in the commit loop.
- S1ngularity / Shai-Hulud (npm, Oct 2025): Two AI-enabled credential-stealing supply chain attacks compromised npm packages.
- Cline/OpenClaw (Feb 2026): Prompt injection in Claude-powered GitHub Actions issue triage led to a compromised npm package installing a persistent daemon on ~4,000 developer machines, exposing credentials and SSH keys.

**Combination attack:** Slopsquatting + Dependency Confusion — registering hallucinated names on public registries that shadow internal private package names.

**Mitigations:** Verify every AI-recommended package before installing (check registry existence, download counts, creation date); use lockfiles and hash pinning; implement SBOM generation; prompt the model to review its own package suggestions (GPT-4 Turbo, DeepSeek self-detect >75% of their hallucinations when asked).

### MCP Security — Expanding Critical Attack Surface

**Status: Multiple critical CVEs. Active exploitation confirmed. Rapidly growing attack surface.**

| CVE | Component | Type | CVSS | Status |
|---|---|---|---|---|
| CVE-2026-23744 | MCPJam Inspector ≤1.4.2 | RCE — listens on 0.0.0.0, no auth, zero-click | 9.8 CRITICAL | Patch status unclear. |
| CVE-2025-68145 | Anthropic mcp-server-git | Path validation bypass | CRITICAL | Part of 3-CVE RCE chain. |
| CVE-2025-68143 | Anthropic mcp-server-git | Unrestricted `git_init` (can turn `.ssh` into a git repo) | CRITICAL | Part of 3-CVE RCE chain. |
| CVE-2025-68144 | Anthropic mcp-server-git | Argument injection in `git_diff` → RCE via `.git/config` | CRITICAL | Part of 3-CVE RCE chain. |
| CVE-2026-25536 | MCP TypeScript SDK v1.10.0–1.25.3 | Cross-client data leak | 7.1 HIGH | Patched. |
| CVE-2025-6514 | mcp-remote v0.0.5–0.1.15 | OS command injection connecting to untrusted servers | 9.6 CRITICAL | First documented real-world MCP RCE. |
| CVE-2025-49596 | MCP Inspector | CSRF → RCE via crafted webpage | — | Patched. |
| CVE-2025-53967 | Figma MCP Server | Command injection → RCE | — | Patched. |

**Infrastructure survey findings:** 492 MCP servers found with no auth or encryption; 36.7% of 7,000+ analyzed servers vulnerable to SSRF; 43% vulnerable to command injection; 22% had arbitrary file read via path traversal.

**Tool Poisoning / Rug Pull attacks:** Malicious instructions embedded inside MCP tool *descriptions* are invisible to users but executed by the AI. A "rug pull" modifies tool definitions after initial approval — the protocol does not re-trigger consent flows on changes. MCPTox benchmark: o1-mini has 72.8% attack success rate against tool poisoning; more capable models are *more* susceptible.

**Confirmed breach:** Supabase's Cursor agent (mid-2025) processed support tickets containing user-supplied SQL as commands. Attackers embedded SQL to read and exfiltrate integration tokens via a privileged service-role connection.

**Protocol design flaws:** Session IDs mandated in URLs (session hijacking); no required authentication standard; no message signing or verification.

**Mitigations:** Use `mcp-scan` for tool description hash pinning (alerts on changes); sandbox MCP clients/servers in containers; apply least-privilege; pin MCP server versions; require human approval for sensitive operations; monitor for behavioral anomalies (e.g., a math tool making network calls). See [OWASP MCP Top 10](https://owasp.org/www-project-top-10-for-large-language-model-applications/).

## Summary

- **No AI coding tool passed security testing in 2025–2026.** IDEsaster affects every major AI IDE — this is an architectural issue, not a product-specific bug.
- **Prompt injection remains unresolved at the model level.** It is OWASP LLM01:2026. All vendors acknowledge it; none have eliminated it.
- **MCP dramatically expands the attack surface.** Tool poisoning, rug pulls, and cross-client data leaks are real, exploited vulnerabilities.
- **Slopsquatting is actively occurring.** Real agents installed hallucinated npm packages in Jan 2026 with no human review.
- **Recommended immediate actions:** Update Cursor to 1.3.9+, VS Code to 1.106.2+, and Claude Code to latest. Treat repositories from unknown sources as untrusted. Pin MCP server versions and deploy `mcp-scan`. Verify all AI-recommended packages before installation.
