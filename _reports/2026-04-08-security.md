# Security Brief — AI Coding Agents
**2026-04-08**

---

## Incidents & Vulnerabilities

### Azure AI Foundry — Critical Privilege Escalation (CVE-2026-32213, CVSS 10.0)

Disclosed April 3, 2026. An improper authorization flaw in Azure AI Foundry allows an **unauthenticated remote attacker** to elevate privileges over the network — no credentials required. The exact vulnerable endpoint is not yet public. Microsoft confirmed no customer action is required; the issue has been silently patched on the managed service side. Relevant to any enterprise using Azure AI Foundry for agentic coding pipelines, where a compromised Foundry instance could pivot to downstream code generation or deployment credentials.

**Status: Patched by Microsoft. No customer action needed.**

---

### Agent Framework RCE Wave — PraisonAI, Flowise, Langflow, CrewAI, Semantic Kernel

A cluster of critical RCE vulnerabilities was disclosed across popular AI agent orchestration frameworks in late March–early April 2026:

| CVE | Product | CVSS | Description | Status |
|-----|---------|------|-------------|--------|
| CVE-2026-34938 | PraisonAI | Critical | `execute_code()` runs attacker-controlled Python with no sandboxing | Patched |
| CVE-2025-59528 | Flowise | 10.0 | Code injection in open-source AI agent builder → RCE; **actively exploited** | Patched |
| CVE-2026-0768 | Langflow | Critical | Remote code execution via unauthenticated endpoint | Patched |
| CVE-2026-26030 | Microsoft Semantic Kernel | High | CWE-94: improper code generation control | Patched |

**CrewAI:** Four chained CVEs (unassigned as of report date) allow prompt injection to chain into RCE, SSRF, and arbitrary file read via the Code Interpreter and default agent configurations. Affected versions unspecified; upstream fix in progress.

Flowise CVE-2025-59528 is the most urgent: active exploitation is confirmed in the wild. Any internet-exposed Flowise deployment should be patched immediately and audited for IOCs.

### RoguePilot — GitHub Copilot / Codespaces GITHUB_TOKEN Exfiltration

Disclosed by Orca Security in February 2026; re-amplified by continued exploitation reports in April. Attack chain:

1. Attacker files a GitHub Issue containing a malicious Copilot instruction.
2. A developer asks Copilot in a Codespace to review the issue.
3. Copilot checks out a crafted PR containing a symbolic link to an internal file.
4. Copilot reads the symlinked file content and, via a remote JSON `$schema` reference, exfiltrates a privileged `GITHUB_TOKEN` to an attacker-controlled server.
5. Attacker uses the token for repository takeover (push to protected branches, modify Actions secrets, inject malicious CI steps).

**No special permissions required for the attacker** — only the ability to open a GitHub Issue. Patched by Microsoft following responsible disclosure.

**Status: Patched.**

---

### prt-scan Campaign — AI-Generated Malware in GitHub Pull Requests (Actively Exploited)

Wiz Research documented six waves of supply-chain activity from threat actor **ezmtebo** beginning March 11, 2026, with public disclosure April 2, 2026. The actor opened **475+ malicious PRs across 26 hours** targeting open-source repositories with `pull_request_target` workflows.

Key escalation from prior campaigns: attackers used **LLMs to generate repository-aware wrapper code** that adapts per tech stack (Go test files for Go repos, `conftest.py` for Python, npm scripts for JavaScript). Surrounding commits — documentation tweaks, version bumps, minor refactors — were stylistically consistent with each target project, strongly suggesting LLM-assisted social engineering of reviewers.

At least **two npm packages confirmed compromised**. Estimated remediation cost: $50,000–$500,000 per affected project.

**Status: Ongoing. GitHub has suspended the identified accounts; new waves expected.**

## Supply Chain

### LiteLLM Follow-Up: PyPI Incident Report & April Hardening CVEs

PyPI published its official incident report on April 2, 2026 covering the March 24 LiteLLM/Telnyx compromise (threat actor TeamPCP, via poisoned Trivy CI action). New post-audit vulnerability disclosures from Veria Labs:

| CVE | Severity | Description | Condition |
|-----|----------|-------------|-----------|
| CVE-2026-35030 | Critical | Authentication bypass in JWT auth pathway | Only if `enable_jwt_auth=true` (off by default) |
| CVE-2026-35029 | High | Privileged escalation via valid API key | Requires valid API key; not remotely exploitable without auth |
| GHSA-69x8-hrgq-fjj8 | High | Second authenticated escalation path | Same condition as above |

No LiteLLM Cloud customers had JWT auth enabled; self-hosted deployments with `enable_jwt_auth=true` should upgrade immediately to **v1.83.0+**.

LiteLLM launched a formal bug bounty program with bounties for P0 (supply chain) and P1 (unauthenticated proxy access) findings. Security advisories committed to 5-business-day SLA.

**Clean version: v1.83.0+. Docker users remain unaffected.**

### GlassWorm — 400+ Repositories Compromised via VSCode/OpenVSX Extensions

Reported by Bleeping Computer and The Hacker News (March 2026, impact extending into April). The GlassWorm campaign abused **72 Open VSX extensions** with unclaimed namespaces — VS Code fork IDEs (Cursor, Windsurf, etc.) inherit Microsoft's extension recommendation lists but source from Open VSX, leaving many recommended extensions unregistered and claimable by attackers. Payloads dropped across 400+ GitHub repositories, npm packages, and VSCode/OpenVSX entries.

**Cursor patched December 2025. Google patched January 2026. Windsurf has not responded to disclosure. Windsurf users remain at risk.**

---

### Slopsquatting Threat Elevated — AI Agents Installing Without Human Review

No new in-the-wild exploitation confirmed as of April 8, 2026, but the threat model escalated significantly with the emergence of fully autonomous coding agents. Key risk delta from prior reporting:

- Autonomous agents (Claude Code, Devin, GitHub Copilot Workspace) can `pip install` hallucinated package names **without human review**, reaching production at machine speed.
- Agents running in CI/CD pipelines may execute in environments where `--require-hashes` is not enforced and no lockfile exists.
- The 20% hallucination rate for package names (576,000 samples, UTSA/Virginia Tech/U. Oklahoma, 2025) has not materially improved in 2026 model generations under agentic prompting patterns.

## Research & Advisories

### Adversa AI — Top Agentic AI Security Resources, April 2026

Adversa AI published a monthly curated list of April 2026 agentic AI security resources, highlighting the "Lethal Trifecta": **prompt injection + memory persistence + autonomous tool use**. When all three are present, a single injected instruction can persist across sessions, escalate through tool calls, and survive agent restarts. The combination is now considered the baseline threat model for any production agentic coding deployment.

---

### Vibe Security Radar — AI-Attributed CVEs Still Accelerating

Tracking CVE.org, NVD, GHSA, and OSV for AI tool commit signatures (Georgia Tech SSLab):

- **35+ new CVEs in March 2026** attributable to AI-authored code (up from 15 in February, 6 in January)
- Trend is roughly doubling each month
- Claude Code remains over-represented (always leaves co-author metadata); inline suggestion tools leave no trace; true figure estimated **5–10× higher** (~400–700 cases in the open-source ecosystem)

The monthly acceleration means the baseline security debt from AI-generated code is compounding faster than remediation capacity in most open-source ecosystems.

---

### CVE Watchtower Weekly Digest (March 30 – April 5, 2026)

SecurityOnline.info weekly digest noted a Chrome zero-day (**CVE-2026-5281**, actively exploited in the wild) alongside the AI framework wave. Electron-based AI IDEs (Cursor, Windsurf) that ship outdated Chromium remain at risk — OX Security previously documented 94 unpatched Chromium CVEs in both products. CVE-2026-5281 is likely weaponizable against these IDEs; no confirmed in-IDE exploit as of report date.

---

### MCP Rug-Pull Attacks — No Consent Re-Prompt on Tool Definition Changes

Red Hat and the Vulnerable MCP Project ([vulnerablemcp.info](https://vulnerablemcp.info)) both published updated advisories noting that the MCP protocol still has **no mechanism to re-prompt user consent when a server silently modifies its tool definitions**. A tool approved as a "read-only file browser" on Day 1 can be changed on the server side by Day 7 to include exfiltration capabilities — the client receives the new definition with no warning and continues to auto-invoke the tool.

Documented attack pattern: MCP servers approved during an initial agent setup session are later updated to add `exfil_api_keys`, `read_ssh_dir`, or `post_to_webhook` capabilities. All subsequent agent sessions invoke the new capabilities without re-consent.

**Status: No protocol-level fix in MCP spec. Mitigations: pin MCP server versions by commit hash; treat every server update as a new approval decision.**

## Key Mitigations

1. **Patch Flowise immediately** — CVE-2025-59528 (CVSS 10.0) is actively exploited; any internet-exposed instance should be treated as compromised until patched and audited.
2. **Upgrade LiteLLM to v1.83.0+** — especially if `enable_jwt_auth=true` is set; rotate all credentials on any host that ran v1.82.7 or v1.82.8.
3. **Pin `pull_request_target` workflows** — review all GitHub Actions workflows that trigger on untrusted forks; require maintainer approval before CI runs on external PRs.
4. **Treat MCP server updates as new approvals** — pin MCP server versions by commit SHA; audit tool definition changes before re-enabling.
5. **Enforce `--require-hashes` in agent-accessible pip environments** — prevents slopsquatting payloads from installing even if a hallucinated name is registered.
6. **Audit Windsurf deployments** — multiple unpatched Chromium CVEs with public PoCs plus unresolved GlassWorm extension risk; no vendor response to disclosures.
7. **Restrict `execute_code()` in agent frameworks** — PraisonAI, CrewAI, and Langflow all exposed unsandboxed code execution; ensure agent orchestration runs in containers without host credential access.
8. **Rotate GitHub tokens after any Copilot Codespace interaction with untrusted issues/PRs** — RoguePilot class of attacks require only an opened GitHub issue to trigger.
